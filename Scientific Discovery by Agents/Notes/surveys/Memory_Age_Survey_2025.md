# Memory Age Survey 2025

## 基本信息
- **论文**: Memory in the Age of AI Agents: A Survey — Forms, Functions and Dynamics
- **作者**: Yuyang Hu, Shichun Liu, Yanwei Yue, Guibin Zhang et al. (47 authors, 12 affiliations)
- **年份**: 2025
- **来源**: arXiv:2512.13564v2 (13 Jan 2026)
- **关键词**: agent-memory, survey, forms-functions-dynamics, multimodal-memory, multi-agent-memory

## 核心思想

本文是一篇大规模 Agent 记忆综述，配套资源库为 https://github.com/Shichun-Liu/Agent-Memory-Paper-List。文章的核心论断是：传统的"长/短期记忆"二分法已不足以刻画当代 Agent 记忆系统的多样性和动态性。为此提出从"形式（Forms）—功能（Functions）—动态（Dynamics）"三维统一视角重新组织该领域，并清晰界定了 Agent 记忆与 LLM 记忆、RAG、上下文工程三个易混淆概念之间的边界。

## 研究问题与动机

**Agent 记忆的正式定义**：将记忆系统形式化为演化状态 M_t，通过三个概念算子运作——形成（F: M_t, φ_t → M^form_{t+1}）、演化（E: M^form_{t+1} → M_{t+1}）、检索（R: M_t, o^i_t, Q → m^i_t）。关键区分在于：(1) Agent 记忆 vs. LLM 记忆——Agent 记忆几乎完全涵盖历史上称为"LLM 记忆"的大部分工作（如 MemoryBank、MemGPT 实质上是早期 Agent 记忆实例），但模型内部 KV 缓存管理、长上下文架构修改等内在于 Transformer 的机制被排除在外；(2) Agent 记忆 vs. RAG——RAG 倾向于访问外部静态知识源进行单次推理增强，Agent 记忆则维持内部、持久、自演化的记忆库，边界在 Agentic RAG 处最模糊；(3) Agent 记忆 vs. 上下文工程——Agent 记忆关注"Agent 知道什么、经历过什么、如何演化"的认知建模，上下文工程关注"如何在有限窗口内高效调度信息"的资源管理。

**历史脉络**：从 Memory Networks (2015) 和 NTM (2014) 引入可微分外部读写存储，到 RAG (2020) 和 RETRO (2022) 证明外部知识库可在推理时动态查询，再到 ReAct (2022) 将推理-行动轨迹作为短期工作记忆、Reflexion (2023) 将口头自我批判作为情景记忆、Generative Agents (2023) 用 25 个模拟角色证明观察-反思-计划循环可产生数月连贯社会行为，Agent 记忆在短短三年内经历了从附带功能到核心模块的转变。指出 2023 年之前"Agent"定义高度异质——有时调用一个计算器即算 Agent——导致术语混乱，近期才收敛到"LLM + 推理 + 规划 + 记忆 + 工具使用 + 多轮交互"的共识框架。

## 方法细节

**第一维——形式（Forms）**：三种主导实现。Token 级记忆按维度递进分为：Flat Memory (1D)（朴素截断、滑动窗口、LLM 摘要压缩等上下文内方案）、Planar Memory (2D)（图结构记忆如 Zep/AriGraph/A-MEM/G-Memory，边显式编码语义/时序/因果/逻辑关系）、Hierarchical Memory (3D)（层级总结/树状索引如 MemTree/MemGPT，在不同抽象粒度上存储和导航）。参数化记忆分为内部（通过微调/adapter 将记忆嵌入模型权重，如 MemLLM）和外部（记忆注入适配器如 Retroformer）。潜空间记忆通过生成-复用-变换三阶段运作：MemoryLLM 用自编码潜变量编码全局记忆，M+ 在潜空间中压缩并传递状态。

**第二维——功能（Functions）**：精细划分三类。事实记忆（Factual Memory）记录 Agent 交互中的客观知识——用户事实记忆（偏好/身份/关系如 MemoryBank）和环境事实记忆（API 文档/代码库/物理规律如 ChatDB）。经验记忆（Experiential Memory）按抽象层级递增分为：案例记忆（保留完整交互轨迹用于上下文学习如 Reflexion）、策略记忆（提炼跨任务推理模式如 ExpeL）、技能记忆（存储可执行代码/工作流如 Voyager 技能库）、以及混合记忆（Agent 自主决定记忆组织方式的"元记忆"如 A-MEM/Self-Controlled Memory）。工作记忆（Working Memory）分为单轮（单次工具调用内的中间结果暂存如 ReAct 推理链）和多轮（跨步骤的多步推理/多工具编排/子目标追踪如 HiAgent）。

**第三维——动态（Dynamics）**：记忆如何形成、演化和检索。原文将完整生命周期拆为 Memory Formation、Memory Evolution、Memory Retrieval 三个过程。形成阶段包含五种范式：语义摘要（将交互轨迹压缩为文本摘要，面临摘要漂移风险）、知识蒸馏（从经验中提炼可复用规则如 Zhao et al. 2024）、结构化构建（构造知识图谱/层级树等显式结构）、潜表示（Encoder 压缩为连续向量）、参数化内化（通过微调将知识写入模型权重）。演化阶段包含整合（跨集经验合并为通用Schema，面临过泛化/错误分组/过拟合风险）、更新（增量修改/冲突消解/事实更新）、遗忘（Ebbinghaus 遗忘曲线衰减/基于重要性的选择性遗忘/MemoryBank 的遗忘机制）。检索阶段包含四个子环节：检索时机与意图（何时触发/检索什么类型）、查询构建（当前输入常非最优检索查询，需 LLM 改写或子目标辅助）、检索策略（语义相似度/图遍历/混合/Agentic检索）、检索后处理（重排序/去重/摘要/融合）。

## 关键结果

形式维度代表：

**Token级图记忆**：Zep（双时态建模，区分有效时间和记录时间）、AriGraph（LLM 从文本观察中提取实体-关系三元组构建世界模型）、G-Memory（双图结构——洞察图与交互图之间双向遍历）、A-MEM（Agent自主决定记忆增删改操作）。**参数化记忆**：MemoryLLM（模型微调为自读写的显式记忆交互系统）、Retroformer（记忆注入适配器）。**潜空间记忆**：MemoryLLM（Transformer 自编码潜变量存储全局记忆）、M+（潜空间中压缩并传递上下文状态）。**经验记忆**：Voyager（Minecraft 中的可执行 JavaScript 技能库，3.3倍更多独特物品，15.3倍更快科技树进展）、ExpeL（系统对比成功与失败轨迹提取"经验法则"）、Reflexion（口头自我批判存储为情景记忆，HumanEval Pass@1 91% vs. GPT-4 80%）。**自动化记忆管理**：Agentic Memory/AgeMem（5种记忆操作作为可调用工具，三步GRPO训练），Kimi K2.5（主动记忆探索）。

功能维度代表：Voyager 技能库（可执行 JavaScript，Minecraft 中 3.3倍更多独特物品和 15.3倍更快里程碑进展）、ExpeL（对比成功vs失败轨迹提取判别性经验法则）、Reflexion（HumanEval Pass@1 从 GPT-4 80% 提升至 91%）、Generative Agents（25角色自主组织情人节派对，多信号记忆评分机制）。

动态维度代表：AgeMem（三步GRPO训练五种记忆操作）、Kimi K2.5（主动记忆探索与外部知识获取）、MEM1（RL优化记忆检索与写入联合策略）、Memory-R1（步骤级GRPO训练记忆管理）。

## 局限性

1. **记忆检索到记忆生成**：从"检索已有记忆"到"基于已有知识生成新记忆内容"的范式跃迁——Agent 不仅回忆过去，还主动推理缺失信息、填补知识空白，代表系统如 MemGen。
2. **自动化记忆管理**：摆脱手工设计的启发式策略（固定摘要周期/硬性过期规则），转向 Agent 自主决策何时形成/整合/遗忘/重检索。AgeMem 通过将五种记忆操作建模为可调用工具并用 GRPO 训练代表了该方向的第一步实践。
3. **强化学习与记忆深度融合**：RL 不仅是检索策略优化工具，更是记忆操作能力本身的训练框架——通过任务级和环境级奖励塑造 Agent 的写入/读取/遗忘的完整行为策略。代表系统：Mem-α（RL优化记忆检索与写入）、Memory-R1（步骤级GRPO训练记忆管理）、RMM（RL训练记忆读写策略）。
4. **多模态记忆**：融合文本/图像/音频/视频的多模态记忆存储与跨模态检索——从早期 VLM 简单图片标注到复杂的联合嵌入空间构建。代表系统：Ella（多模态记忆增强对话）、ViloMem（视觉-语言记忆）、JARVIS-1（Minecraft中的多模态记忆）。
5. **多 Agent 共享记忆**：从独立记忆孤岛到共享认知基底——定义共享vs私有的边界、处理并发写入冲突、支持角色敏感的访问控制。代表系统：ChatDev（CEO/CTO/程序员/测试员沿开发阶段共享结构化文档记忆）、MetaGPT（将共享记忆编码为 PRD/设计文档等标准化文档）。
6. **记忆驱动的世界模型**：记忆不仅存储事实，更构建 Agent 对环境的内部因果模型——支持反事实推测和前瞻规划，超越简单的历史回放功能。
7. **可信记忆**：检测并纠正记忆库中的错误/过时/矛盾信息，建立记忆审计追溯链，在参数化记忆场景中实现可靠的机器遗忘（machine unlearning）以支持隐私合规。
8. **人与认知科学连接**：将互补学习系统理论（快速情景系统与缓慢语义系统的结构性分离）、扩散激活（访问一个记忆自动预激活相关记忆）、记忆再巩固（检索使记忆变得不稳定可被修订）等人类记忆理论转化为 Agent 记忆的可操作机制设计原则。

## 核心贡献
Memory Age Survey 的核心贡献是提出 Forms / Functions / Dynamics 三维分类，将 agent memory 从传统长短期二分扩展为可分析其承载形式、用途和生命周期的统一框架，并系统比较其与 LLM memory、RAG、context engineering 的边界。

## 与综述的关联

该综述为本项目科学发现 Agent 记忆章节提供了目前最系统的"形式-功能-动态"三维分析框架。科学发现场景对三类记忆功能均有强烈需求——事实记忆（领域知识库如化学原理、蛋白质结构、反应路径数据库）、经验记忆（实验日志、失败轨迹、假设-验证循环、同行评审记录）、工作记忆（多步骤实验设计中试剂配比、中间产物表征、产率计算的实时状态管理）。经验记忆中的"案例-策略-技能"层级抽象与科学发现的知识演化路径高度对应——从单次具体实验案例提炼领域策略（如"该催化剂在此类底物中需低温条件"），再将策略固化为可复用的实验技能（如特定分析流程模板）。其提出的自动化记忆管理方向直接对应科学知识管理的核心挑战——何时整合多组实验证据形成通用假说（整合门控）；可信记忆方向对应科学记忆库的准确性保障（事实校验与矛盾检测），多 Agent 共享记忆对协作式科学发现系统（多角色实验室：实验设计Agent+执行Agent+文献Agent+评审Agent）的架构设计提供直接参考。
