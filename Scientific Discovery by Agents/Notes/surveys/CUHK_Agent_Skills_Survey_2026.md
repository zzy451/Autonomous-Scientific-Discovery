# CUHK Agent Skills Survey 2026

## 基本信息
- **论文**: A Comprehensive Survey on Agent Skills: Taxonomy, Techniques, and Applications
- **作者**: Yingli Zhou, Shu Wang, Yaodong Su, Wenchuan Du, Yixiang Fang, Xuemin Lin (The Chinese University of Hong Kong, Shenzhen)
- **年份**: 2026
- **来源**: arXiv:2605.07358 (8 May 2026)
- **关键词**: survey, skills, taxonomy, lifecycle, agent-ecosystem

## 核心思想
LLM驱动的Agent正从被动回复转向主动任务执行，但当Agent面对开放、长周期任务时，每次都从零开始进行推理和底层工具调用变得越来越低效、易错且难以维护。本文通过"Agent Skills"这一视角审视该挑战，将Skills定义为协调工具、内存和运行时上下文的可复用程序性构件。Agent负责高层推理与规划，Skills构成可靠、可复用、可组合的执行操作层。文章围绕Agent Skills生命周期的四个阶段——表示、获取、检索、演化——组织现有文献，并讨论了质量控制、互操作性、安全更新和长期能力管理等开放挑战。

## 方法细节

**核心定义**：Agent Skill = (M, R, C)三元组，其中M为根指令文档，R为辅助资源集(参考文档、可执行脚本、模板等)，C为适用条件(元数据、自然语言描述或嵌入向量)。

**四阶段生命周期分类体系**：

1. **Skill表示 (Representation, §III)**
   - 按R的资源配置分为三类：
     - Text-Backed：纯文本资源(参考、示例、模板、评分标准)
     - Code-Backed：可执行脚本和代码资源
     - Hybrid-Based：文本与可执行资源混合
   - 关键洞察：表示形式决定了后续检索和选择可用的信号类型。

2. **Skill获取 (Acquisition, §IV)**
   - 按获取来源分为四个族类：
     - Human-Derived：专家手动编写，精度高但可扩展性差
     - Experience-Derived：从Agent执行轨迹中提取，是目前研究最密集的路线，包含四个处理操作——选择、抽象/总结、记忆组织、程序性归纳与打包
     - Task-Derived：从当前任务需求中即时生成技能候选
     - Corpus-Derived：从外部文本/结构化资源(文档、代码库、接口)中提取
   - 关键洞察：四者互补，最强大的技能库需要组合使用。

3. **Skill检索与选择 (Retrieval & Selection, §V)**
   - 检索(缩减候选集)：密集嵌入检索、稀疏/关键词检索、生成式检索、结构感知检索(层次式、依赖图式)
   - 选择(决策执行)：上下文感知动态选择、技能组合、成本/效用感知选择、反馈驱动重排序
   - 关键洞察：检索解决候选召回问题，选择解决面向执行的决策问题，两者必须联合建模。

4. **Skill演化 (Evolution, §VI)**
   - 五个子阶段：技能修订→技能验证→策略耦合→仓库演化→运行时治理
   - 核心机制：反馈修改已命名的持久性构件，验证决定修改是否存活，仓库管理与策略耦合保证变化传播。
   - 关键洞察：演化不等同于积累；安全删除和版本回滚与新增同等重要。

**生态系统统计**：论文表 I 汇总了代表性技能平台规模，包括 SkillNet(300k+)、ClawHub(40k+)、SkillHub(80k+)、SkillsMP(700k+) 和 Skills.sh(90k+)。这些数字是论文截稿时的平台快照，应作为生态规模参考而非稳定常量。

## 核心贡献
CUHK Agent Skills Survey 2026 的核心贡献是把 agent skill 定义为可存储、可检索、可组合、可修订的程序性构件，并围绕表示、获取、检索/选择、演化四个生命周期阶段组织文献，为本综述提供通用技能工程框架。

## 与综述的关联

**CUHK Survey覆盖的内容**：
- Agent Skills的完整四阶段生命周期：从技能如何表示、如何获取、如何检索/选择到如何演化，提供了一个自洽的"技能工程"全栈视图。
- 技能生态系统基础设施：SkillNet、ClawHub、SkillsMP等平台及其在技能管理中的角色。
- 应用场景分类：软件工程、Web/GUI、对话系统、机器人、金融、医疗、游戏、社会模拟八大类。
- 开放挑战聚焦工程层面：抽象质量、触发条件弱、资源漂移、统一模式、因果诊断等。

**我们的综述可以覆盖的不同内容**：
- 面向科学发现的垂直领域：聚焦Skills/Agent系统在科研场景中的应用，而非通用的软件工程或Web任务。
- 科学工作流的特定挑战：实验可复现性、科学假设生成的质量评估、文献检索与引用正确性、实验数据管理的特殊性。
- 从Science-First视角审视Skills：CUHK的分类体系(表示/获取/检索/演化)可作为我们组织"科研Agent能力层"的参考框架，但我们关注的是这些能力在科学发现中如何具体体现。
- 跨领域对比：不同科学领域(材料、生物、化学、物理)对Agent Skills的需求差异。
- 基准与评估：面向科研任务效果的专门基准，而非通用的SkillsBench。

## 关键结果

1. **"程序性鸿沟"是核心动机**：工具访问不等于可靠执行，Skills是桥接"能做什么"到"如何做"的关键抽象。这一概念对理解为什么科研Agent需要超越简单的API调用有重要启发。

2. **四阶段生命周期框架可迁移**：表示→获取→检索→演化的分类体系清晰且通用，可作为我们综述中"科学Agent能力架构"的组织线索。

3. **Experience-derived acquisition是当前研究重点**：从执行轨迹中提取可复用技能是目前最活跃的研究方向，科研Agent在实验迭代中积累经验形成可复用实验技能的思路与此高度一致。

4. **技能选择是策略问题而非排序问题**：不能仅依赖语义相似度选择技能，必须考虑当前状态、组合约束、成本和效用——这对设计科研Agent的实验规划模块有直接启示。

5. **演化比积累更难**：当前系统更擅长增加新技能，而非安全地重写或退役旧技能——科研Agent同样面临知识更新和旧假设淘汰的问题。

6. **生态系统规模大但不统一**：技能格式、触发条件、依赖关系缺乏标准化，影响跨平台互操作性——科研Agent领域同样需要标准化的实验协议描述格式。

7. **评估是薄弱环节**：端到端成功指标无法分离各生命周期阶段的贡献，需要因果级诊断——我们的综述可以回应这一需求。

## 局限性
- **概念边界有些模糊**：部分被归为Skills的工作(如RAG、Tool Use)与技能概念的边界有时不够清晰，与传统的tool use和memory survey存在概念重叠。
- **科学发现场景覆盖不足**：尽管覆盖了软件工程、机器人等八个应用领域，但对科学发现这一特定垂直领域的讨论几乎缺失——这正是我们综述的切入点。
- **缺少统一的形式化质量度量**：虽然提出了开放挑战(抽象质量、弱触发条件等)，但未给出可操作的定量评估框架。
- **对安全性讨论较浅**：仅在开放挑战中简要提及PoisonedSkills和运行时治理，缺乏对Skills安全风险的深入系统分析。
- **历史线较短**：覆盖文献主要集中在2023-2026年，对更早期的可复用子程序、模块化AI等前置工作的历史脉络关注不足。
- **缺乏与认知科学的深度对话**：Skills作为可复用的程序性知识编码，与认知科学中的程序性记忆(procedural memory)和技能习得理论存在天然关联，但论文未充分展开这一跨学科讨论。
