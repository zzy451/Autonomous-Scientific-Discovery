# Mozi 2026

## 基本信息

- **论文**: Mozi: An Agentic AI System for Collaborative Drug Discovery
- **年份**: 2026
- **来源**: arXiv:2603.03655
- **系统名称**: Mozi
- **关键词**: drug discovery, supervisor-worker, skill graph, human-in-the-loop, multi-agent governance

## 摘要要点

Mozi 是面向药物发现长流程的 agentic AI system。它不是单一 drug-design model，而是一个 supervisor-worker control plane，将多个 specialist agents、工具、数据契约、状态管理和 human-in-the-loop checkpoints 组织在一起。论文强调 stateful skill graphs，使系统能够在复杂药物发现任务中复用技能、跟踪状态并进行治理。

## 方法动机

药物发现任务跨越靶点选择、分子设计、数据处理、模型预测、候选排序和验证决策。单 agent 很难持续管理上下文、工具状态和跨阶段依赖。Mozi 的核心动机是用 agentic control plane 解决长流程 drug discovery 的协同、状态和可审计问题。

## 方法设计

系统流程可概括为：

1. Supervisor 接收高层 drug discovery goal。
2. 将任务拆分为 worker agents 可以执行的子任务。
3. 每个 worker 绑定特定工具、数据接口或领域能力。
4. stateful skill graph 记录可复用步骤、依赖、输入输出契约和执行状态。
5. Supervisor 根据中间结果和失败反馈重规划。
6. human-in-the-loop checkpoints 对关键候选、风险和决策进行检查。

这里的演化对象不是单个分子候选，而是 skill graph、control plane 和长流程治理能力。

## 实验与结果

论文展示 Mozi 在药物发现相关任务中组织 specialist workers 和工具链，强调数据契约、反思式重规划和人工检查点对可靠性的作用。它更像 drug discovery governance/harness than one-shot design model。

## 局限性

- 是否能直接产生经湿实验确认的新药候选仍需更多外部验证。
- supervisor-worker 系统会带来通信、状态一致性和错误传播成本。
- skill graph 的构建和维护需要工程投入。
