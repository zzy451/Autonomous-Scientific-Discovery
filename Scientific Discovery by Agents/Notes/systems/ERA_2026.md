# ERA 2026

## 基本信息
- **论文**: An AI system to help scientists write expert-level empirical software
- **作者**: Eser Aygun, Anastasiya Belyaeva, Gheorghe Comanici, Marc Coram, Hao Cui, Jake Garrison, Renee Johnston, Anton Kast, Cory Y. McLean, Peter Norgaard, Zahra Shamsi, David Smalling, James Thompson, Subhashini Venugopalan, Brian P. Williams, et al.
- **年份**: 2026
- **来源**: Nature, published 2026-05-19, doi:10.1038/s41586-026-10658-6; arXiv:2509.06503v3; Google Research publication page
- **关键词**: empirical software, computational discovery, tree search, code generation, scorable scientific tasks, Google Research, Google DeepMind

## 核心思想

ERA (Empirical Research Assistance) 要解决的问题是：许多科学发现依赖 empirical software，也就是以某个可测质量指标为目标、通过计算实验不断改进的科研软件。但这类软件通常需要大量人工设计、调试、比较和领域知识注入，成为计算实验和假设探索的瓶颈。

论文提出 ERA，把科学软件开发建模为可评分搜索问题：给定任务、评价函数和研究想法，系统让 LLM 生成或改写代码，在 sandbox 中运行并得到分数，再用 tree search 决定下一步探索哪些候选代码。它的核心不是一次性生成代码，而是在“研究想法注入 - 代码变异 - 执行评分 - 搜索选择 - 复用重组”之间形成迭代闭环。

## 方法细节

ERA 的输入是 scorable task、可运行的评价环境、已有代码或 baseline，以及来自论文、教材、搜索结果、Gemini Deep Research 或 AI co-scientist 的研究想法。LLM 根据这些信息改写代码，sandbox 运行代码并返回分数。Tree search 维护候选代码树，在 exploitation 与 exploration 之间平衡，选择值得继续扩展的节点。

论文特别强调两种能力：

- **研究想法注入**：用户可直接提供专家建议，系统也可从高引用论文、专门教材、搜索结果或 AI research tools 中获得 idea summary，再把它们加入 prompt。
- **idea recombination**：系统不仅复现单个已知方法，还会把多个成功实现的思想组合成 hybrid strategy。论文在多个任务中显示，重组往往优于单个 base method。

从本综述角度看，ERA 是典型的 evolutionary/search-based workflow。它没有显式称为 multi-agent system，但它和 AlphaEvolve、Jupiter、SciNav 一样，把科学执行面转化为大量候选代码的生成、测试、选择和迭代。

## 关键结果

- 在 Kaggle Playground benchmark 上，ERA 的 tree search 明显优于单次 LLM 调用、best-of-1000 LLM calls 和 AIDE；benchmark 包含 **16 个**2023 season playground competitions。
- 在 single-cell batch integration benchmark 中，ERA 在 holdout OpenProblems datasets 上生成的 BBKNN (TS) 相比最佳 published method ComBat 有 **14% overall improvement**，并在与对应已发表方法比较时，**8/9** 个方法的 ERA 实现 overall score 更高。论文摘要称 ERA 发现 **40** 个超过 public leaderboard 顶尖人类方法的新方法。
- 在 COVID-19 hospitalization forecasting 的回顾性评估中，ERA 的 Google Retrospective model 平均 WIS 为 **26**，优于 CovidHub Ensemble 的 **29**；系统性探索得到 **14** 个超过 official CovidHub-ensemble 的策略。
- 在 numerical integration 任务中，ERA 针对标准 `scipy.integrate.quad()` 失败的积分构建替代算法；held-out set 上最佳 routine 在 **17/19** 个积分上误差低于 3%。
- 论文还报告了 geospatial segmentation、ZAPBench neural activity prediction、GIFT-Eval time series forecasting，以及 gravitational radiation/cosmic strings 相关 symbolic mathematics 发现等案例。

## 局限性

ERA 依赖可运行的评价函数和可量化质量指标，因此更适合 scorable scientific tasks。它能在明确 benchmark 或 leaderboard 上快速探索，但不等于能独立定义科学问题、判断发现的重要性，或完成不可量化的理论解释。

系统比单次 prompting 更耗计算，因为每个 search node 都需要 LLM 推理和 sandbox 执行。论文补充材料也说明不同 benchmark 的 per-node token、执行时间和硬件需求不同，整体成本随搜索节点数近似线性增长。

ERA 的成果需要人类检查代码是否忠实实现了给定方法，尤其是当系统声称复现或重组已有研究想法时。论文在若干实验中做了专家人工检查，但这也说明系统尚不能完全自动保证 method fidelity。

此外，部分任务是 retrospective 或 leaderboard-style evaluation；它们证明了 ERA 在可评分计算任务上的强搜索能力，但不能直接推出所有科学发现都能被转化为同样清晰的分数优化问题。

## 核心贡献

ERA 的核心贡献是把 empirical scientific software 写作转化为可迭代、可评分、可重组的搜索过程，使科学 agent 不只是“写一段代码”，而是能够在大量候选实现中持续探索、比较、回退和组合研究思想。

## 与综述的关联

在 Scientific Workflow 中，ERA 覆盖 Experiment Design、Execution、Result Analysis、Verification 和 Evolution。它最适合放在“software-mediated scientific discovery”或“computational discovery workflow”一节，说明许多科学发现的执行层不是湿实验，而是可运行代码和可评分 benchmark。

在 Skill Lifecycle 中，ERA 对应：

- **Representation**：baseline code、task description、research idea summaries；
- **Execution**：sandbox code execution；
- **Composition**：把论文方法、专家建议和已有实现组合为 candidate software；
- **Evolution**：tree search、candidate mutation、idea recombination；
- **Verification**：quality metric、leaderboard、holdout evaluation、manual inspection。

Evidence Role 应标为 **Direct + Infrastructure**。ERA 直接展示了科研软件能力如何通过搜索和评分演化，同时也为 Gemini for Science 的 Computational Discovery 提供底层研究证据。

## 写作时避免

- 不要写成 ERA 能自动完成所有科学发现；它主要解决可评分科研软件生成与优化。
- 不要把 leaderboard improvement 等同于真实世界部署效果。
- 不要把生物医学案例写成实验 protocol；本综述中只取其计算工作流和评价机制。
- 不要忽略人类检查和任务形式化在 ERA 成功中的作用。
