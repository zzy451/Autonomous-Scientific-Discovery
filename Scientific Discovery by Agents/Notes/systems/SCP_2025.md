# SCP 2025

## 基本信息

- **论文**: SCP: Accelerating Discovery with a Global Web of Autonomous Scientific Agents
- **年份**: 2025
- **来源**: arXiv:2512.24189
- **系统名称**: Science Context Protocol
- **关键词**: science context protocol, autonomous scientific agents, federated servers, resource integration, experiment lifecycle

## 摘要要点

SCP 提出 Science Context Protocol，目标是用开放协议连接软件工具、模型、数据集和物理仪器，使 autonomous scientific agents 能跨平台、跨机构发现、调用和组合科学资源。论文强调两个支柱：Unified Resource Integration 和 Orchestrated Experiment Lifecycle Management，后者通过 centralized SCP Hub 与 federated SCP Servers 管理实验注册、规划、执行、监控和归档。

## 方法动机

许多 agentic science 系统局限在单实验室、单平台或定制 workflow 中。科学发现需要跨 dry lab / wet lab、模型、数据库和仪器的协同。SCP 的直觉是：ASD 需要类似互联网协议层的科学上下文标准，让 agents 能可靠发现资源、调用资源、保存 provenance 并管理实验生命周期。

## 方法设计

SCP 的系统设计包括：

1. 用通用 schema 描述 scientific resources，如 tools、models、datasets、physical instruments。
2. agent 或应用通过协议发现、调用和组合这些资源。
3. centralized SCP Hub 负责注册、认证、授权和全局上下文管理。
4. federated SCP Servers 接入不同机构或平台的实际资源。
5. 协议层管理实验生命周期：registration、planning、execution、monitoring、archival。
6. 系统记录 traceable workflows 和 provenance，支持复现与审计。

它的核心不是某个单一 agent，而是开放科学能力网络的协议底座。

## 实验与结果

论文基于 SCP 构建科学发现平台，提供超过 1,600 个 tool resources，并展示跨多种 use cases 的大规模异构 AI systems 与人类研究者协作。作者强调 SCP 降低 integration overhead，提高 reproducibility，并为 multi-institution agent-driven science 提供基础设施。

## 局限性

- SCP 是协议/平台型工作，严格科学发现结果需要具体应用进一步验证。
- 开放网络会带来认证、授权、资源质量、恶意节点和治理问题。
- 协议标准化不能自动保证 scientific correctness。
