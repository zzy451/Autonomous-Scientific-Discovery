## 核心思想
基于 LLM 的智能体越来越多地被部署处理流式任务，但它们通常仍是一对一的问题求解器，无法从过去的交互中学习。从经验中提炼的可复用技能为自我进化提供了自然基础，而高质量技能管理（skill curation）成为关键瓶颈。现有方法依赖人工管理、启发式操作或短视技能适应，难以从间接延迟反馈中学习复杂的长期管理策略。我们提出 SkillOS，一个基于经验驱动的 RL 训练方案，用于在自我进化智能体中学习技能管理。SkillOS 将一个冻结的智能体执行器（检索并使用技能）与一个可训练的技能管理器（从累积经验中更新外部 SkillRepo）配对。为了提供学习信号，我们在基于技能相关性的分组任务流上训练，早期任务轨迹更新 SkillRepo，后续相关任务评估这些更新。我们进一步设计复合奖励以更好地将下游执行器反馈归因于管理决策。在多轮智能体任务和单轮推理任务上，SkillOS 在效果和效率上都持续优于无记忆和强记忆基线，学习到的技能管理器能泛化到不同执行器骨干和任务域。分析表明，学习到的管理器产生更有针对性的技能使用，而不断进化的 SkillRepo 随时间发展出更丰富的内部结构和更高级的元技能。

## 研究问题与动机
a) 在真实世界的流式部署中，智能体面临一系列持续到达的任务，应当从过去经验中不断积累、提炼和复用知识，而非每次都从零开始。可复用技能是自我进化的最佳载体，然而技能管理（从经验中提取高质量教训并整合到技能集合中）是核心瓶颈。作者提出 SkillOS 的核心驱动力是：**让技能管理策略能够从长期任务执行反馈中通过 RL 自动学习**。

b) 现有方法的不足：
- **人工管理技能**（如 Anthropic 的技能仓库）：需要大量人类专家投入，无法扩展到智能体可能遇到的任务多样性
- **启发式方法**（如 ReasoningBank、MemP、A-Mem）：依赖固定规则决定记忆操作，无下游性能反馈信号，无法适应执行器的实际需求
- **现有 RL 方法**：要么训练智能体使用技能（SkillRL、D2Skill），要么在短任务流内优化技能操作（ARISE、UMEM），学习信号密度低，且无法掌握技能更新和删除等复杂管理操作

c) 研究假设：如果将技能管理构建为一个长时域的学习问题——在分组相关任务上训练，让管理决策通过其对后续相关任务执行器表现的因果影响来获得信号——那么 RL 训练的策略可以学会提取、更新、删除和精炼技能，产生比零样本或启发式管理更强的自我进化能力。

## 方法细节
a) **系统架构：多智能体模块化设计**

**组件**：
- **SkillRepo**：外部技能仓库 S_t，包含 N_t 个可复用技能。每个技能使用 Markdown 格式（YAML frontmatter 含 name 和 description + Markdown 指令体），通过文件 I/O 操作管理（类 OS 设计）
- **Agent Executor (π_L，冻结)**：接收任务 x_t，通过 BM25 从 SkillRepo 检索 Top-K 个相关技能 (K=5)，基于当前环境观察和检索到的技能采样动作。使用 ReAct（智能体任务）/ CoT（推理任务）
- **Skill Curator (π_S，可训练)**：在执行器完成任务后，观察轨迹 ξ_t、自判断正确性信号 1_ξ_t、检索到的相关技能集，生成结构化管理操作序列 c_t = (u_1, ..., u_M)，每个操作为 insert_skill / update_skill / delete_skill 之一

**工作流**：新任务到达 → BM25 检索相关技能 → 执行器用技能解决问题 → 管理器基于轨迹更新 SkillRepo → 更新后的 SkillRepo 用于下一任务

b) **RL 训练方案**：

**训练实例构建**：
- Stage 1 (潜在属性标注)：用 Gemini-2.5-Pro 为每个任务 x_i 标注 5 维属性 Z_i = (Topic, Skills/Capabilities, Concepts/Theorems, Heuristic Strategies, Common Pitfalls)
- Stage 2 (分组构建)：基于 phrase-level 软 Jaccard 相似度将任务分组。门控条件包括：共享基础（匹配概念数 ≥ κ_C、技能数 ≥ κ_S）、共享推理（匹配启发式 + 陷阱数 ≥ 1）、非近似重复（话题相似度 ≤ θ_T）、不太无关（总体相似度 ≥ σ_min）、递进性（引入新概念或技能）、课程方向
- 智能体任务（ALFWorld/WebShop）：利用默认任务类型注释作为自然分组
- 推理任务（DeepMath-103K）：管道构建分组，含难度课程信号

**复合奖励函数**：
```
r = r_task (任务结果) + λ_f·r_fc (函数调用有效性) + λ_u·r_cnt (内容质量) + λ_c·r_comp (压缩/简洁性)
```
- r_task：后续任务的平均成功率（第一个任务用空 SkillRepo 作为基线，不计入）
- r_fc：生成的有效函数调用比例
- r_cnt：外部评判器（Qwen3-32B）对技能质量的评分（抽象性、可复用性、可行动性、忠实性 4 维）
- r_comp：1 - |S_i|/|χ_i|（仓库大小/上下文长度的比值），抑制逐字复制轨迹
- λ_f=1.0, λ_u=0.1, λ_c=0.05

**策略优化**：
- 使用 GRPO（Grouped Reward Policy Optimization），学习率 1e-6，batch_size=32，group_size=8
- 对每个任务组采样 N 个独立 rollout，计算优势 A_n = r_n - mean(r)，裁剪代理目标，丢弃 KL 项以鼓励探索
- 训练在 16 块 H100 GPU 上的 verl 框架进行，ALFWorld ~3 天，推理 ~2.5 天，WebShop ~5 天
- 基础模型：Qwen3-8B（管理器和训练时执行器）

c) **关键设计决策**：
- 执行器冻结，仅训练管理器——解耦贡献，模块化可迁移
- 分组任务流——将延迟反馈转化为学习信号，暴露管理器到更长技能演化轨迹
- 复合奖励——将间接的环境信号分解为可归因的中间信号

## 与其他方法对比
a) 本质不同：
- vs 人工管理（Anthropic Skills）：从人类专家知识 → 从任务执行经验中自动学习
- vs 启发式方法（ReasoningBank、MemP）：从固定规则驱动 → RL 从下游任务反馈学习
- vs 短时域 RL（ARISE、UMEM）：从局部适应 → 长期分组流式训练，包含 update/delete 等复杂操作
- vs SkillRL/D2Skill：从训练智能体使用已有技能 → 训练智能体管理技能本身
- 关键机制区分：SkillOS 通过**分组相关任务**使前后任务间存在因果依赖，早期管理决策的效果通过对后续任务表现的影响来衡量

b) 创新点：
- 首次将技能管理构建为长时域 RL 问题，支持插入/更新/删除全谱操作
- 分组任务流训练实例构造方法（含潜在属性标注 + 阶段式群组构建）
- 复合奖励设计（任务结果 + 函数调用有效性 + 内容质量 + 压缩性）解决信用分配问题
- 模块化设计：管理器到执行器的解耦泛化

c) 适用场景：
- 流式任务部署（需要持续自我进化）
- 多任务环境（技能可跨任务复用）
- 资源受限场景（通过技能管理减少重复探索）

d) 方法对比总结：

| 维度 | 无记忆 | 启发式记忆 | 短时域 RL | SkillOS |
|------|--------|-----------|----------|---------|
| 学习机制 | 无 | 固定规则 | RL (局部) | RL (长期分组) |
| 管理操作 | — | insert 为主 | insert 为主 | insert/update/delete 全谱 |
| 反馈信号 | 无 | 无 | 即时 | 延迟 + 复合 |
| ALFWorld Avg SR (Qwen3-8B) | 47.9 | 49.7-55.7 | — | 61.2 |
| WebShop SR (Qwen3-8B) | 9.8 | 11.4-12.0 | — | 16.5 |
| 推理 Avg Acc (Qwen3-8B) | 69.6 | 69.1-69.6 | — | 73.8 |
| 泛化性 | — | 弱 | 弱 | 强（跨执行器/跨任务） |

## 关键结果
a) 实验设置：
- **数据集**：ALFWorld（140 个评估任务，6 种子类型）、WebShop（500 测试指令）、推理任务（AIME24 30题、AIME25 30题、GPQA-Diamond 198题）
- **执行器**：Qwen3-8B（训练时）、Qwen3-32B、Gemini-2.5-Pro、Gemini-3.1-Flash-Lite（测试时泛化）
- **基线**：No Memory、ReasoningBank、MemP、SkillOS-base（无 RL 训练的管理器）、SkillOS-gemini（Gemini-2.5-Pro 作管理器）
- **指标**：Success Rate / Accuracy（效果）、Steps / Tokens（效率）
- 所有结果报告 3 次运行均值 ± 标准差

b) 关键数据（以 Qwen3-8B 执行器为例，除非另有说明）：

**ALFWorld**：
- SkillOS 平均 SR: 61.2%，相比最强外部基线 ReasoningBank (55.7%) 提升 +9.8% 相对
- 相比 No Memory (47.9%) 提升 +13.3 绝对点
- 效率：18.9 steps vs No Memory 21.1 (-10.4%)
- 更强的执行器：Gemini-2.5-Pro 执行器下 SR 达 80.2%（+9.5 点 vs SkillOS-base）

**WebShop**：
- SkillOS SR: 16.5，Score: 40.6，Steps: 19.4
- 相比最强基线 MemP (SR 12.0, Score 35.7) 全面提升
- Gemini-2.5-Pro 执行器下：SR 41.3, Score 56.0

**推理任务**：
- SkillOS Avg Acc: 73.8%（AIME24 80.0, AIME25 76.7, GPQA 64.6）
- 相比 No Memory (69.6%) 提升 +4.2 点
- Gemini-2.5-Pro 执行器下 Avg Acc 达 88.6%

**跨洞察**：
- 训练后的 8B 管理器超越了 Gemini-2.5-Pro 直接作管理器（SkillOS-gemini），证明针对性 RL 训练可超越原始模型规模
- 智能体任务收益 > 推理任务收益（前者有更明显的程序性规律可捕获）
- 管理技能使用更精准：技能使用率 100%（基线 87.9%），成功使用率 88.6%（基线 53.6%），平均每任务使用技能数更少

c) 最优势场景：
- 多步骤交互式智能体任务（导航/操作/状态推理）：程序性规律（动作排序、探索策略、恢复行为）可被提炼为可复用技能
- 不同执行器骨干泛化：训练时用 Qwen3-8B，测试时在所有更强执行器上持续提升
- 跨任务迁移：Reasoning→Agentic 任务迁移效果尤其好（抽象推理策略如分解/验证可跨域复用）

## 局限性
- **检索机制**：使用 BM25 关键词检索，非可学习的密集检索器，可能限制大规模 SkillRepo 下的召回
- **技能表示简化**：技能为单文件 Markdown，丢弃了 Anthropic SKILL.md 格式中的脚本/外部资源和层级化组织
- **执行器冻结**：管理器只能通过写入 SkillRepo 的内容来影响系统，与执行器之间的错配完全由管理器吸收
- **分组构建依赖 LLM 标注**：属性注释和分组质量依赖于 Gemini-2.5-Pro 的标注能力
- **评估任务范围**：当前限于 ALFWorld/WebShop/数学推理，未测试更开放域的科学发现任务

## 学习与应用
a) **开源情况**：论文未声明代码将开源，但给出了详细的提示模板（附录 A）、超参数（附录 B）和架构，可复现。训练基于开源框架 verl 和公开模型 Qwen3-8B

b) **关键实现细节**：
- RL 超参数：学习率 1e-6，batch_size=32，GRPO group_size=8，temperature=1.0，max_prompt_len=16384，max_response_len=4096
- 训练步数：ALFWorld 60 步，WebShop 50 步，推理 100 步
- 分组大小：ALFWorld/WebShop 各 10 任务/组，推理随机 5-12 任务/组
- Top-K 技能检索：K=5
- 奖励权重：λ_f=1.0, λ_u=0.1, λ_c=0.05
- 内容质量评判：Qwen3-32B（外部评判器）
- 正确性信号：LLM-as-a-Judge，使用冻结执行器骨干
- 分组建设的相似度编码器：all-MiniLM-L6-v2

c) **可迁移性**：
- 管理器-执行器解耦设计使训练后的管理器可直接配对不同执行器，无需重训练
- 分组训练方案可应用于任何有任务属性标记的流式任务序列
- 复合奖励框架可扩展到其他需要信用分配的多步决策问题
- 管理器的操作空间可扩展（当前为 insert/update/delete，可加入 compose/link 等）

## 核心贡献
a) **一句话核心思想**：用 RL 在分组相关任务流上训练技能管理策略。

b) **速记版 Pipeline**：
1. 将训练任务按潜在属性标注并分组（使组内任务技能相关）
2. 每个任务组内顺序执行：执行器检索技能完成任务，管理器观察轨迹后生成技能操作（插入/更新/删除）
3. 用复合奖励（后续任务表现 + 操作有效性 + 内容质量 + 压缩性）评估本轮技能管理质量
4. 通过 GRPO 更新管理器策略，使管理决策优化长期任务表现
5. 冻结的管理器可迁移到不同的执行器骨干和任务域

c) **与本综述的具体关联**：
- 属于智能体**技能自我进化**的核心工作，填补了"技能管理如何自动化"的关键空白
- 可放在综述的"Self-Evolving Agents"或"Skill Lifecycle Management"章节
- 支持论点：技能管理的自动化（从提取到更新到删除）是可扩展技能生态系统的必要条件
- 与 SSL（技能结构化表示）形成互补：SSL 定义技能的静态解耦结构，SkillOS 定义技能的动态演化策略

## 与综述的关联
SkillOS 在综述框架中处于**技能演化与管理**的位置——它回答的是"技能如何从经验中自动生长、精炼和淘汰"的问题。在科学发现智能体的语境下，SkillOS 的管理训练范式可以直接应用于：科学实验技能库的自动构建、实验策略的跨领域迁移、以及从失败实验中提炼可复用知识的自动化。它和 ARA（研究产物的结构化协议）在理念上一致——都强调让机器（而非人类）成为知识组织和进化的主要执行者。
