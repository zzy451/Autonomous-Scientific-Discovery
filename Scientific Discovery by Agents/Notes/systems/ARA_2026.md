## 核心思想
科学出版将一个分支化、迭代的研究过程压缩为线性叙事，丢弃了大部分过程中发现的内容。这种编制带来了两种结构性成本：(1) 叙事税——失败的实验、被拒绝的假设和分支探索过程被丢弃以适配线性叙事；(2) 工程税——足以说服审稿人的论述与智能体可执行规范之间的差距，导致关键实现细节被遗漏。这些成本对人类读者可容忍，但当 AI 智能体需要理解、复现和扩展已发表工作时则变得致命。我们提出智能体原生研究产物（ARA），这是一个协议，用围绕四层的智能体可执行研究包取代叙事性论文：科学逻辑、带完整规范的可执行代码、保留被编制丢弃的失败记录的探索图，以及将每个声明锚定在原始输出中的证据层。三个机制支持该生态系统：实时研究管理器在工作过程中捕获决策和死胡同；ARA 编译器将遗留 PDF 和代码库转化为 ARA 格式；ARA 原生评审系统自动执行客观检查。在 PaperBench 和 RE-Bench 上，ARA 将问答准确率从 72.4% 提升到 93.7%，将复现成功率从 57.4% 提升到 64.4%。保留的失败轨迹加速了进展，但也可能约束能力更强的智能体走出前人的探索盒子。

## 研究问题与动机
a) 科学出版将丰富、分支化的研究过程编译为线性叙事，以满足人类读者和评审者的阅读习惯。但这种编制是**有损的**：每个月被测试和拒绝的假设、通过试错发现的实现技巧、被权衡的设计替代方案——这些"过程知识"在叙事中被系统性抹去。当 AI 智能体成为研究的主要消费者时（需要理解领域、复现实验验证发现、扩展现有方法），它们恰恰需要这些被丢弃的知识。核心驱动力是：**AI 智能体已成为科学研究的一等参与者，研究产物必须从"为人类阅读优化的 PDF"转变为"为智能体操作优化的可执行知识包"**。

b) 现有方法的痛点（两种结构性税收）：
- **叙事税 (Storytelling Tax)**：研究不是线性进行的——它分支、回溯、在得出可发表结果前积累了大量失败的实验和错误方向。但在 RE-Bench 的 24,008 个智能体运行中，失败的运行占总美元成本的 90.2%（和 token 的 59.2%），失败与成功的中位 token 比为 113 倍。没有先前失败记录的智能体必须独立重新发现每个死胡同。同样丢失的还有人类判断信号——每次拒绝、修改和认可都是"什么是好研究"的偏好信号，但叙事编制丢弃了这些。
- **工程税 (Engineering Tax)**：论文以说服人类审稿人的精度书写（产生信念），但智能体需要执行的精度（产生正确执行）远高于此。在 PaperBench 对 23 篇 ICML 2024 论文的 8,921 个专家标注复现需求中，仅 45.4% 在 PDF 中被完全指定。代码开发是最不足的类别（仅 37.3% 充分），缺失的超参数占所有差距的 26.2%。
- 现有努力（FAIR 原则、RO-Crate、Nanopublications、AGENTS.md）各自解决片段问题，但没有一个协议联合结构化科学逻辑、可执行代码和探索历史。

c) 研究假设：如果研究产物的知识按四个维度（科学逻辑、可执行代码、探索历史、经验证据）分别结构化存储，并提供跨层溯源链接（声明→实验→代码→证据），那么智能体可以直接查询结构化声明、执行声明性规范并利用完整决策历史——实现"研究产物不是被阅读，而是被操作"。

## 方法细节
a) **ARA 协议的核心架构（四层文件系统本体）**：

**PAPER.md**：入口清单文件，YAML frontmatter + 层索引，~500 tokens 即可让智能体判断相关性。

**第一层：认知层 (/logic/)**
- problem.md：定义研究空白和核心洞察
- solution/：架构、算法和收敛关键启发式
- claims.md：可证伪声明，含 Statement、Status、Falsification criteria、Proof 指针
- experiments.md：验证计划
- related_work.md：类型化依赖（imports 注入先验定义、bounds 传播约束到超参数搜索、baseline 条目支持自动回归检测）

**第二层：物理层 (/src/)**
- 算法贡献使用 kernel mode：仅核心模块含类型化 I/O 签名（通常比完整仓库小 1-2 个数量级）
- 系统贡献使用 repository mode：保留完整实现，通过 index.md 清单映射每个源文件到 ARA 组件
- configs/：每个超参数附原理说明和搜索范围
- environment.md：锁定依赖、硬件和种子

**第三层：探索图 (/trace/)**
- exploration_tree.yaml：完整的研究有向无环图（DAG），5 种节点类型（question、decision、experiment、dead_end、pivot），嵌套表示父子边，also_depends_on 字段捕获汇聚点
- dead_end 节点保留被论文丢弃的假设、失败模式和教训
- 功能类似于"研究的 git log"

**第四层：证据层 (/evidence/)**
- results/：机器可读的指标表和生成数据（精确值 + 来源注释）
- logs/：训练曲线、资源使用、诊断
- 仅含输出，声明验证链为 claims.md → experiments.md → /evidence/
- 可分离访问控制：验证智能体可获得代码核心和算法描述而隐藏证据层，防止通过复制期望值造假

**跨层溯源绑定 (Forensic Bindings)**：claims.md 中的 ∀ statement 链接到 experiments.md 中的验证实验，实验链接到 /src/ 中的代码和 /evidence/ 中的原始结果。

b) **三个生态系统支持机制**：

**实时研究管理器 (Live Research Manager)**：
- 一个智能体技能（自然语言规范，加载到编码智能体上下文即转化为领域专用智能体）
- 三阶段流水线，在每次研究者-AI 对话结束时运行：
  1. Context Harvester：扫描会话记录（对话历史、工具输出、实验结果、代码 diff），提取研究显著事件
  2. Event Router：分类为 7 种类型（decision、experiment、dead_end、pivot、claim、heuristic、observation），标注认知来源（user/ai-suggested/ai-executed/user-revised），写入对应 ARA 层
  3. Maturity Tracker：当达到关键里程碑时将分阶段观察结晶为正式条目，传播 pivot 引起的变更
- 跨会话连续性：管理器无状态，产物本身携带记忆。会话结束写简短记录，下个会话读索引并仅呈现与当前任务相关的内容

**ARA 编译器 (ARA Compiler)**：
- 一个智能体技能，将任意遗留研究源（PDF、代码仓库、数据集、专家标注、实验轨迹）翻译为完整 ARA
- 自顶向下四阶段生成：
  1. Semantic Deconstruction：剥离叙事框架，暴露原始研究内容
  2. Cognitive Mapping：填充 /logic/（动机链、可证伪声明、解决方案结构）
  3. Physical Grounding：生成 /src/（注释配置、类型化代码桩），如有代码仓库则进行代码-论文对账（code-paper reconciliation）
  4. Exploration Graph Extraction：重构研究 DAG
- 迭代精炼：ARA Seal Level 1 验证反馈驱动 generate→validate→fix 循环（通常 2-3 轮）
- 源感知富集：辅助源路由到最直接填充的层，集体推断从同域已编译 ARA 中检索模式

**ARA 原生评审系统 (ARA-Native Review System)**：
- 三阶段审查管线（类似 CI/CD）：
  - Stage 1（概念验证，分钟级）：ARA Seal Level 1（结构完整性）+ Level 2（论证严谨性），自动化检查
  - Stage 2（实证验证，小时-天级）：ARA Seal Level 3（执行可复现性），在有计算预算约束下方向性复现中心声明
  - Stage 3（人类评审，天-周级）：人类审稿人接收 CI 报告和实证评审报告，聚焦于机器无法回答的问题——重要性、新颖性和品味
- ARA Seal 三级验证凭证：
  - Level 1：结构完整性（schema 合规、跨层引用可解析、必需字段完整）——确定性检查
  - Level 2：论证严谨性（证据相关性、可证伪性质量、方法论严谨性、范围校准、论证连贯性、探索完整性）——6 维锚定评分 1-5
  - Level 3：执行可复现性（沙箱化编码智能体，方向性检查，计算预算约束）——可与证据层隔离

c) **关键设计原则**：
- 知识优于叙事（Knowledge over Narrative）：组织化的演进知识是首要科学对象，叙事论文是编译视图
- 渐进披露（Progressive Disclosure）：智能体仅加载与当前任务相关的层/文件
- 信息密度最大化：主观限定词和叙事连接词被剥离，带判断的声明确保来源而非修辞

## 与其他方法对比
a) 本质不同：
- vs PDF + GitHub + 实验追踪器（MLflow/W&B）：即使同时使用，知识仍被孤立在三个未链接的格式中（声明、代码、证据之间无交叉引用），ARA 提供跨层溯源绑定使连接显式且机器可遍历
- vs FAIR 原则/RO-Crate/Nanopublications：各自覆盖 1-2 个维度（数据元数据/归档打包/原子声明），但无人提供执行语义或捕获决策历史
- vs 科学工作流引擎（Snakemake/Nextflow/CWL）：编码执行管线但无语义声明，管线无声明语义
- vs 知识图谱方法（ORKG/Conceptual Tensor）：结构化声明但无执行层
- vs AGENTS.md：面向代码仓库的智能体文档，但不处理研究本身的认知结构

b) 创新点：
- **首次提出完整协议**：将科学逻辑、可执行代码、探索历史和证据四层联合结构化，带跨层溯源绑定
- **过程知识的首次结构化捕获**：探索图将实验失败从"被丢弃的噪声"提升为"一等研究产物"
- **实时研究管理器**：利用 AI 原生研究范式中研究轨迹已存在于人机对话的事实，以零额外文档负担捕获完整的决策和死胡同记录
- **编译器 + 审查系统的全栈设计**：从遗留转化到自动化评审的闭环生态
- **可执行可验证性**：ARA Seal 将可复现性从"代码应要求提供"变为"机器验证的属性"

c) 适用场景：
- AI 智能体驱动的科学研究流程
- 计算科学领域（ML、计算社会科学、计算经济学、干实验生物学）
- 不适合：湿实验科学（物理世界干预），除非底层实验记录实现数字化

d) 方法对比总结（维度覆盖）：

| 维度 | PDF | GitHub | 实验追踪器 | RO-Crate | ARA |
|------|-----|--------|-----------|----------|-----|
| 结构化科学逻辑 | ~ | x | x | ~ | ✓ |
| 可执行代码 | x | ✓ | x | x | ✓ |
| 探索轨迹 | x | x | ~ | x | ✓ |
| 锚定证据 | ~ | x | x | ~ | ✓ |

| 机器可验证复现性 | x | x | x | x | ✓ |

## 关键结果
a) 实验设计：
- **理解层 (Understanding)**：450 个问题（30 个目标 × 15 个问题），三类——Category A（表面结果/方法/条件，300 题）、Category B（配置细节：超参数/环境/预处理，115 题，仅 PaperBench）、Category C（失败知识：死胡同/替代方案/教训，35 题，仅 RE-Bench）。Claude Sonnet 4.6 问答子智能体，Claude Opus 4.6 三元评分（1.0/0.5/0.0）
- **复现层 (Reproduction)**：15 篇 PaperBench 论文，10 个复现子任务/篇（150 子任务，1,743 评分细则需求），分层难度（50 easy/49 medium/51 hard）。Claude Sonnet 4.6 编码智能体，难度加权成功率（1:2:3 权重），令牌预算 14-20M/篇
- **扩展层 (Extension)**：RE-Bench 5 个开放改进任务，8h SLURM 墙钟 + $50 API 消费上限。Claude Agent SDK，Claude Sonnet 4.6（+ 2 任务 Sonnet 4.5 对比），工具面 {Bash, Read, Edit, Write, Glob, Grep}

b) 关键数据：

**理解层 (Understanding)**：
- ARA 总体准确率 93.7% vs 基线 72.4%（+21.3 百分点）
- Category A（答案可在 PDF 中找到）：ARA 96.7% vs 基线 89.8%（PaperBench），92.1% vs 51.4%（RE-Bench），总体 +14.8 点，token 消耗少 12%
- Category B（配置详情，PDF 系统性缺失）：ARA 92.6% vs 基线 67.8%（+24.8 点）
- Category C（失败知识，仅 MALT 轨迹有）：ARA 81.4% vs 基线 15.7%（+65.7 点），基线放弃大多数查询
- Token 使用量随问题深度缩放（ARA 上显式 61K → 隐式 153K），基线保持平坦

**复现层 (Reproduction)**：
- 难度加权成功率：ARA 64.4% vs 基线 57.4%（+7.0 点）
- 按难度分解：Easy 85.1% vs 80.2%（+4.9%），Medium 68.5% vs 62.9%（+5.6%），Hard 54.5% vs 46.0%（+8.5%）
- 15 篇论文胜负平分布：8 胜 / 5 平 / 2 负
- 最大 ARA 优势在有多阶段训练管线的论文（fre: ARA 在 PyTorch 中重实现 JAX 代码，1.8GB GPU vs JAX 30.8GB）
- 结果伪造：基线 2 次，ARA 1 次（结构化减少但未消除）

**扩展层 (Extension)**：
- 5 个任务全部：ARA 智能体在所有任务上更快到达第一个有用操作
- 3/5 任务 ARA 最终得分领先：rust_codecontests（ARA 在 t=9min 提交思路 vs 基线 t=395min）、nanogpt_chat_rl、fix_embedding
- 2/5 任务 ARA 被反超：triton_cumsum（基线在 t=47.7min 发现 int8 压缩，轨迹内无此方案）、restricted_mlm（基线深度聚焦单一架构，ARA 智能体忠实执行所有轨迹推荐但被分散）
- **关键发现**：在较弱的 Sonnet 4.5 上模式反转——ARA 在两个任务上均显著领先，说明探索图的价值等于"轨迹记录的"与"智能体自己能发现的"之间的差距

**评审系统 (Review)**：
- Level 2 (Rigor Auditor) 突变基准：5 类型 × 23 ARA，总体检测率 82.6%。Critical 级别 100% 召回（伪造声明、被反驳分支泄露、过度声明），Missing falsification 91%，Orphan experiment 仅 22%（系统性盲点）
- LLM-as-judge 两个病理：分数膨胀（17/23 四舍五入刚好过线）和发现-评分脱耦

c) 最优势场景：
- 配置密集型复现（超参数/环境/预处理细节）：ARA src/configs/ 集中化 vs PDF 分散
- 失败知识密集型扩展：探索图提供结构化菜单而非从零推理
- PDF 质量差的论文：基线在 RE-Bench 类任务上 Cat A 准确率仅 51.4%（无结构化摘要可导航）

## 局限性
- **领域评估范围**：仅限 ML/CS，物理层和探索图的前提是可迭代计算实验，湿实验科学需大幅适配
- **忠实度天花板**：编译器仅能恢复 PDF 中包含的信息，缺失内容无法通过提取恢复
- **部署前提条件未实现**：对抗鲁棒性、隐私保证、沙箱化执行、schema 演化策略仅为设计意图
- **实时研究管理器假设 AI 原生工作流**：编码智能体需在整个项目中存在
- **扩展实验中轨迹可能约束强智能体的创造力**：当智能体能力超过轨迹记录时，选择性隐藏或上下文化轨迹部分可能更好
- **人类标注偏差**：基准由熟悉 ARA 格式和选定论文的标注者构建

## 核心贡献
a) **开源情况**：
- ARA 协议和实时研究管理器开源：github.com/Orchestra-Research/Agent-Native-Research-Artifact
- 在线平台：www.orchestra-research.com/ara（发布、评审和构建 ARAs）
- 编译器技能规范和 Rigor Auditor 技能规范附录中给出
- 数据/代码包在接收后释放
- 可复现性依赖托管模型推理，本地计算较轻松

b) **关键实现细节**：
- 验证/编译循环：generate → ARA Seal Level 1 check → fix，通常 2-3 轮收敛
- 编译器 prompt：全 ARA schema、字段级要求、四阶段自顶向下生成协议
- 实时研究管理器：无 SDK 依赖，纯自然语言规范，后台静默运行
- ARA 后缀标准：PAPER.md 含 ara_schema 版本标签，验证器接受未知字段（前向兼容）
- 嵌入存储：一种表示的存储嵌入约 25 MB
- 主要复现成本：LLM 推理调用（~6.3k 归一化调用 + 其他），非训练

c) **可迁移性**：
- ARA 协议的设计理念（分层结构化 + 跨层溯源）可适配其他有迭代实验范式的学科
- 认知层和证据层可能领域无关，物理层和探索图需要学科特定适配
- 编译器流水线可作为从非结构化文档到结构化知识包的通用反向工程框架
- ARA Seal 验证范式可作为 AI 驱动科学评审的通用模板
- 实时研究管理器可推广为任何 AI 辅助研究流程的自动知识捕获机制

## 总结
a) **一句话核心思想**：将研究论文重构为智能体可执行的四层知识包。

b) **速记版 Pipeline**：
1. 定义四层文件系统结构：认知层（声明/实验/相关工作）、物理层（代码核心/配置/环境）、探索图（决策树/死胡同/转向）、证据层（原始输出/日志）
2. 在研究过程中通过后台管理器自动捕获每个决策、失败和确认的声明到对应层
3. 对遗留论文运行编译器：自顶向下从 PDF + 代码仓库 + 轨迹中提取声明、重构代码规格、恢复探索历史和证据
4. 经 ARA Seal 三级验证（结构完整性 → 论证严谨性 → 执行可复现性）后注册为可引用的研究产物
5. 下游智能体通过 PAPER.md 索引导航、按需加载层，直接查询声明、执行代码、利用失败历史

c) **与本综述的具体关联**：
- 这是智能体驱动科学发现的**基础设施级工作**，定义了研究产物从"人类叙事文档"到"智能体可执行知识包"的范式转变
- 可放在综述的"Infrastructure for Agent-Mediated Science"或"Toward Fully Autonomous Research"章节
- 支持核心论点：AI 智能体成为科学研究的一等参与者，需要与之匹配的研究沟通基础设施——结构化、可执行、可验证、可迭代
- ARA 的"探索图"概念与 SkillOS 的"从经验中学习"和 SSL 的"结构化表示"形成互补：ARA 将探索过程结构化保留，SkillOS 从轨迹中提炼技能，SSL 将技能内部解耦为类型化图形

## 与综述的关联
ARA 在综述框架中处于**研究基础设施与范式变革**的位置——它将"研究产出"定义为可操作的结构化产物。与 SSL（技能内部表示）和 SkillOS（技能动态管理）不同，ARA 的目标是让科学交流系统（从写作、发布、评审到复现和扩展）围绕智能体可操作的研究产物而非人类可读的 PDF 重建。在三篇论文中，ARA 的框架性最强：它为"Scientific Discovery by Agents"提供底层的信息交换协议，SSL 和 SkillOS 则分别补充"技能如何表示"和"技能如何演化"的具体机制。
