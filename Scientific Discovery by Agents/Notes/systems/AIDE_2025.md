# AIDE 2025

## 基本信息
- **论文**: AIDE: AI-Driven Exploration in the Space of Code
- **作者**: Zhengyao Jiang, Dominik Schmidt, Dhruv Srikanth, Dixing Xu, Ian Kaplan, Deniss Jacenko, Yuxiang Wu
- **年份**: 2025
- **来源**: arXiv:2502.13138
- **关键词**: code-search, ML-engineering, tree-search, agent

## 核心思想
AIDE 将机器学习工程视为代码空间中的搜索问题。

## 方法细节
系统通过 tree search 在候选解决方案空间中生成、调试、复用和改进代码。

## 关键结果
论文报告 AIDE 在 Kaggle、MLE-Bench 和 RE-Bench 等评估中取得强表现。

## 局限性
主要是 ML engineering 和 benchmark optimization。

## 核心贡献
把 trial-and-error ML engineering 工程化为代码搜索。

## 与综述的关联
适合放入 `X0-Y3-Z3/Z4/Z5/Z7,Z8`，支撑代码 artifact search。

## 原文核对与分类复核
- **原文核对**：原文将 ML engineering 视为 code optimization problem，并将 trial-and-error 建模为 tree search in the space of potential solutions。
- **机制判断**：它不是多智能体系统，但非常强地支撑 code-space trajectory / tree search。
- **分类复核**：保持 `ASD 相关系统`；`X0` 正确；`Y3` 正确；`Z3,Z4,Z5,Z7,Z8` 正确。
- **写作用途**：适合放在 code-centered search / trajectory search 小节。

## 深读补充（按 MARS 标准）

### 研究问题
AIDE 关注 ML 工程中大量 trial-and-error 和代码迭代耗费人力的问题。它希望把 ML solution development 转化为可搜索的代码空间问题。

### 系统架构 / 工作流
系统是 LLM-powered ML engineering agent，将候选解决方案组织为代码空间中的 tree search，并复用、精修有希望的方案。

### 关键机制
核心是 tree search in code space。系统用执行反馈评估候选代码路径。

### 实验验证与证据
原文在 Kaggle evaluations、OpenAI MLE-Bench 和 METR RE-Bench 等 benchmark 上取得强结果。

### 局限性补充
AIDE 不是多智能体系统，也不直接处理自然科学假设；它主要支撑可执行代码 artifact 搜索。

### XYZ 分类逐项解释
- `X0`：单 agent / 非显式多智能体。
- `Y3`：tree search 是核心。
- `Z3,Z4,Z5,Z7,Z8`：覆盖代码设计、执行、分析、评估和迭代。
