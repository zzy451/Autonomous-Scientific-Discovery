# AWM 2024

## 基本信息
- **论文**: Agent Workflow Memory (AWM)
- **作者**: Zora Zhiruo Wang, Jiayuan Mao, Daniel Fried, Graham Neubig
- **年份**: 2024
- **来源**: arXiv:2409.07429
- **关键词**: skill-ecosystem

## 核心思想
LM Agent 在长周期复杂 Web 任务上仍然困难。AWM 让 Agent 从过往经验中
归纳可复用通用例行程序（workflows），在后续任务中选择性提供以指导行动。
支持离线（从训练示例预先提取）和在线（从测试查询实时归纳）双模式。在
Mind2Web 和 WebArena（合计 1000+ 任务、200+ domains，覆盖旅游/购物/社交媒体）
上，相对成功率分别提升 24.6% 和 51.1%，步数显著减少。在线 AWM 在跨任务
/网站/域评估中稳健泛化；Mind2Web 上 offline/online 的 step SR 相对 MindAct
分别提升 7.4-8.9、3.6-3.8、14.0-16.9 个百分点，跨域场景收益最大。

## 研究问题与动机
现有 Agent 三缺陷：(a) 每任务独立解决，不从过往成功/失败学习；(b) 固定示
例 in-context 无法覆盖任务和环境多样性；(c) 未把握关键——提取跨任务共享
的可复用操作模式。洞察来自认知科学（Chi et al., 1981）：人类从经验中抽象
通用例程指导未来行为，新场景中叠加已有子例程构建更复杂流程。假设：Agent
能从轨迹归纳高频子例程（如"搜索商品→加购物车→结账"），以去除具体值的
抽象形式存储，即可提成功率同时降步数，形成雪球效应。

## 方法细节
- Workflow 表示：(a) NL 描述 d，(b) 轨迹三元组 (NL 环境状态, 推理, 可执行
动作)。关键：归纳时将具体值（"dry cat food"）替换为占位符（"{product-name}"）。
- LM 归纳模块：将成功经验输入 LM 提取公共子例程。vs 规则方法（序列去重）
：LM 归纳粒更细、抽象更高。
- 离线模式：一次性归纳训练标注为 W_offline，测试时注入 Agent 记忆。
- 在线模式（无监督）：流式处理 → evaluator 判断成功/失败 → 成功则归纳
workflow → 追加记忆 → 增强记忆指导下一任务。WebArena case study 显示，在线 AWM 在前几十个 streaming examples 中快速形成基础 workflow，并继续组合成更复杂 workflow。
- 两种利用：(a) 记忆增强——workflow 作上下文；(b) AWM AS——workflow
包装为高级函数（如 find_place()）扩展动作空间。

## 关键结果
GPT-4 (gpt-4-0613, T=0)，按网站分组：
- WebArena（812 任务，5 网站）：AWM 在线 35.5%，比 BrowserGym(23.5%)
+12.0pct（51.1% 相对），也高于使用 14 个人类专家 workflow 的 SteP(33.0%)。五网站全升
11.8-30.7pct。平均 5.9 步（比 BrowserGym 少约 2.0 步；比 AutoEval 46.7 步少 40.8 步）。跨模板子集仍 33.2%，
证明跨任务泛化非来自同模板记忆。
- Mind2Web：离线 step SR 45.1%（vs MindAct 36.2%，+8.9pct；元素准确率
+9.0pct）。在线：跨任务 +7.4，跨网站 +3.8，跨域 +16.9pct；相对离线 AWM，
在线在未见网站/域更有优势。比 Synapse 检索完整示例元素准确率高 5.0pct、
step SR 高 4.0pct——抽象避免具体值偏差。
- 消融：LM vs 规则成功率相近但 LM 步数更少（5.9 vs 6.3）；代码 vs 文本
workflow 相仿；NL 描述 > 过滤 HTML（+0.8 step SR）。
## 局限性
- Workflow 偶引导错误方向，且系统较难及时识别偏离。
- 动态 UI 缺少中间状态感知，网页状态变化会削弱抽象 workflow 的可靠性。
- 离线模式受 train-test gap 影响；在线模式能缓解但依赖持续成功样本积累。

## 核心贡献
核心：Agent 从成功经验归纳抽象化可复用操作例程，雪球效应增强记忆，大幅
提升长周期 Web 任务成功率和效率。
Pipeline（在线）：流式处理 → LM 评估成功/失败 → 成功归纳 workflow（值→
占位符）→ 追加记忆 → 增强记忆指导下一任务 → 循环累积。workflow 从简单
（find_place）叠加为复杂（get_zipcode_of_place）。

## 与综述的关联
为技能生态提供"经验→可复用技能"完整技术路径：workflow 表示（描述+抽象
轨迹）+ 双模式 + 选择性检索 + 在线持续学习。科学发现映射：从成功实验轨迹
归纳操作模板（制备→表征→分析→报告），抽象化参数（{温度}/{浓度}），
新问题上检索适配模板指导设计。跨域泛化对跨学科发现尤为关键。
