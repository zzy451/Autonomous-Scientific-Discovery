# SkillsBench 2026

## 基本信息
- **论文**: SkillsBench: Benchmarking How Well Agent Skills Work Across Diverse Tasks
- **作者**: Xiangyi Li, Yimin Liu, Wenbo Chen et al.（30+ 位作者，多机构）
- **年份**: 2026
- **来源**: arXiv:2602.12670v3（2026-03-13）；Preprint（2026-03-16）
- **关键词**: 技能评测基准, 86任务, 84实测任务, 7,308轨迹, 精选技能, 自生成技能, 程序性知识

## 核心思想
SkillsBench 是一个大规模智能体技能评测基准，旨在回答一个核心问题：技能到底有没有用？尽管 Agent Skills 在工业界被快速采用，此前缺少系统性的效用量化。SkillsBench 收集 86 个任务，其中 84 个进入完整评测；在 11 个领域、7,308 条执行轨迹上，评测了 7 种智能体-模型配置在三种条件下（无技能、精选技能、自生成技能）的表现。论文将 Skill 定义为一种在推理时增强 agent 的结构化程序性知识包，而不是普通 prompt、RAG 文档或工具说明。

## 基准设计
- **规模**：86 个任务（84 个实测任务）覆盖 11 个领域，评测 7 种 agent-model 配置，并在无技能、精选技能、自生成技能三种条件下运行，最终形成 7,308 条有效轨迹。每个实测任务配有确定性验证器（deterministic verifier），确保评分客观可复现。
- **三种条件对比**：
  - 无技能基线：智能体仅依赖自身知识与推理。
  - 精选技能（Curated Skills）：由人类专家为该任务编写或挑选的技能。
  - 自生成技能（Self-Generated Skills）：由智能体自身为任务生成的技能。
- **技能质量维度**：除通过率外，还分析每任务提供技能数量、技能复杂度、领域差异等因素对效果的调节作用。
- **构造流程**：105 位学界和业界贡献者提交 322 个候选任务，最终 86 个任务通过审查进入 SkillsBench，其中 2 个任务因 GPU 需求或验证器超时未纳入主评测。任务要求人类编写 instruction、提供 Docker 环境、oracle solution 和程序化 verifier，并经过自动校验与人工审查，避免把单题答案伪装成 Skill。

## 关键结果
- **精选技能有效但领域差异巨大**：精选技能平均提升通过率 **+16.2pp**。但软件工程仅 **+4.5pp**，医疗健康高达 **+51.9pp**；同时 **16/84** 个任务上技能产生负面效果，说明技能并不总是帮助。
- **模型无法自主生成有效技能**：自生成技能平均为 **-1.3pp**，整体无收益且在某些任务上产生负效应。结论是模型无法可靠地编写它们受益于消费的程序性知识（models cannot reliably author the procedural knowledge they benefit from consuming）。这对自主技能生成路线构成重要警示，也与 ExpeL 的经验自动提取形成张力。
- **少而精优于多而全**：每任务提供 **2-3 个 Skills** 的条件收益最高（+18.6pp），4+ Skills 的收益降至 +5.9pp；按复杂度看，Detailed（+18.8pp）和 Compact（+17.1pp）技能优于 Comprehensive（-2.9pp）。更长、更全面的文档可能增加上下文负担或引入冲突指导。
- **小模型 + 技能可弥补部分模型规模差距**：配备精选技能的小模型在程序性任务上可匹敌或超过未配备技能的更大模型，例如 Claude Haiku 4.5 with Skills（27.7%）超过 Claude Opus 4.5 without Skills（22.0%），表明技能可在一定程度上弥补模型能力差距。
- **失败模式收益主要来自验证质量提升**：附录把 **5,171** 个 agent failures 归类为 Timeout、Execution、Coherence、Verification、Unknown。Skills 将总体失败率从 **78.4%** 降到 **61.1%**，其中 Quality Below Threshold 类失败从 **1,184** 次降至 **819** 次，是收益的主要来源。
- **仍有不可解任务**：**16/84** 个实测任务在所有模型和条件下通过率均为 0%，集中在计算量过大、复杂多步骤管线和严格规格任务三类。

## 核心贡献
SkillsBench 2026 的核心贡献是把 Skill 从工程直觉变成可做因果对比的评测对象：同一任务在 no-skill / curated-skill / self-generated-skill 条件中重复运行，并用确定性 verifier 度量 Skill 对 agent 成功率、失败模式和成本的影响。

## 与综述的关联
- 科学发现智能体的“技能效用度量”——SkillsBench 首次系统量化了技能对智能体表现的因果影响，为科学发现系统中的技能投资决策（哪些科研技能值得精心编写？何时应放弃技能？）提供了实证依据。
- “自生成技能无效”的发现直接警示：科学发现智能体的能力扩展当前仍依赖人类专家精心设计的技能，而非模型自主生成。AI 自动生成科研方法论的路线需要更审慎的验证与人类监督机制。
- 领域差异巨大的发现提示：科研技能的效果可能因学科而异——计算领域的技能效果可能弱于实验科学领域，这为指导科研技能的设计优先级提供了方向。

## 局限性
- 86 个任务相对于真实世界的技能使用多样性仍有限——科研领域的技能（如特定实验协议、统计方法）尚未充分纳入。
- 自生成技能的评估主要基于单次生成——多轮迭代的“技能精炼”能否缩小与精选技能的差距，尚待研究。
- 部分结论依赖当前 agent harness 与模型组合；随着 Claude Code、Gemini CLI、Codex 等工具链更新，绝对通过率可能快速变化。
- 基准使用高质量精选技能，论文也指出真实生态中技能质量平均更低，因此结果更接近“优质技能的上界效果”而非普通技能市场的平均效果。
- Skills 注入会增加上下文长度，收益可能部分来自“更多上下文”而非纯粹程序性结构；论文将其列为未来需要更强控制实验的问题。
