# Agent Skills Spec 2026

## 基本信息
- **论文**: Agent Skills Specification
- **作者**: Agent Skills 社区 (agentskills.io)
- **年份**: 2026
- **来源**: https://agentskills.io/specification
- **关键词**: Skills 标准, 互操作规范, 技能目录结构, SKILL.md, 开放标准, 跨平台

## 核心思想
Agent Skills Specification 是由 agentskills.io 维护的智能体技能格式规范。该标准定义了技能目录结构、SKILL.md 元数据格式与渐进式加载协议，目标是让不同智能体客户端能够解析同一种技能包格式。规范本身只定义文件系统与 frontmatter 约定，不等同于保证 Claude Code、Cursor、Gemini CLI、GitHub Copilot 等平台已经完全互操作。

## 方法细节
- **标准化目录结构**：`skill-name/ → SKILL.md（必需）+ scripts/ + references/ + assets/（可选）`。每个技能是自包含的文件夹，不依赖外部路径或特定平台特性。
- **SKILL.md 元数据字段**：
  - 必填——`name`（1-64 字符，小写字母/数字/连字符，不能以连字符开头或结尾、不能包含连续连字符，且必须与父目录名匹配）、`description`（1-1024 字符，描述功能与触发场景，是平台自动发现技能的核心依据）。
  - 可选——`license`（许可证标识）、`compatibility`（1-500 字符的环境依赖声明）、`metadata`（自定义键值对，支持扩展）、`allowed-tools`（空格分隔的预授权工具列表，实验性字段）。
- **渐进式披露协议**：三级加载——元数据（系统启动时加载至上下文，用于技能匹配与路由）→ 完整指令（技能激活时加载，指导智能体行为）→ 按需资源（脚本、参考文件仅在执行时加载）。此协议确保即使技能库规模达数百个，上下文窗口仍保持高效利用。
- **平台无关设计**：规范不假设特定平台特性，仅定义文件系统层面的接口约定。验证工具 `skills-ref` 提供格式校验参考实现。遵循规范的平台可一致读取与校验技能包；脚本执行、工具授权和运行时行为仍由具体客户端决定。

## 设计哲学
- **可发现性（Discoverable）**：通过标准化 `description` 字段，平台可自动将用户意图路由至匹配的技能，无需手动指定。
- **可组合性（Composable）**：技能作为独立模块可通过工作流编排组合为复杂流程，如文献检索技能 + 数据分析技能 + 论文撰写技能 = 科学发现智能体流水线。
- **可移植性（Portable）**：技能以文件夹、Markdown 指令和相对路径资源组织，便于在不同平台间迁移；但脚本语言、系统依赖、网络访问和工具权限仍可能需要按客户端适配。
- **渐进式复杂性**：从简单的一文件技能到包含脚本与参考文档的复杂技能，规范支持渐进式增强。

## 关键结果
该规范的关键价值不是实验性能数字，而是给 agent skill 提供了一套可验证的文件系统约定：标准目录、必填元数据、渐进式披露和参考校验工具。这使技能能够从单个提示词升级为可分发、可组合、可迁移的软件制品。

## 与综述的关联
- 科学发现智能体的"技能标准化"维度——统一规范使科研技能可在不同智能体系统间共享，加速科学发现工具生态的构建。
- 与 Anthropic Skills 实现（平台方）、SkillsBench 评测（效用量化）、GStack 与 Osmani Agent Skills（工程实践）共同构成完整技能生态——标准 + 实现 + 评测 + 应用。
- 标准化是科学发现智能体从单点工具走向生态协作的关键基础设施，类似于 HTTP 对 Web 的标准作用。

## 局限性
- 规范仍在早期阶段（2026），平台采纳度与生态成熟度有待观察——不同客户端对可选字段、工具权限与执行语义的支持程度可能不同。
- `allowed-tools` 字段为实验性，安全模型尚不完善——跨平台、跨厂商的权限管理与沙箱机制是未解决的开放问题。
- 规范的完整技术细节需直接访问 agentskills.io/specification 获取（该站点使用 Mintlify JS 渲染，自动化工具抓取受限）。
- 规范对技能的执行时行为（错误处理、超时、重试、状态持久化）缺乏标准约定，导致同一技能在不同平台上的执行语义可能不一致——这对需要确定性行为的科学计算技能尤其关键。
- `metadata` 字段作为开放键值对虽具扩展性，但也带来了互操作风险：不同平台可能对相同键赋予不同语义，导致技能在平台间迁移时产生未定义行为。
- 渐进式披露协议的具体阈值（元数据令牌数、完整指令加载时机）在各平台的实现中可能不同，标准仅定义了"三级"结构而未约束每个级别的硬性大小限制。

## 核心贡献
Agent Skills Spec 的核心贡献是将 agent skill 从平台私有实践抽象为开放规范，为科学发现中的文献检索、实验设计、数据分析和验证技能提供可共享的包装格式。
