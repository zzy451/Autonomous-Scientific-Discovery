## 分析：技能产生-存储-检索-演化链的现状与空白

**相对成熟的环节。** 存储与检索已有较多工程共识——从 Voyager 的向量 DB 到 Agent Skills Spec 的标准化目录+渐进式披露协议，技能的持久化格式和按需加载机制正在形成事实约定，数百技能规模下的发现与路由已有可用实践。

**部分解决的环节。** 技能的产生是当前最突出的断裂带之一——SkillsBench 显示模型自主生成技能平均收益接近零甚至可能负效果（精选 +16.2pp vs 自生成 ~0pp），与 ExpeL"LLM 可从经验中提取可迁移洞察"的乐观结论形成张力。这意味着在具体工具层面，"AI 自动生成技能"尚不可靠；在抽象策略层面或许可行，但缺乏统一质量度量。

**早期探索但远未成熟的环节。** 技能演化已出现 SkillOS 这类把增/改/删建模为长期管理策略的早期系统，A-MEM 也在记忆层展示了新记忆触发旧记忆演化的思路。但面向科研技能库的版本管理、冲突解决、跨版本兼容、技能淘汰退化、证据锚定与安全回滚仍缺乏成熟方案。当技能库从几十条增长到数千条时，旧技能如何被新经验更新、冗余技能如何被合并、矛盾技能如何被仲裁——这些问题才刚开始被系统化触及。
