# IdeaBench 2024

## 基本信息
- **论文**: IdeaBench: Benchmarking Large Language Models for Research Idea Generation
- **作者**: Sikun Guo*, Amir Hassan Shariatmadari*, Guangzhi Xiong, Albert Huang, Eric Xie, Stefan Bekiranov, Aidong Zhang (University of Virginia)
- **年份**: 2024
- **来源**: arXiv:2411.02429
- **关键词**: benchmark, idea-generation, biomedical

## 核心思想
IdeaBench 是一个评估 LLM 研究创意生成能力的基准系统，包含数据集和评估框架两部分。数据集涵盖 2,374 篇 2024 年生物医学领域高影响力论文的标题/摘要及其 29,408 篇参考文献。系统通过将 LLM 刻画为特定领域研究者，基于参考文献摘要上下文，利用其参数化知识动态生成新创意。评估采用两阶段流程：首先由 GPT-4o 根据用户指定的质量指标（如新颖性、可行性）对所有创意进行排序，然后基于目标论文创意在排序列表中的相对位置计算"Insight Score"（I ∈ [0,1]）。实验覆盖 OpenAI GPT、Google Gemini、Meta Llama 三个系列的 7 个模型。核心发现：LLM 能生成与人类论文高度相似的创意，且在新颖性上常有超越（GPT-4o 高资源模式下 I=0.766），但可行性普遍不及人类研究者，呈现明显的新颖性—可行性权衡。

## 评测目标
系统化、可扩展地评估 LLM 从给定文献背景中生成研究创意的能力，支持用户自定义质量维度（新颖性、可行性、清晰度等），填补科学创意生成领域缺乏统一评估框架的空白。与前人工作（SciMON、MOOSE、ResearchAgent）不同，IdeaBench 聚焦于评估而非方法改进。

## 基准设计
- **数据集构造**：通过 Semantic Scholar API 收集 2024 年生物医学领域论文。目标论文来源：(a) 顶级生物医学会议录用且被引 ≥1 次；(b) 其他会议但被引 ≥20 次。去重后得到 2,374 篇目标论文及其 29,408 篇参考文献。
- **参考文献过滤**：三条件筛选——被引 ≥5 次、排除非原始研究（综述/社论/信件/书籍）、仅保留目标论文背景章节引用的文献。过滤后剩 23,460 篇（每篇目标论文中位数 8 篇参考文献）。
- **创意生成**：Prompt 将 LLM 刻画为生物医学研究者，提供参考文献摘要作为背景信息，要求生成简洁的新假设（每查询生成 n=3 个创意）。LLM 无法看到目标论文摘要。
- **两阶段评估**：(1) 个性化质量排序——GPT-4o 将目标论文创意（经 LLM 摘要化处理）与 n 个生成创意混合后按用户指定质量指标排序，不告知哪个是原始创意；(2) Insight Score 量化——I(LLM,q) = (1/m) × Σ(rtarget_i|q − 1)/n，范围 [0,1]。0 = 所有目标论文排名第一，1 = 所有生成创意优于目标论文。
- **高低资源设置**：低资源模式使用 5 篇过滤后参考文献，高资源模式使用全部未过滤参考文献。
- **基线比较指标**：BERTScore（语义相似度）和 LLM 相似度评分（创意重叠度，1-10 分），BERTScore 实际上限 0.718。

## 关键数字
- 数据集：2,374 篇目标论文 + 29,408 篇参考文献（过滤后 23,460 篇）
- 评估模型：GPT-4o（评判器）评估 7 个 LLM（GPT-4o/4o Mini/3.5 Turbo、Gemini 1.5 Pro/Flash、Llama 3.1 405B/70B）
- 最佳新颖性 Insight Score：GPT-4o 高资源 0.766（>0.5 表示多数生成创意排名优于目标论文）
- 可行性 Insight Score：所有模型 <0.5，最高为 GPT-3.5 Turbo 高资源 0.305 和 Gemini 1.5 Pro 低/高资源 0.305
- 语义相似度最高：GPT-4o Mini 高资源 0.620（BERTScore F1，80th percentile）
- 创意重叠度最高：多组模型达到 8/10（80th percentile LLM similarity rating）
- Llama 3.1 8B 因生成大量不连贯文本被排除在主结果之外
- 消融实验使用 GPT-4o Mini 在 ≥15 篇参考文献的目标论文子集上进行

## 设计哲学
- **模拟人类研究流程**：模仿"确定主题→综述文献→识别空白→提出创意"的四步研究范式，将参考文献作为 LLM 的背景语境而非让模型凭空生成。
- **角色剖析（Profiling）**：在 Prompt 中将 LLM 明确塑造为"生物医学研究者"，以最大化利用其参数化记忆中的领域知识，而非仅依赖通用语言能力。
- **相对排序优于绝对评分**：承认创意质量的绝对评分极为困难，采用排名制——将 LLM 生成创意与人类（已发表论文）创意放在同一列表中对决，通过相对位置间接反映生成能力。
- **用户自定义质量维度**：允许用户自由定义"新颖性""可行性"等指标的具体含义（如 Bob 认为新颖 = 开发新方法，Alice 认为新颖 = 现有技术的新应用），提供个性化评估而非一刀切标准。
- **GPT-4o 作为评判器**：利用 LLM 的上下文学习能力解释用户自定义的质量定义，无需训练专门评估模型，实现了"零样本可扩展评估"。
- **双资源场景对比**：通过高/低资源设置模拟不同计算约束条件，揭示参考过滤对小模型的帮助和对大模型的限制。

## 局限性
- 仅覆盖生物医学领域，跨领域泛化性未知；不同学科的研究创意形式和评价标准可能差异显著。
- 依赖 GPT-4o 作为评判器可能引入 LLM 偏见（如对特定风格、长度的偏好）；评判器本身可能与被评测模型共享相似的偏见。
- Insight Score 基于相对排序而非绝对质量，无法回答"生成的创意本身有多好"——只能回答"与已发表论文创意相比如何"。
- 小模型（如 Llama 3.1 8B）生成的不连贯文本会被 LLM 评估器误判为高分（评估器未见过劣质文本），揭示了 LLM-as-Judge 方法的盲区。
- 生成创意数量 n 影响 Insight Score 的粒度和可比性，不同 n 下无法公平比较。
- 不涉及创意的实验验证或执行环节，仅停留在"纸面评估"层面。
- 目标论文收集要求有引用，可能排除了真正新颖但尚未被引用的前沿工作，但也可能存在时间偏差——2024 年论文的引用数仍在积累中。

## 核心贡献
IdeaBench 2024 的核心贡献是为 biomedical research idea generation 建立数据集与 Insight Score 评估框架，用相对排序方式比较 LLM 生成创意与真实论文创意在新颖性、可行性等用户定义维度上的表现。

## 与综述的关联
IdeaBench 直接服务于本综述的核心议题——科学发现智能体的"创意/假设生成"能力评估。它为综述中讨论 LLM 在科学发现前端（假设生成、研究方向规划）的表现提供了目前最系统的评测工具和量化指标。其"新颖性—可行性权衡"的发现对理解 AI 科学家的能力边界至关重要——LLM 擅长提出大胆新想法但缺乏落地判断力，这意味着人机协同或创意→筛选→执行的流水线可能是更有效的范式。Insight Score 的定义方式和 GPT-4o-as-Judge 的评估范式也为其他科学创意评测工作提供了方法论参考。
