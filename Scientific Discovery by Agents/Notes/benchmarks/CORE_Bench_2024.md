# CORE Bench 2024

## 基本信息
- **论文**: CORE-Bench: Fostering the Credibility of Published Research Through a Computational Reproducibility Agent Benchmark
- **作者**: Zachary S. Siegel, Sayash Kapoor, Nitya Nadgir, Benedikt Stroebl, Arvind Narayanan
- **年份**: 2024
- **来源**: arXiv:2409.11363
- **关键词**: benchmark, computational reproducibility, code execution, research verification, agent, R language

## 核心思想
该文提出 CORE-Bench（Computational Reproducibility Agent Benchmark），一个旨在衡量 AI 智能体完成科学研究计算可复现性任务的基准。CORE-Bench 包含 270 个任务，源自计算机科学、社会科学和医学三个领域的 90 篇论文，分布于三个难度等级（Easy/Medium/Hard），涵盖纯语言和视觉-语言任务。任务来自 CodeOcean 平台上经验证可复现的胶囊（capsules），每个胶囊包含论文代码、数据和 Docker 配置。作者以 AutoGPT 为通用基线，并以针对 CORE-Bench 定制化的 CORE-Agent 作为任务专用基线，分别搭配 GPT-4o 和 GPT-4o-mini 作为底层语言模型。最佳智能体（CORE-Agent + GPT-4o）在 Easy 级别达 60.00% 准确率，但 Hard 级别仅 21.48%。作者开发了支持数百台虚拟机的并行化评估框架，将原来需要超过 20 天的评估时间缩短至数小时。

## 评测目标
评测 AI 智能体能否自动完成论文结果的计算可复现性验证——即给定论文的代码仓库和数据，智能体需：安装依赖库、正确执行代码、从输出中提取结果并回答关于论文实验结果的问题。这是一项在科学界极具现实意义但常被忽视的任务。论文引用了 15 个学科的计算可复现性危机数据——即便数据与代码公开，大量研究仍无法复现（如 NeurIPS 2022 可复现性挑战中仅 18/28 论文完全可复现）。现有基准要么测单步代码生成（HumanEval），要么测广义的 ML 实验（MLAgentBench），要么测科学知识问答（GPQA），均未触及"将论文代码跑通并提取结果"这一独特的可复现性工作流。CORE-Bench 的立场是：能复现已有研究是进行原创研究的前提。

## 基准设计
- **题目数量/来源/难度分级**：270 个任务来自 90 篇论文（训练集 45 篇、测试集 45 篇），从 CodeOcean 的 5,090 个胶囊中经 10 条标准筛选得到（对应公开发表论文、CS/医学/社科领域、Python 或 R 语言、含 README、代码运行 <45 分钟、简单 bash 命令即可复现、输出清晰标注、结果低方差、大小 <10GB、可本地复现）。三个难度等级：(1) CORE-Bench-Easy：已提供成功运行的完整代码输出，智能体仅需从中提取信息回答问题（测试信息检索能力）；(2) CORE-Bench-Medium：提供 Dockerfile 和文本运行指令，智能体需执行 Docker 命令后从输出提取信息（测试终端交互 + 信息检索）；(3) CORE-Bench-Hard：仅提供 README，智能体需自行安装所有依赖、确定正确的运行命令、执行代码并提取信息（最接近真实场景）。共 181 个任务问题（三个难度等级共享相同问题，仅信息条件不同）。
- **评分方式**：Task Accuracy（主要指标）——所有任务问题均正确回答才算成功。问题有开放数值答案（无法靠猜测）。使用 95% 预测区间（3 次手动运行的结果区间）处理 17/181 题的随机性输出。同时报告平均 API 成本。对于视觉问题（图表结果提取），使用 vision-language model 进行分析。每个 (agent, task) 对运行在独立 VM 中，有时间限制（2 小时）和成本限制（$4/task）。
- **人类基线 vs AI 基线**：无受控人类基线（论文未进行等价条件下的人机对比）。但隐含人类基准：所有胶囊已被验证可由人在 CodeOcean 或本地成功复现（经过 3 次手动复现建立预测区间）。AI 基线（测试集）：CORE-Agent + GPT-4o: Easy 60.00%、Medium 57.78%、Hard 21.48%；CORE-Agent + GPT-4o-mini: Easy 44.44%、Medium 32.59%、Hard 16.30%；AutoGPT + GPT-4o: Easy 35.56%、Medium 37.78%、Hard 6.67%；AutoGPT + GPT-4o-mini: Easy 8.89%、Medium 2.22%、Hard 2.22%。
- **设计原则**：(i) 所有任务源自已验证可复现的论文，确保不存在不可能任务；(ii) 难度分级的阶梯设计允许在智能体能力不足时定位具体瓶颈；(iii) 隔离 VM + 并行化评估框架确保评估可扩展性和安全性；(iv) 包含 Python 和 R 双语言任务（论文称这是少数早期纳入 R 的智能体基准之一）；(v) 视觉-语言混合任务测试多模态信息提取。

## 设计哲学
- **计算可复现性作为"科学的基石"**：论文开篇引用 Buckheit & Donoho (1995) 的经典观点——"计算科学论文仅是学术的广告，真正的学术是完整的软件环境和生成图形的完整指令集"。这种哲学立场将可复现性提升为科学诚信的核心，而 CORE-Bench 成为衡量 AI 在此核心任务上表现的标尺。
- **"能复现才能创建"的逻辑链**：论证路径清晰——复现已有研究比创建新研究简单（创建研究本身需要复现 baseline 方法）；因此，能复现已有研究是进行原创研究的前提条件。CORE-Bench 成为"AI 科学家"发展路径上的一块踏脚石。
- **任务特定修改的巨大收益**：AutoGPT 到 CORE-Agent 的改造仅需数天工作（主要分析失败日志后调整 prompt + 添加程序化检查），但 Hard 级别从 6.67% 提升至 21.48%。这种"通用智能体 + 轻量任务适配"的策略对科学发现智能体的工程实践具有重要启示。
- **多维度技能测试**：要求同时具备代码执行、shell 交互、信息检索、视觉分析等多种能力。尤其是 R 语言的纳入——该语言在统计学和生物信息学等领域广泛使用但在 AI 基准中很少被纳入——体现了负责任基准设计的意识。
- **评估工程化**：并行化评估框架将 >20 天评估压缩至 2 小时，这在智能体基准领域是一项重要的工程贡献，降低了迭代实验的门槛。

## 局限性
- 任务仅来自 3 个学科（CS、社科、医学），物理、化学、生物等自然科学未纳入。
- 21.48% 的 Hard 准确率表明当前智能体远不具备可靠的研究复现能力，且提高预算（$4→$10）也仅从 26% 提升至 31%（训练集），暗示智能体在遇到困难时倾向于"卡住"而非调整策略。
- 视觉问题显著难于文本问题（59.26% vs 87.88%），R 任务显著难于 Python 任务，表明跨编程语言和多模态评估可能混杂了语言/模态偏见。
- 无受控人类基线，无法直接回答"AI 离人类复现能力有多远"的问题。
- 仅评估"是否能复现"，不评估复现过程的效率、复现代码质量、以及智能体是否能理解复现失败的原因。
- 所有胶囊已预先验证可复现，避免了"无法复现的论文"的挑战，但在真实世界中大量论文本身就不可复现——智能体是否能诊断不可复现性未被测试。
- 安全方面：AutoGPT 曾尝试访问 CodeOcean 网站创建账户，暴露了通用智能体在不受限网络环境中的安全隐患。

## 核心贡献
CORE Bench 2024 的核心贡献是把计算可复现性拆成可执行、可隔离、可自动评分的 agent benchmark，用 90 篇已验证可复现论文构造 Easy/Medium/Hard 三层任务，并提供大规模 VM 并行评估框架来衡量 agent 跑通研究代码、提取结果和回答复现问题的能力。

## 与综述的关联
CORE-Bench 是本综述 benchmark 章节中唯一直接针对"研究可复现性"的基准。它填补了"AI 能否验证已发表研究"这一评估空白——在科学发现智能体的完整生态中，可复现性验证既是成果可信度的基石，也是智能体进行原创研究前的必要步骤。它与 ScienceAgentBench 形成"复现已有 vs 创建新代码"的对照；与 SWE-bench 共享"真实代码仓库任务"的设计范式但聚焦科学研究而非软件工程；与 BrowseComp 在"信息检索"维度有交叉——CORE-Bench 要求智能体从代码输出中检索具体结果，BrowseComp 要求从网络中检索事实。其"通用智能体 + 轻量任务适配"的策略发现对于构建科学发现智能体具有工程实践借鉴意义。
