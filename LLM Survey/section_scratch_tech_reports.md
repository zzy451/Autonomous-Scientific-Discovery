# Section 7: 大厂 LLM 架构实践 -- Section Scratch

> 基于 28 份技术报告整理，覆盖 24 家公司/机构

## 7.0 引言

本节从工业实践角度，分析 24 家公司的 LLM 架构选择，揭示六维度设计空间中的收敛趋势和分化方向。我们不按公司分组叙述，而是按架构演进维度组织，展示大厂如何在以下六个核心问题上做出不同选择，并从每个选择中提炼出设计原则和未来方向。

**覆盖模型列表**：Llama 2/3/4, Mistral 7B, Mixtral 8x7B, DeepSeek-V2/V3/R1, Gemma 2/3, Qwen2.5/3, GLM-4, Yi-Lightning, Phi-4, Hunyuan-Large, MiniMax-01, OLMo 2, Kimi k1.5, Baichuan-M1, PanGu-Sigma, ERNIE 4.5, DBRX, Nemotron 3/340B, OpenELM, Step-2/Step-2 mini, MPT, Falcon 3

## 7.1 架构收敛：工业共识与事实标准

### 7.1.1 核心命题

到 2024-2025 年，大厂 LLM 在六维度中的多个子维度上已形成高度一致的工业共识。这些"事实标准"的形成不是偶然的，而是反映了深层的理论和工程考量。

### 7.1.2 五大事实标准

#### (a) RMSNorm 统一 Normalization

- **分支理由**：28 个模型中 26 个使用 RMSNorm（仅 PanGu-Sigma 推测为 LayerNorm，MPT 使用 LayerNorm），RMSNorm 已完全取代 LayerNorm。
- **技术介绍**：RMSNorm 是 LayerNorm 的简化版，去掉了均值中心化（mean-centering），仅保留方差归一化。计算公式：$\text{RMSNorm}(x) = \frac{x}{\sqrt{\text{mean}(x^2) + \epsilon}} \cdot \gamma$。
- **理论支撑**：(1) Zhang & Sennrich (2019) 证明 RMSNorm 可达到与 LayerNorm 相同的训练效果，但计算量降低约 30%；(2) 均值中心化对训练稳定性的贡献是可忽略的（re-centering invariance 不是必须的）。
- **待解决问题**：RMSNorm 在极端精度（如 FP4）训练下是否仍然足够？是否需要更精细的归一化方案？
- **未来影响**：RMSNorm 的统一性使其成为"默认选择"，未来的归一化创新更可能发生在 RMSNorm 之上（如 QK-Norm 的叠加），而非替换 RMSNorm 本身。
- **潜在技术**：Adaptive RMSNorm（根据训练阶段调整归一化强度）、与量化训练协同的归一化方案。

#### (b) Pre-Norm Residual 统一残差连接

- **分支理由**：27/28 个模型采用 Pre-Norm Residual（仅 OLMo 2 采用 Reordered-Norm/Post-Norm 变体）。
- **技术介绍**：Pre-Norm 将归一化置于子层之前（Norm -> SubLayer -> Add），与 Post-Norm（SubLayer -> Add -> Norm）相对。Pre-Norm 的梯度流更稳定，深层网络更容易训练。
- **理论支撑**：(1) Xiong et al. (2020) 证明 Pre-Norm 使残差流中的梯度量级在各层间更均匀；(2) Pre-Norm 允许更大的学习率，加速收敛。
- **待解决问题**：OLMo 2 的 Reordered-Norm 显示 Post-Norm 变体在特定条件下可提升训练稳定性，这一结果是否可复现和泛化？
- **未来影响**：残差连接的设计空间已高度收敛，创新更可能在 Normalization 位置和 Module Sequence 的组合上发生。
- **潜在技术**：Reordered-Norm 的大规模验证、Deep Residual Learning 的新变体。

#### (c) SwiGLU 统一 GLU 变体

- **分支理由**：26/28 个模型使用 SwiGLU（Gemma 2 使用 GeGLU，Nemotron-4 340B 使用 Squared ReLU），SwiGLU 已成为 FFN 激活函数的事实标准。
- **技术介绍**：SwiGLU = Swish 激活 + Gated Linear Unit，公式：$\text{SwiGLU}(x, W_1, W_2, W_3) = (\text{Swish}(xW_1) \odot xW_2) W_3$。引入门控机制使 FFN 具有更强的表达能力。
- **理论支撑**：(1) Shazeer (2020) 的消融实验表明 SwiGLU 在多数基准上优于 ReLU、GELU、GeGLU 等变体；(2) 门控机制可视为动态特征选择，增加了 FFN 的非线性表达能力。
- **待解决问题**：SwiGLU 的门控参数增加了约 50% 的 FFN 参数，是否存在更高效的门控方案？GeGLU（Gemma 2）与 SwiGLU 的差异是否在更大规模上显著？
- **未来影响**：SwiGLU 短期内不太可能被替代，但 MoE 中专家内部的 FFN 激活函数可能出现分化。
- **潜在技术**：GLU 变体的自动搜索（NAS）、与 MoE 路由机制协同设计的激活函数。

#### (d) RoPE 统一位置编码

- **分支理由**：27/28 个模型使用 RoPE 或其变体（仅 MPT 使用 ALiBi，PanGu-Sigma 推测为学习式 PE）。
- **技术介绍**：RoPE (Rotary Position Embedding) 将位置信息编码为旋转矩阵，作用于 Query 和 Key 向量。支持相对位置建模，且可通过基频调整实现长度外推。
- **理论支撑**：(1) Su et al. (2021) 证明 RoPE 具有良好的数学性质：旋转不变性、相对位置感知、距离衰减；(2) RoPE 与 FlashAttention 等加速框架兼容性好。
- **待解决问题**：RoPE 在超长上下文（>1M tokens）下的衰减特性是否仍然合理？iRoPE (Llama 4) 的 NoPE 层是否意味着部分层不需要位置编码？
- **未来影响**：RoPE 本身已成为标准，未来创新集中在 RoPE 的"使用方式"上——基频调整、分层差异化、与注意力模式的协同。
- **潜在技术**：可学习的 per-layer RoPE 基频、位置编码与 KV cache 压缩的联合优化。

#### (e) GQA 主导 Token Mixer

- **分支理由**：28 个模型中 22 个采用 GQA 或其变体（另有 MLA 3 个、MFA 1 个、ALiBi+MHA 1 个、标准 MHA 1 个）。GQA 的 8 KV 头配置最为常见。
- **技术介绍**：GQA 介于 MHA（每头独立 KV）和 MQA（所有头共享 KV）之间，将 Query 头分组共享 KV 头，在推理效率和表达能力之间取得平衡。
- **理论支撑**：(1) Ainslie et al. (2023) 证明 GQA 仅需少量训练就能从 MHA 检查点转换，且性能损失极小；(2) KV cache 减少带来的推理加速在实际部署中价值巨大。
- **待解决问题**：最优 GQA 比例（4:1? 8:1? 16:1?）是否存在理论最优解？极端比例（如 GLM-4 的 16:1）的性能边界在哪里？
- **未来影响**：GQA 仍将是主流基线，但 MLA 和 MFA 的出现表明 KV cache 优化存在更优方案。
- **潜在技术**：自适应 GQA 比例（按层或按训练阶段调整）、GQA 与 MLA 的混合方案。

### 7.1.3 收敛的深层原因

- **工程实用性**：RMSNorm 计算便宜、SwiGLU 性能稳定、GQA 推理高效——这些选择的共同特征是"在不牺牲太多精度的前提下最大化效率"。
- **生态效应**：Llama 2/3 的广泛采用使其架构选择成为"默认模板"，后来者（如 Falcon 3 从 ALiBi 转向 RoPE）倾向于向主流靠拢以利用现有生态工具。
- **消融实验成本**：在千亿参数规模上进行完整的架构消融实验成本极高，导致团队倾向于沿用已验证的选择。

## 7.2 Token Mixer 的工业分化

### 7.2.1 核心命题

尽管 GQA 是主流 Token Mixer，但 2024-2025 年出现了多条分化路线，每条路线都在尝试解决 KV cache 瓶颈和长上下文挑战。

### 7.2.2 四条 KV Cache 优化路线

#### (a) GQA：头维度压缩（主流路线）

- **分支理由**：22/28 个模型采用，是最成熟和广泛验证的方案。
- **技术介绍**：通过减少 KV 头数（典型从 128 减至 8），在头维度上压缩 KV cache。典型压缩比 4-16 倍。
- **理论支撑**：GQA 的信息论分析表明，当 KV 头数充分（>=4 个）时，组间信息冗余足以支撑共享而不显著损失表达能力。
- **代表模型**：Llama 2/3/4 (8 KV头), Qwen2.5/3, Mistral/Mixtral, Phi-4, Gemma 2/3, GLM-4 (2 KV头), Falcon 3
- **待解决问题**：KV 头数的最优选择缺乏理论指导——GLM-4 使用 2 个 KV 头（16:1 压缩）仍有竞争力，说明 GQA 的压缩潜力可能被低估。
- **未来影响**：GQA 将持续作为基线，但可能被 MLA/MFA 在高端模型中逐步替代。
- **潜在技术**：自适应 GQA（按层差异化 KV 头数）、与注意力模式（SWA/全局）联合优化的 GQA 配置。

#### (b) MLA：秩维度压缩（DeepSeek 路线）

- **分支理由**：DeepSeek 系列（V2/V3/R1）独创，实现 93.3% KV cache 压缩。
- **技术介绍**：将 KV 联合压缩到低维潜在向量（如 512 维），推理时仅缓存该向量。需要 Decoupled RoPE 解决位置编码与压缩的冲突。
- **理论支撑**：KV 矩阵的低秩近似理论——实证发现 KV 矩阵的有效秩远低于其物理维度，大部分信息可被低秩向量捕获。
- **代表模型**：DeepSeek-V2 (236B), DeepSeek-V3 (671B), DeepSeek-R1 (671B)
- **待解决问题**：MLA 的压缩维度（512）是否在所有规模上都是最优的？Decoupled RoPE 增加了架构复杂性，是否存在更简洁的解耦方案？
- **未来影响**：MLA 展示了 KV cache 压缩的理论上限远高于 GQA，可能成为下一代大模型的标准选择。
- **潜在技术**：自适应压缩维度（按层或按任务调整）、MLA 与其他注意力模式（SWA、线性注意力）的混合。

#### (c) CLA：层维度共享（Hunyuan/Yi-Lightning 路线）

- **分支理由**：Hunyuan-Large 和 Yi-Lightning 采用，与 GQA 正交互补。
- **技术介绍**：相邻层共享 KV cache（如每 2 层共享一次），从层维度压缩存储。GQA+CLA 组合可实现约 95% 的 KV cache 节省。
- **理论支撑**：相邻层的 KV 表示高度相似（层间余弦相似度分析），共享不会显著损失信息。
- **代表模型**：Hunyuan-Large (每2层共享), Yi-Lightning (跨层KV共享)
- **待解决问题**：共享间隔（每 2 层 vs 每 3 层 vs 动态间隔）的最优选择？CLA 与 MLA 能否组合？
- **未来影响**：CLA 可能成为 GQA 的标准增强组件，尤其在超长上下文场景。
- **潜在技术**：自适应层间共享（根据层间 KV 相似度动态决定是否共享）、与 MoE 交替层设计的协同。

#### (d) MFA：矩阵分解压缩（StepFun 路线）

- **分支理由**：阶跃星辰 Step-2 mini 采用，实现约 94% KV cache 节省。
- **技术介绍**：在 QK 交互阶段引入低秩矩阵分解；MFA-Key-Reuse 变体通过重参数化将 Key cache 复用为 Value cache，消除单独存储 Value cache 的需要。
- **理论支撑**：注意力权重矩阵 QK^T 的低秩结构可被矩阵分解高效逼近，且 Key 和 Value 之间存在可利用的冗余性。
- **代表模型**：Step-2 mini
- **待解决问题**：MFA 在大规模模型（>100B）上的验证尚不充分；与其他 KV cache 优化路线的对比消融实验缺乏。
- **未来影响**：MFA 提供了 KV cache 优化的第四条路线，丰富了设计空间。
- **潜在技术**：MFA + CLA 的组合、MFA 在 MoE 架构中的应用。

### 7.2.3 异构层注意力模式

除了 KV cache 优化，Token Mixer 的另一重要分化是"层间异构"——不同层使用不同的注意力模式。

#### (a) 交替 Local/Global 注意力

- **分支理由**：Gemma 2 (1:1交替), Gemma 3 (5:1交替), Mistral 7B (全SWA), Baichuan-M1 (部分SWA)
- **技术介绍**：部分层使用滑动窗口注意力（SWA, 局部），部分层使用全局注意力。只有全局层需要存储完整 KV cache。
- **理论支撑**：Transformer 各层的功能分化——浅层倾向处理局部模式（语法、短语），深层倾向处理全局语义，因此可按功能分配注意力范围。
- **待解决问题**：最优 Local:Global 比例（Gemma 2 的 1:1 vs Gemma 3 的 5:1）如何确定？
- **未来影响**：异构注意力层将成为标准设计模式，比例和配置将更加精细化。
- **潜在技术**：自动学习最优层间注意力模式分配、结合 NAS 搜索异构层配置。

#### (b) 线性注意力 + Softmax 混合

- **分支理由**：MiniMax-01 的 Lightning Attention + Softmax (7:1) 混合
- **技术介绍**：7 层 Lightning Attention（O(n) 复杂度）后接 1 层 Softmax Attention（O(n^2)），少量全局层保证信息整合质量。
- **理论支撑**：线性注意力的信息瓶颈可通过周期性的 Softmax 层补偿——线性层累积局部信息，Softmax 层进行全局整合。
- **代表模型**：MiniMax-01 (456B/45.9B激活, 4M上下文)
- **待解决问题**：7:1 比例是否对所有任务类型都最优？纯线性注意力的信息损失量级如何定量分析？
- **未来影响**：线性注意力混合是实现百万级上下文的关键技术路径之一。
- **潜在技术**：动态比例调整（根据输入长度自动切换）、与 Mamba/SSM 的对比选择。

#### (c) SSM + Transformer 混合

- **分支理由**：Nvidia Nemotron 3 Nano 的 Mamba-2 + Self-Attention 混合
- **技术介绍**：以 Mamba-2（SSM）层为主，穿插少量 Self-Attention 层。Mamba-2 仅需恒定状态，无需 KV cache。
- **理论支撑**：SSM 的循环状态可视为对历史信息的固定容量压缩，Self-Attention 层提供可变容量的全局检索能力。两者互补。
- **代表模型**：Nemotron 3 Nano (31.6B/3.2B激活, 1M上下文)
- **待解决问题**：Mamba vs Linear Attention 哪种次二次方案更优？SSM 的恒定状态是否在超长上下文中导致信息丢失？
- **未来影响**：SSM-Transformer 混合可能与 Linear-Softmax 混合形成两大阵营。
- **潜在技术**：自适应 SSM 状态大小、三方混合（SSM + Linear Attention + Softmax）。

#### (d) iRoPE 交替与 Short Convolution

- **分支理由**：Llama 4 的 iRoPE（交替 RoPE/NoPE 层），Baichuan-M1 的 Short Convolution on K/V
- **技术介绍**：iRoPE 中 RoPE 层用分块注意力处理局部信息，NoPE 层无位置编码处理全局语义。Short Convolution 在 K/V 计算中引入卷积操作，增强上下文模式捕获。
- **待解决问题**：NoPE 层为何能有效建模全局关系（没有位置编码为何能工作）？
- **潜在技术**：更精细的位置编码-注意力模式联合设计。

## 7.3 MoE 的工业实践

### 7.3.1 核心命题

MoE 已从 2023 年的少数派（Mixtral, PanGu-Sigma）发展为 2024-2025 年大模型的主流选择。但 MoE 的具体设计在专家粒度、路由策略和共享专家等维度上呈现显著分化。

### 7.3.2 从 Dense 到 MoE 的转变

- **分支理由**：2023 年仅 2/28 模型使用 MoE；2024-2025 年超过 14/28 模型使用 MoE，包括几乎所有旗舰模型。
- **技术介绍**：MoE 将 FFN 层替换为多个并行的"专家"网络，通过路由器选择性激活少数专家，实现"总参数大、激活参数少"的参数效率。
- **理论支撑**：(1) Scaling Laws 显示 MoE 在固定计算预算下可达到更高性能；(2) 稀疏激活使推理成本与激活参数（而非总参数）成正比。
- **从 Dense 转向 MoE 的代表**：Llama 2/3 (Dense) -> Llama 4 (MoE), Qwen2.5 (Dense开源) -> Qwen3 (MoE旗舰), Falcon 2 -> Falcon 3 (仍Dense，但小规模)
- **待解决问题**：Dense 模型是否还有竞争力？Llama 3 405B (Dense) 和 Gemma 3 27B (Dense) 表明"标准架构 + 极致 Scaling"仍可匹配 MoE。
- **未来影响**：MoE 将成为旗舰模型的默认架构，Dense 模型将主要存在于小模型（<30B）领域。
- **潜在技术**：Dense-to-MoE upcycling（从 Dense 模型转换为 MoE）、动态专家数量（根据输入难度调整激活专家数）。

### 7.3.3 专家粒度的三大路线

#### (a) 粗粒度：8-16 个大专家

- **代表模型**：Mixtral 8x7B (8专家Top-2), Hunyuan-Large (16专家+1共享Top-1), DBRX (16专家Top-4), MiniMax-01 (32专家Top-2)
- **技术特点**：每个专家参数量大，专家间差异化明确，路由决策影响大。
- **优势**：路由简单，专家负载容易均衡，通信开销小。
- **劣势**：专家组合多样性有限（C(8,2)=28），知识分解粒度粗。

#### (b) 中等粒度：64-128 个专家

- **代表模型**：ERNIE 4.5 (64专家Top-8), Qwen3 (128专家Top-8), Llama 4 Maverick (128专家)
- **技术特点**：平衡专家数量和参数规模，专家组合多样性高。
- **优势**：C(128,8) >> C(8,2)，专家组合空间极大，知识分解更细。
- **劣势**：路由器学习难度增加，需要更精细的负载均衡。

#### (c) 超细粒度：160-256 个小专家

- **代表模型**：DeepSeek-V2 (160专家+2共享Top-6), DeepSeek-V3/R1 (256专家+1共享Top-8)
- **技术特点**：每个专家非常小（中间维度约 2048），知识高度分解。
- **优势**：最大化专家专精化，理论上可实现更精细的知识路由。
- **劣势**：路由器决策空间巨大，需要 Auxiliary-Loss-Free 等高级负载均衡策略。

### 7.3.4 共享专家之争

- **有共享专家**：DeepSeek-V2 (2个), DeepSeek-V3/R1 (1个), Hunyuan-Large (1个), Llama 4 (1个)
  - 设计理由：共享专家处理跨领域通用模式（如语法、常识），路由专家处理领域专精知识。
- **无共享专家**：Mixtral, DBRX, Qwen3, MiniMax-01, ERNIE 4.5
  - 设计理由：所有专家均通过路由选择，通过负载均衡机制自然形成通用/专精分化。Qwen3 从 Qwen2.5-MoE（有共享专家）转为无共享专家，说明这一选择并非确定性的。
- **待解决问题**：两种方案在什么条件下各有优势？是否与专家粒度相关（超细粒度更需要共享专家？）
- **潜在技术**：动态共享专家（根据输入类型自动决定是否激活共享路径）。

### 7.3.5 路由策略演进

- **标准 Top-K Softmax 路由**：Mixtral (Top-2), DBRX (Top-4), DeepSeek-V2 (Top-6), DeepSeek-V3 (Top-8)
- **Auxiliary-Loss-Free 路由**：DeepSeek-V3 的可学习偏置项替代辅助损失函数，实现无损负载均衡。
- **Global-Batch 负载均衡**：Qwen3 在全局 batch 层面鼓励负载均衡，减少对模型性能的负面影响。
- **随机路由**：PanGu-Sigma 的 RRE，基于哈希的随机路由确保计算均衡。
- **模态隔离路由**：ERNIE 4.5 将文本/视觉 token 路由到不同专家子集。
- **潜在技术**：自适应 Top-K（根据输入难度动态调整 K 值）、层级化路由（先粗分组再细分配）。

### 7.3.6 异构层 MoE 设计

- **交替 Dense/MoE 层**：Llama 4 不是所有层都用 MoE，而是 Dense 层和 MoE 层交替出现。
- **Dense 下层 + Sparse 上层**：PanGu-Sigma 的异构设计，底层 Dense 共享通用特征，上层 Sparse 专精化。
- **异构专家**：ERNIE 4.5 的模态隔离专家、阶跃星辰 Step-2 的异构化专家设计。
- **未来影响**：从"每层相同 MoE"向"层间差异化 MoE"演进。

## 7.4 位置编码的工业演进

### 7.4.1 核心命题

RoPE 已成为唯一主流的位置编码方案，但其"使用方式"正在快速分化：从统一基频到分层差异化，从标准应用到解耦/交替。

### 7.4.2 RoPE 基频的指数增长

| 模型 | 年份 | RoPE Base | 目标上下文 |
|------|------|-----------|------------|
| 原始 RoPE | 2021 | 10,000 | 2K-4K |
| Llama 2 | 2023 | 10,000 | 4K |
| Llama 3 | 2024 | 500,000 | 128K |
| Qwen2.5/Baichuan-M1/Gemma 3 Global | 2024-2025 | 1,000,000 | 128K-1M |
| MiniMax-01 | 2025 | 10,000,000 | 1M-4M |

- **分支理由**：RoPE 的高频分量（对应高索引维度）在长距离上衰减过快，提高基频可以"拉伸"有效编码范围。
- **技术介绍**：RoPE 基频 $\theta$ 决定了旋转角频率 $\omega_i = \theta^{-2i/d}$，基频越大，同一距离的旋转角越小，位置信号衰减越慢。
- **理论支撑**：NTK-aware scaling 理论表明，基频调整等效于在频率空间进行非均匀插值，可在不增加计算量的情况下扩展有效上下文长度。
- **待解决问题**：基频增长是否有理论上限？MiniMax-01 的 10M 基频已远超原始设计的合理范围，是否存在数值精度问题？
- **未来影响**：基频将继续增长，但可能需要新的频率编码方案来替代简单的基频缩放。
- **潜在技术**：可学习的 per-dimension 基频、频率空间自适应编码。

### 7.4.3 分层差异化位置编码

#### (a) Decoupled RoPE (DeepSeek)

- **技术介绍**：将 RoPE 与 MLA 的 KV 压缩解耦，Q/K 各有专用 RoPE 维度。位置信息不参与低秩压缩，避免信息冲突。
- **代表模型**：DeepSeek-V2/V3/R1

#### (b) 双基频 RoPE (Gemma 3)

- **技术介绍**：Global 注意力层使用高基频（1M），Local 注意力层使用低基频（10K）。Global 层需要远距离位置感知，Local 层只需近距离精度。
- **代表模型**：Gemma 3

#### (c) iRoPE 交替 (Llama 4)

- **技术介绍**：每 4 层交替 RoPE 层（有位置编码 + 分块注意力）和 NoPE 层（无位置编码 + 全局注意力）。RoPE 层捕获局部位置关系，NoPE 层关注全局语义。
- **代表模型**：Llama 4 (Scout 10M上下文)
- **待解决问题**：NoPE 层为何能有效工作——是否因为全局语义关系不依赖绝对/相对位置？

#### (d) 半维度 RoPE (MiniMax-01)

- **技术介绍**：RoPE 仅应用于注意力头维度的一半（64/128 维），另一半不带位置信息。
- **代表模型**：MiniMax-01
- **理论支撑**：减少 RoPE 应用维度可降低位置编码对表示空间的"侵占"，保留更多维度用于语义编码。

### 7.4.4 ALiBi 的兴衰

- **分支理由**：MPT（2023）和 Falcon 1/2 曾采用 ALiBi，但 Falcon 3（2024）转向 Llama 兼容架构（推测 RoPE），ALiBi 已基本退出主流。
- **技术介绍**：ALiBi 不使用位置嵌入参数，而是在注意力分数中直接添加与距离成比例的线性偏置（负值）。
- **ALiBi 退出的原因**：(1) RoPE 在同等上下文长度下精度更高；(2) RoPE 生态更成熟（FlashAttention 等优化工具）；(3) RoPE 的基频调整提供了更灵活的长度扩展。
- **未来影响**：ALiBi 的设计理念（无参数、距离衰减）可能以其他形式融入未来的位置编码方案。

## 7.5 上下文长度的工业竞赛

### 7.5.1 核心命题

上下文长度已成为大厂竞争的关键维度，从 2023 年的 4K-32K 发展到 2025 年的 1M-10M，每一步都需要架构层面的支撑。

### 7.5.2 上下文长度演进时间线

| 阶段 | 年份 | 典型长度 | 代表模型 | 关键技术 |
|------|------|----------|----------|----------|
| 第一阶段 | 2023 | 4K-32K | Llama 2 (4K), Mixtral (32K) | 标准 RoPE, SWA |
| 第二阶段 | 2024 | 128K | Llama 3 (128K), DeepSeek-V3 (128K), Qwen2.5 (128K) | RoPE 基频提升, ABF, YaRN |
| 第三阶段 | 2025 | 1M | MiniMax-01 (4M), Llama 4 Maverick (1M), Nemotron 3 Nano (1M) | 线性注意力混合, iRoPE, Mamba混合 |
| 第四阶段 | 2025 | 10M | Llama 4 Scout (10M) | iRoPE + 分块注意力 + 温度缩放 |

### 7.5.3 各公司技术路线

#### (a) RoPE 基频扩展路线

- **代表**：Llama 3 (500K), Qwen2.5 (ABF->1M), Baichuan-M1 (1M)
- **特点**：最简单的长度扩展方案，仅需调整 RoPE 超参数。
- **局限**：随着基频增长，位置编码的分辨率下降，近距离位置区分能力减弱。

#### (b) 外推方法路线

- **代表**：Qwen3 (YaRN), DeepSeek-V2 (YaRN)
- **特点**：在预训练基频基础上，通过频率空间插值实现外推。
- **局限**：外推比例有限，通常不超过 8 倍。

#### (c) 异构注意力路线

- **代表**：Gemma 3 (5:1 Local/Global, 128K), MiniMax-01 (7:1 Lightning/Softmax, 4M), Llama 4 (iRoPE, 10M)
- **特点**：通过层间差异化注意力模式，使大部分层仅需处理局部信息，少数全局层承担长距离建模。
- **优势**：KV cache 总量大幅减少，使百万级上下文在工程上可行。

#### (d) SSM 混合路线

- **代表**：Nemotron 3 Nano (Mamba-2 + Attention, 1M)
- **特点**：SSM 层仅需恒定状态（与序列长度无关），是理论上最优的长上下文方案。
- **局限**：SSM 的"检索"能力不如 Attention，需要混合使用。

### 7.5.4 开放问题

- **长上下文的实际利用率**：Needle-in-a-Haystack 等测试显示，模型在超长上下文中的信息检索能力随长度下降。10M 上下文是否真正有用？
- **训练成本**：超长上下文训练的显存和计算成本极高，是否有更高效的训练策略？
- **推理成本**：即使 KV cache 被压缩，超长序列的首次前向传播（prefill）仍然昂贵。
- **潜在技术**：稀疏注意力 + 检索增强（只关注与查询相关的上下文段）、分层压缩（早期层激进压缩，后期层精细保留）。

## 7.6 规模法则与架构效率

### 7.6.1 核心命题

MoE 架构使"总参数"与"激活参数"脱钩，重新定义了模型规模的度量方式和参数效率的评估标准。

### 7.6.2 Dense vs MoE 的参数效率对比

| 模型 | 总参数 | 激活参数 | 激活比 | 架构 |
|------|--------|----------|--------|------|
| Llama 3 405B | 405B | 405B | 100% | Dense |
| Gemma 3 27B | 27B | 27B | 100% | Dense |
| Phi-4 | 14B | 14B | 100% | Dense |
| Mixtral 8x7B | 46.7B | 13B | 27.8% | MoE |
| DeepSeek-V3 | 671B | 37B | 5.5% | MoE |
| Qwen3-235B-A22B | 235B | 22B | 9.4% | MoE |
| Llama 4 Maverick | 400B | 17B | 4.3% | MoE |
| Llama 4 Scout | 109B | 17B | 15.6% | MoE |
| MiniMax-01 | 456B | 45.9B | 10.1% | MoE |
| Hunyuan-Large | 389B | 52B | 13.4% | MoE |
| ERNIE 4.5 | 300B | 47B | 15.7% | MoE |
| Nemotron 3 Nano | 31.6B | 3.2B | 10.1% | MoE |

### 7.6.3 关键观察

- **分支理由**：MoE 模型的激活比（激活参数/总参数）从 Mixtral 的 27.8% 持续下降到 DeepSeek-V3 的 5.5% 和 Llama 4 Maverick 的 4.3%，表明专家利用效率在提升。
- **技术介绍**：更低的激活比意味着每个 token 只使用总知识的一小部分，但通过精细的路由策略，模型能为每个 token 选择最相关的专家组合。
- **理论支撑**：MoE Scaling Laws (Shenngsheng Wan et al.) 表明，在固定训练 FLOPs 下，MoE 的最优策略是增加专家数量（降低激活比）而非增大每个专家的规模。
- **待解决问题**：激活比的下限在哪里？当每个 token 仅使用总参数的 3-5% 时，是否存在知识碎片化问题？
- **未来影响**：未来的旗舰 MoE 模型可能达到万亿级总参数但仅激活 10-50B 参数。
- **潜在技术**：动态激活比（简单输入激活少量专家，复杂输入激活更多专家）、专家合并/剪枝（推理时减少专家数量）。

### 7.6.4 Dense 模型的生存空间

- **分支理由**：Gemma 3 27B (Dense) 在多项基准上与 Qwen3 30B-A3B (MoE) 竞争，Phi-4 14B (Dense) 通过极致数据质量超越其规模。
- **技术介绍**：Dense 模型的优势在于推理时无路由开销、部署简单、对硬件要求更统一。
- **未来影响**：Dense 模型将主要在小模型（<30B）和边缘部署场景中保持竞争力。
- **潜在技术**：结构化剪枝（将训练好的 MoE 压缩为 Dense）、知识蒸馏（从 MoE 教师到 Dense 学生）。

## 7.7 训练工程与架构的协同

### 7.7.1 核心命题

架构选择不能脱离训练工程讨论——FP8 训练、训练稳定性方案和数据规模都与架构设计深度耦合。

### 7.7.2 FP8 混合精度训练

- **分支理由**：DeepSeek-V3 首次在 671B 参数模型上验证 FP8 训练的可行性，将训练成本降至 $5.57M。
- **技术介绍**：FP8 使用 8 位浮点数（1 符号 + 4 指数 + 3 尾数或 1+5+2）替代 FP16/BF16，将显存占用和通信带宽降低约一半。
- **理论支撑**：大部分权重和激活值的动态范围可被 FP8 覆盖，关键在于精细的缩放因子（scaling factor）设计和梯度累积策略。
- **架构耦合**：FP8 训练对归一化（RMSNorm 的数值稳定性）、注意力（QK 值的动态范围）和路由器（softmax 精度）都有影响。
- **代表模型**：DeepSeek-V3 (FP8), Yi-Lightning (FP8 优化)
- **待解决问题**：FP8 训练是否适用于所有架构？某些架构（如 MLA 的低秩压缩）是否对精度更敏感？
- **未来影响**：FP8 将成为大模型训练的标准精度，FP4 训练可能在 2026-2027 年出现。
- **潜在技术**：混合精度策略（关键层使用 FP16，其余使用 FP8）、与量化训练（QAT）的统一框架。

### 7.7.3 训练稳定性方案

训练不稳定性（loss spikes, divergence）是大模型训练的主要工程挑战，多种架构级解决方案已出现：

#### (a) QK-Norm

- **代表**：Gemma 3, Qwen3, Llama 4 (额外 Q/K RMSNorm)
- **原理**：对 Query 和 Key 矩阵归一化，防止注意力 logit 值爆炸。
- **历史**：MPT 的 QK LayerNorm -> Gemma 2 的 Logit Soft-capping -> Gemma 3/Qwen3 的 QK-Norm。
- **趋势**：QK-Norm 正在成为新标准，取代 Gemma 2 的 logit soft-capping。

#### (b) QKV Clipping

- **代表**：OLMo 2 (受 DBRX 启发)
- **原理**：裁剪 Q/K/V 向量的极端值，防止注意力计算中的数值不稳定。

#### (c) Reordered Norm

- **代表**：OLMo 2
- **原理**：将 RMSNorm 从子层输入移至子层输出，改变梯度流动路径，减少 loss spikes。

#### (d) 双重归一化

- **代表**：Gemma 2 (Pre+Post RMSNorm)
- **原理**：在每个子层的输入和输出都应用 RMSNorm，双重约束激活值范围。

### 7.7.4 数据规模与架构选择的关系

| 模型 | 参数量 | 训练数据 | 数据/参数比 | 架构特点 |
|------|--------|----------|-------------|----------|
| Llama 3 405B | 405B | 15.6T | 38.5 | Dense + 标准架构 |
| Qwen3-235B | 235B | 36T | 153.2 | MoE + QK-Norm |
| DeepSeek-V3 | 671B | 14.8T | 22.1 | MoE + MLA + MTP |
| Llama 4 | 400B | 40T | 100.0 | MoE + iRoPE |
| Nemotron 3 Nano | 31.6B | 25T | 791.1 | Mamba+Transformer+MoE |
| Phi-4 | 14B | 9.8T | 700.0 | Dense + 合成数据 |

- **分支理由**：小模型的数据/参数比远高于大模型（Nemotron 3 Nano: 791 vs Llama 3 405B: 38.5），表明小模型需要更多"过度训练"。
- **架构-数据交互**：Phi-4 (14B) 通过极致数据质量（大量合成数据）弥补参数不足；DeepSeek-V3 通过 MTP 训练目标从相对较少的数据中提取更多信号。
- **未来影响**：架构创新（如 MTP）可能减少对数据量的依赖，数据质量创新（如合成数据）可能降低对参数规模的需求。

## 7.8 开放问题与未来趋势

### 7.8.1 架构是否会完全收敛？

- **收敛的证据**：RMSNorm、Pre-Norm、SwiGLU、RoPE、GQA 已成为事实标准；Falcon 3 从 ALiBi 转向 RoPE 兼容，GLM-4 从 GLM 填空范式转向标准因果 LM。
- **分化的证据**：Token Mixer 出现 GQA/MLA/CLA/MFA 四条路线；MoE 设计高度分化（粒度、路由、共享专家）；位置编码使用方式快速创新（iRoPE、双基频、解耦）。
- **判断**：底层组件（Norm, Activation, Residual）已收敛，但系统级设计（Token Mixer 策略、MoE 配置、Module Sequence）仍在快速分化。架构设计的创新焦点正从"单层组件"转向"层间组合与异构设计"。

### 7.8.2 下一代 LLM 架构的可能方向

#### (a) 全面异构层设计

- **趋势**：从"每层完全相同"向"每层（或每组层）使用不同配置"演进。
- **已有实践**：Gemma 3 (5:1 Local/Global), MiniMax-01 (7:1 Linear/Softmax), Llama 4 (交替 RoPE/NoPE + Dense/MoE), PanGu-Sigma (Dense下层/Sparse上层), OpenELM (逐层缩放宽度)
- **预测**：未来模型可能在注意力类型、FFN 宽度、专家配置、位置编码等多个维度上实现层间差异化，形成"异构层超级块"的设计模式。

#### (b) 次二次注意力的工业成熟

- **趋势**：Linear Attention (MiniMax-01) 和 SSM/Mamba (Nemotron 3 Nano) 已在超大规模模型上验证可行性。
- **预测**：次二次注意力组件将成为超长上下文模型的标配，但可能以混合模式（与 Softmax Attention 搭配）而非完全替代的形式存在。

#### (c) KV Cache 优化的统一框架

- **趋势**：GQA (头维度)、MLA (秩维度)、CLA (层维度)、MFA (矩阵分解) 四条路线并存。
- **预测**：未来可能出现统一框架，在一个模型中组合多种 KV cache 优化策略（如 MLA + CLA + 异构注意力）。

#### (d) 训练目标的架构化

- **趋势**：MTP (DeepSeek-V3, Nemotron 3 Nano) 将训练目标（多 token 预测）深度嵌入架构设计。
- **预测**：更多训练目标将以架构组件的形式被集成，如推理链路（reasoning pathway）作为架构的一部分而非后训练产物。

#### (e) 动态架构

- **趋势**：MoE 已实现"按输入动态选择专家"，未来可能扩展到动态选择注意力类型、动态调整层数、动态分配计算预算。
- **潜在技术**：Early Exit（简单输入提前退出）、动态 Top-K 路由、输入自适应层跳过。

### 7.8.3 总结性框架

```
架构设计空间的三层结构：

第一层（已收敛）：基础组件选择
  RMSNorm | SwiGLU | RoPE | Pre-Norm Residual
  -> 事实标准，创新空间有限

第二层（活跃分化）：系统级策略
  Token Mixer: GQA vs MLA vs CLA vs MFA vs 混合注意力
  Channel Mixer: Dense vs MoE (粒度/路由/共享专家)
  Position Encoding: 基频调整 vs 解耦 vs 交替 vs 差异化
  -> 当前创新焦点，各公司差异显著

第三层（新兴方向）：异构与动态设计
  Module Sequence: 从均匀层到异构超级块
  动态架构: 输入自适应的计算分配
  训练目标架构化: MTP 等嵌入架构的训练信号
  -> 下一代 LLM 架构的核心方向
```
