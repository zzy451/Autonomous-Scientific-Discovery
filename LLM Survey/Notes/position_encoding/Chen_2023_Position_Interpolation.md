# Extending Context Window of Large Language Models via Positional Interpolation
**作者**: Shouyuan Chen, Sherman Wong, Liangjian Chen, Yuandong Tian | **年份**: 2023 | **来源**: Meta AI / arXiv | **arXiv**: 2306.15595

## 0. 摘要翻译
本文提出位置插值（Position Interpolation, PI），一种简单高效的方法，将基于 RoPE 的大型语言模型（如 LLaMA）的上下文窗口从 2048 扩展至 32768 个 token。核心思路是将位置索引线性下缩至原始预训练范围内，而非外推至超出训练范围的位置。该方法仅需 1000 步微调即可完成适配，计算成本极低。

## 1. 方法动机
a) **为什么提出**: LLM 预训练的上下文窗口有限（如 LLaMA 为 2048），直接使用超出训练长度的位置索引（外推）会导致性能崩溃。
b) **现有痛点**: 
   - 直接外推 RoPE 至未见过的位置会产生极大的旋转角度，导致注意力分数异常
   - 重新预训练长上下文模型成本极高
   - 当时缺乏有效的 RoPE 上下文扩展方法
c) **研究假设**: 将新的位置索引范围 $[0, L')$ 线性压缩至原始范围 $[0, L)$，使所有位置索引都落在模型见过的范围内，可以避免外推带来的不稳定性。

## 2. 方法设计
a) **pipeline**: 
   1. 给定目标长度 $L'$ 和原始训练长度 $L$
   2. 计算缩放因子 $s = L / L'$
   3. 将所有位置索引 $m$ 替换为 $m \cdot s$
   4. 使用 $f'(x, m) = f(x, ms)$（缩放后的 RoPE）
   5. 进行少量步数的微调

b) **模块功能**: 通过线性缩放将超长序列的位置索引"压缩"到模型预训练时见过的范围内。

c) **公式解释**:
   - 原始 RoPE: $f(x, m) = R_{\Theta,m} x$，其中 $m \in [0, L)$
   - PI 修改: $f'(x, m) = R_{\Theta, m \cdot L/L'} x$，使得当 $m \in [0, L')$ 时，$m \cdot L/L' \in [0, L)$
   - 理论分析：插值的上界约为外推上界的 $\sim 600$ 倍小，解释了为何插值远比外推稳定
   - 本质上等价于将 RoPE 的所有频率 $\theta_i$ 均匀除以缩放因子 $s = L'/L$

## 3. 与其他方法对比
a) **本质不同**: 
   - 与直接外推：PI 确保所有位置索引在已见范围内，避免 OOD 问题
   - 与 NTK-aware/YaRN：PI 均匀缩放所有频率维度，NTK/YaRN 选择性缩放
   - 与 ALiBi：ALiBi 天然支持外推，PI 是 RoPE 的后处理方法
b) **创新点**: 
   - 提出"插值优于外推"的核心洞察
   - 理论分析给出了插值与外推的稳定性上界比较
   - 实现极其简单（一行代码修改）
c) **适用场景**: 适用于所有 RoPE-based 模型的上下文扩展；作为 YaRN、NTK-by-parts 等更精细方法的基线和理论基础。

## 4. 实验表现
a) **验证方式**: 在 LLaMA 7B/13B/33B/65B 上评估，将 2048 扩展至 8192/16384/32768。
b) **关键数据**: 
   - 仅需 1000 步微调即可完成适配
   - 在 passkey retrieval 任务上达到 100% 准确率
   - 32768 上下文的 LLaMA 65B 在长文本任务上表现优秀
   - 对原始短上下文任务的性能影响很小
c) **局限性**: 
   - 均匀缩放导致高频维度的位置分辨率降低（YaRN 的出发点）
   - 仍需微调才能发挥最佳效果
   - 极大的扩展倍数（>8x）下，性能下降较明显

## 5. 总结
a) **一句话概括**: 位置插值通过将超长序列的位置索引线性压缩至预训练范围内，以极低成本（1000 步微调）实现 RoPE-based LLM 的上下文窗口扩展，开创了 RoPE 扩展方法的研究方向。
b) **速记 pipeline**: Position Index $m$ → Scale by $L/L'$ → Apply Standard RoPE → Fine-tune 1000 steps → Extended Context
