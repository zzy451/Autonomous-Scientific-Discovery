# iRoPE: Interleaved Rotary Position Embedding (Llama 4)
**来源**: Meta AI (Llama Team) | **年份**: 2025 | **arxiv**: 2504.00927

## 0. 背景
Llama 4 是 Meta 于 2025 年 4 月发布的基础模型系列，包括 Scout（16 专家，10M 上下文）和 Maverick（128 专家，1M 上下文）。iRoPE（interleaved Rotary Position Embedding）是其核心位置编码创新，通过交替使用带 RoPE 的注意力层和不带位置编码（NoPE）的注意力层，突破了传统 RoPE 在超长上下文中的瓶颈。

**模型概览**:
| 模型 | 总参数 | 激活参数 | 专家数 | 最大上下文 |
|------|--------|---------|--------|-----------|
| Scout | 109B | 17B | 16 | 10M tokens |
| Maverick | 400B | 17B | 128 | 1M tokens |

## 1. 方法动机
- **标准 RoPE 的长度瓶颈**: 标准 RoPE 在所有层施加相对位置编码，导致长距离 token 间的注意力分数因高频旋转而快速衰减。当序列长度远超训练长度时，位置编码的外推能力有限
- **纯 NoPE 的局限性**: 完全不使用位置编码的模型虽然理论上不受位置限制，但缺乏位置感知能力，在需要精确位置信息的任务（如代码补全、指令跟随）上表现不佳
- **核心洞察**: 并非每一层都需要位置信息。部分层可以作为"语义层"自由地关注全局上下文（global context），而其他层负责提供位置感知（positional awareness）。这种分工使得模型可以兼得局部精确和全局自由
- **与现有长度外推方法的区别**: PI、NTK-aware、YaRN 等方法均在频率域修改 RoPE 参数，本质上是"修补"；iRoPE 从架构层面重新设计，是"重构"

## 2. 方法设计（详细）

### 2.1 RoPE+NoPE 3:1 交替设计
iRoPE 的核心架构创新是在 Transformer 层堆叠中采用 **3:1 交替模式**：

```
[RoPE层] → [RoPE层] → [RoPE层] → [NoPE层] → [RoPE层] → [RoPE层] → [RoPE层] → [NoPE层] → ...
```

- **3 个 RoPE 层**（局部注意力层）：
  - 使用标准 RoPE 旋转位置编码
  - 配合 **局部注意力掩码**（chunked attention，chunk size 约 8192 tokens）
  - 负责构建强局部上下文和精确的相对位置关系
  - 连续 3 层的局部处理逐步丰富 token 的局部表示

- **1 个 NoPE 层**（全局注意力层）：
  - 完全不施加任何位置编码
  - 使用 **全局注意力掩码**（full attention），可关注整个上下文窗口中的所有 token
  - 由于没有位置编码约束，不受训练长度限制，天然支持长度外推
  - 负责全局信息整合和远程依赖建模

### 2.2 为什么选择 3:1 比例
- **局部信息为主**: 大多数语言理解任务中，局部上下文（相邻几千 token）是最重要的信息来源。3 层 RoPE 确保模型有足够的局部建模能力
- **全局整合为辅**: 每 4 层插入 1 层全局注意力，定期将局部表示整合到全局视野中，避免信息孤岛
- **计算效率**: NoPE 全局注意力层的计算量随序列长度平方增长，比例过高会导致计算成本过大。3:1 比例在性能和效率之间取得平衡
- **与 local-global attention 设计的联系**: 类似于 Longformer、BigBird 等模型的 local + global attention 思路，但 iRoPE 在位置编码层面实现这种混合，更加彻底

### 2.3 推理时温度缩放（Inference-Time Temperature Scaling）
- NoPE 层在推理阶段引入注意力 logits 温度缩放因子
- 公式：`Attention(Q, K, V) = softmax(QK^T / (sqrt(d) * τ)) V`，其中 τ 是温度参数
- 温度参数根据推理时的实际序列长度动态调整，使注意力分布适应训练时未见过的超长序列
- 这一设计使得模型可以在不重新训练的情况下适应从 256K（训练长度）到 10M（推理长度）的跨越

### 2.4 渐进式上下文扩展训练
- 训练分多阶段进行，逐步增加上下文长度
- 阶段示例：8K → 32K → 128K → 256K
- 每阶段微调 iRoPE 的温度参数和注意力配置
- 最终通过温度缩放在推理时外推到 10M

### 2.5 与六维度框架的关联
- **位置编码维度**: iRoPE 是 RoPE 的混合演进变体，属于"部分层使用相对位置编码"的新范式
- **Token Mixer 维度**: NoPE 层的全局注意力 + RoPE 层的局部注意力构成隐式的混合注意力模式（local-global attention）
- **长度外推**: 通过 NoPE 层消除位置编码对长度的硬约束，配合温度缩放实现训练-推理长度解耦

## 3. 与其他方法对比

| 方法 | 位置编码类型 | 最大上下文 | 长度外推策略 | 架构修改 |
|------|------------|-----------|-------------|---------|
| RoPE (Llama 3) | 全层 RoPE | 128K | 频率缩放 | 无 |
| ALiBi | 线性偏置 | ~8K | 天然外推但性能有限 | 替换 PE |
| PI (Position Interpolation) | 修改 RoPE | ~32K | 线性插值频率 | 无 |
| NTK-aware | 修改 RoPE 基频 | ~64K | 高频保留低频压缩 | 无 |
| YaRN | 混合 RoPE 修改 | ~128K | NTK-aware + 注意力缩放 | 无 |
| LongRoPE | 搜索 RoPE 因子 | ~2M | 每维度缩放因子搜索 | 无 |
| **iRoPE (Llama 4)** | **交替 RoPE+NoPE (3:1)** | **10M** | **NoPE 全局注意力 + 温度缩放** | **架构层面** |

**关键差异**:
- PI/NTK/YaRN/LongRoPE 都是在频率域"修补"RoPE，不改变架构
- iRoPE 从架构层面解决问题：部分层不需要位置编码，让模型自由学习全局依赖
- iRoPE 的长度外推能力更强（10M vs 前者的 128K-2M），且方法更简洁

## 4. 实验/工程验证
- **长上下文能力**: Scout（10M 上下文）在 RULER 等长上下文基准上表现优异，能有效利用超长上下文中的信息
- **短上下文无损**: 在标准基准（MMLU、HumanEval、GSM8K 等）上，iRoPE 不损害短上下文性能
- **Maverick 多模态**: Maverick（128 专家，1M 上下文）在多模态和文本任务上达到同级别模型领先水平
- **训练-推理解耦**: 模型在 256K 上下文长度训练，通过 iRoPE 的温度缩放机制在推理时外推到 10M，无需重新训练
- **工程可行性**: chunked attention 的 RoPE 层显著降低了超长序列的内存和计算需求，使 10M 上下文在工程上可行

## 5. 对本综述的启示
- **位置编码不必全层统一**: iRoPE 打破了"每层都需要相同位置编码"的惯例，为位置编码设计开辟了新维度——层级异构位置编码（layer-heterogeneous PE）
- **位置编码与注意力模式的耦合**: RoPE 层搭配局部注意力、NoPE 层搭配全局注意力，展示了位置编码与 token mixer 设计的深度耦合
- **长度外推的架构解法**: 相比频率域的修补方法，架构层面的重设计可能是实现超长上下文的更根本路径
- **与 local-global attention 的联系**: iRoPE 可以看作是 local-global attention 思想在位置编码维度的实现，两者共同服务于超长上下文建模
- **设计空间**: 3:1 比例并非唯一选择，不同比例（2:1、4:1 等）可能适用于不同场景，这一设计空间值得进一步探索

## 6. 总结
iRoPE 通过 3:1 交替使用 RoPE 层（局部注意力+位置编码）和 NoPE 层（全局注意力+无位置编码），从架构层面解决了 RoPE 的长度外推瓶颈，配合推理时温度缩放实现了 10M token 的上下文窗口。这一设计的核心洞察是"并非每一层都需要位置信息"，开创了层级异构位置编码的新范式。

**速记 pipeline**: Input → [3x RoPE Layer(chunked local attn) + 1x NoPE Layer(global attn)] × N → inference-time temperature scaling → 10M context
