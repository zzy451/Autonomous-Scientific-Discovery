# M-RoPE: Multimodal Rotary Position Embedding (Qwen2-VL)
**作者**: Peng Wang, Shuai Bai, Sinan Tan, Shijie Wang, Zhihao Fan, Jinze Bai, Keqin Chen 等 (Alibaba Qwen Team)
**年份**: 2024 | **arxiv**: 2409.12191

## 0. 摘要翻译
Qwen2-VL 引入了多模态旋转位置编码（M-RoPE），将 RoPE 从一维文本序列扩展到多模态场景，为文本、图像和视频中的 token 提供统一的位置编码方案。M-RoPE 将 RoPE 的频率维度分解为三个独立的分量——时间、高度、宽度——分别编码不同模态的位置信息。

## 1. 方法动机
- 标准 RoPE 仅编码一维序列位置，无法表达图像的二维空间结构或视频的时空结构
- 简单地将图像 patch 展平为一维序列会丢失空间邻近关系
- 多模态模型需要在统一框架下处理文本（1D）、图像（2D）、视频（3D）的位置信息
- 需要一种方法既保持 RoPE 的相对位置编码优势，又能扩展到多维数据

## 2. 核心方法

### 2.1 三分量分解
将 RoPE 的 $d$ 维频率向量分解为三个等长的子向量：
- **时间分量** ($d/3$ 维)：编码时间/序列位置
- **高度分量** ($d/3$ 维)：编码垂直空间位置
- **宽度分量** ($d/3$ 维)：编码水平空间位置

### 2.2 不同模态的位置赋值
**文本 token**：
- 时间 = 高度 = 宽度 = 文本序列位置 $t$
- 三个分量使用相同的位置索引，退化为标准 1D RoPE

**图像 token**：
- 时间 = 图像在序列中的起始位置（固定）
- 高度 = patch 的行索引 $h$
- 宽度 = patch 的列索引 $w$
- 保留了图像的二维空间结构

**视频 token**：
- 时间 = 帧索引 $t$
- 高度 = patch 的行索引 $h$
- 宽度 = patch 的列索引 $w$
- 完整编码了视频的三维时空结构

### 2.3 数学形式
对于位置 $(t, h, w)$，M-RoPE 的旋转矩阵为：
$$R(t,h,w) = \text{diag}(R_{\text{time}}(t), R_{\text{height}}(h), R_{\text{width}}(w))$$
其中每个子矩阵是标准 RoPE 旋转矩阵，作用于对应的维度子空间。

## 3. 设计优势
1. **统一性**：文本、图像、视频使用同一套编码机制，无需模态特定的位置编码
2. **结构保持**：图像的空间邻近性和视频的时空连续性被自然编码
3. **分辨率无关**：支持任意分辨率的图像输入（Naive Dynamic Resolution）
4. **向后兼容**：文本模态退化为标准 RoPE，可复用预训练权重

## 4. 实验结果
- Qwen2-VL 在多个多模态基准上达到 SOTA
- 支持任意分辨率图像理解，无需固定分辨率预处理
- 视频理解能力显著提升，能处理 20 分钟以上的视频
- M-RoPE 使模型能更好地理解空间关系和布局

## 5. 后续发展
- Qwen2.5-VL 进一步改进为 2D RoPE + 动态分辨率
- 启发了其他多模态模型的位置编码设计
- 与 3D-RPE 等工作形成互补的多维 RoPE 扩展方向

## 6. 与本综述的关联
- 代表了 RoPE 从 NLP 向多模态扩展的重要里程碑
- 三分量分解是一种简洁而有效的多维位置编码策略
- 展示了位置编码在多模态统一架构中的关键作用
- 与 3D-RPE、LieRE 等工作共同构成多维位置编码的研究前沿
