# Contextual Position Encoding: Learning to Count What's Important
**作者**: Olga Golovneva, Tianlu Wang, Jason Weston, Sainbayar Sukhbaatar | **年份**: 2024 | **来源**: Meta FAIR / arXiv | **arXiv**: 2405.18719

## 0. 摘要翻译
本文提出上下文位置编码（Contextual Position Encoding, CoPE），一种根据输入上下文动态确定位置的编码方法。传统位置编码将位置定义为 token 在序列中的索引，但许多任务需要模型按更抽象的单元（如句子、名词、特定类型 token）来计数和定位。CoPE 使用可学习的门控机制决定何时递增位置计数器，从而实现上下文感知的位置编码。

## 1. 方法动机
a) **为什么提出**: 标准位置编码仅按 token 索引计数（0, 1, 2, ...），无法表达"第 3 个句子"或"最近的名词"等更高层次的位置概念。
b) **现有痛点**: 
   - Token 级别的位置计数在不同粒度的任务中表现不佳
   - 例如，"第 5 个单词"在不同上下文中可能有完全不同的语义重要性
   - 现有方法无法根据内容调整位置的递增方式
c) **研究假设**: 如果位置编码能够根据上下文内容学习"什么算一个位置单元"，模型在计数、选择性关注和长距离依赖任务上将表现更好。

## 2. 方法设计
a) **pipeline**: 
   1. 对每个 token $x_t$，计算门控值 $g_t = \sigma(w^T h_t)$（$\sigma$ 为 sigmoid）
   2. 位置 $p_i$ 是从 token $i$ 到当前位置的门控值累积和: $p_i = \sum_{j=i}^{t} g_j$
   3. 由于门控值为连续值 $\in (0,1)$，位置值为分数（fractional position）
   4. 使用插值在最近的两个整数位置的学习嵌入之间获取最终位置编码

b) **模块功能**: 门控网络学习在何时"计数"一个位置单元；位置嵌入表提供位置到向量的映射；插值机制处理分数位置。

c) **公式解释**:
   - 门控: $g_t = \sigma(w_g^T x_t + b_g)$
   - 上下文位置: $p_{ij} = \sum_{k=j+1}^{i} g_k$（从 key 位置 $j$ 到 query 位置 $i$ 的累积门控值）
   - 位置编码: 对 $\lfloor p_{ij} \rfloor$ 和 $\lceil p_{ij} \rceil$ 对应的学习嵌入进行线性插值
   - 最终注意力: 将位置编码作为偏置加入注意力分数

## 3. 与其他方法对比
a) **本质不同**: 
   - 与 RoPE/ALiBi：这些方法使用固定的 token 距离，CoPE 使用内容相关的动态距离
   - 与 T5 relative bias：T5 用固定分桶，CoPE 用可学习的动态门控
   - 唯一能"跳过"不重要 token 的位置编码方法
b) **创新点**: 
   - 位置不再是 token 索引的简单函数，而是上下文的函数
   - 可学习的门控使模型自动发现"什么应该被计数"
   - 通过连续门控值和插值实现可微分训练
c) **适用场景**: 特别适合需要计数特定类型元素的任务（如"找到第 k 个名词"）；适合需要选择性关注的长上下文场景。

## 4. 实验表现
a) **验证方式**: 在 Flip-Flop 任务（选择性复制）、选择性计数任务、语言建模任务上评估。
b) **关键数据**: 
   - 在 Flip-Flop 任务上显著优于 RoPE 和 ALiBi
   - 在选择性计数任务上接近完美（>99% 准确率），而标准 PE 约 60-70%
   - 语言建模困惑度与 RoPE 可比或略优
c) **局限性**: 
   - 引入额外的门控参数和计算开销
   - 门控机制的学习可能需要更多训练数据
   - 在标准语言建模任务中优势不如在特化任务中明显
   - 尚未在大规模 LLM 中被广泛验证

## 5. 总结
a) **一句话概括**: CoPE 通过可学习的门控机制实现上下文感知的位置计数，突破了传统位置编码仅按 token 索引编码的限制，特别适合需要抽象计数和选择性关注的任务。
b) **速记 pipeline**: Token → Gate $\sigma(w^T x)$ → Cumulative Gate Sum = Fractional Position → Interpolated Position Embedding → Attention Bias
