# Revisiting Residual Connections: Orthogonal Updates for Stable and Efficient Deep Networks
**作者**: Sejun Oh, Jongmin Kim, Donghyeon Baek, Namjun Park, Jaeho Lee | **年份**: 2025 | **arxiv**: 2505.11881

## 0. 摘要翻译
标准残差连接将子模块输出直接加到输入流上，但这种方式可能导致特征冗余——子模块输出中与输入平行的分量被重复累加，既浪费表达能力又影响训练稳定性。本文提出"正交残差更新"(Orthogonal Residual Update, ORU)：将子模块输出相对于输入流进行分解，仅保留与输入正交的分量加入残差流。这种策略促进更丰富的特征学习和更高效的梯度传播。在 ResNetV2、Vision Transformer 等多种架构和 CIFAR、TinyImageNet 等数据集上，ORU 均提升了泛化精度和训练稳定性。

## 1. 方法动机
- 标准残差连接 $\mathbf{x}_{l+1} = \mathbf{x}_l + F_l(\mathbf{x}_l)$ 中，$F_l(\mathbf{x}_l)$ 可能包含与 $\mathbf{x}_l$ 平行的分量
- 平行分量仅放大已有特征，不引入新信息，造成表示冗余
- 随着层数增加，冗余累积导致残差流中有效信息密度下降
- 这与 ShortGPT 等工作观察到的"中间层近似恒等映射"现象一致
- 需要一种机制确保每层只贡献"新"信息

## 2. 方法设计
- **正交分解**: 将子模块输出 $F_l(\mathbf{x}_l)$ 分解为平行分量和正交分量
  - 平行分量: $F_l^{\parallel} = \frac{\langle F_l(\mathbf{x}_l), \mathbf{x}_l \rangle}{\|\mathbf{x}_l\|^2} \mathbf{x}_l$
  - 正交分量: $F_l^{\perp} = F_l(\mathbf{x}_l) - F_l^{\parallel}$
- **正交残差更新**: $\mathbf{x}_{l+1} = \mathbf{x}_l + F_l^{\perp}(\mathbf{x}_l)$
  - 仅将与输入正交的"新"信息加入残差流
  - 丢弃平行分量，避免冗余累积
- **实现简洁**: 仅需一次内积和一次投影操作，计算开销极小
- **梯度传播**: 正交更新使梯度在反向传播中更稳定，减少梯度爆炸/消失

## 3. 对比
| 方法 | 残差更新方式 | 信息效率 | 额外开销 |
|------|------------|---------|---------|
| 标准残差 | $x + F(x)$ | 含冗余 | 无 |
| ReZero | $x + \alpha \cdot F(x)$ | 缩放控制 | 1个标量 |
| SkipInit | $x + \alpha \cdot F(x)$, $\alpha$初始化为0 | 缩放控制 | 1个标量 |
| ORU | $x + F^{\perp}(x)$ | 仅新信息 | 1次投影 |

## 4. 实验
- **架构**: ResNetV2, Vision Transformer (ViT)
- **数据集**: CIFAR-10, CIFAR-100, TinyImageNet
- **结果**:
  - ORU 在所有架构和数据集上均提升泛化精度
  - 训练过程更稳定，loss 曲线更平滑
  - 深层网络上的改进尤为显著
  - 与标准残差连接相比，ORU 的特征表示具有更高的有效秩 (effective rank)
- **消融实验**: 验证了正交分量是性能提升的关键，而非简单的范数控制

## 5. 总结
a) 一句话: ORU 通过仅保留子模块输出中与输入正交的分量来更新残差流，消除冗余累积，提升深层网络的特征多样性和训练稳定性。
b) 速记 pipeline: `F(x) → 分解为平行分量 F∥ + 正交分量 F⊥ → 丢弃 F∥ → x_{l+1} = x_l + F⊥`; 核心: 每层只贡献"新"信息。
