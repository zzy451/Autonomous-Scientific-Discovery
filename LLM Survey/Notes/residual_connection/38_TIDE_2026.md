# TIDE: Token-Informed Depth Execution for Per-Token Early Exit

**论文信息**: Token-Informed Depth Execution for Per-Token Early Exit in LLM Inference (2026)
**arXiv**: 2603.21365
**分类**: Dynamic Depth (动态深度 / 后训练早退)

## 核心思想
LLM 对每个 token 都执行全部层的计算，不论难度。TIDE 是一个后训练（post-training）系统，在周期性的检查点层（checkpoint layers）附加微型学习路由器（tiny learned routers），让每个 token 自主决定是否提前退出。

## 关键机制
- **后训练部署**: 不需要重新训练原始模型，仅在预训练模型上附加轻量路由器
- **周期性检查点层**: 不在每一层都设置退出点，而是在固定间隔的检查点层设置路由器，降低开销
- **微型路由器**: 每个检查点层的路由器参数极少，基于当前 hidden state 决定是否退出
- **Per-Token 决策**: 同一序列中不同 token 可在不同深度退出

## 实验结果
- 在不显著损失质量的前提下减少推理计算量
- 路由器训练成本极低，适合后训练部署
- 简单 token（如标点、常见词）倾向于在浅层退出，复杂 token 使用更多层

## 与综述的关联
属于 **Dynamic Depth** 维度的工作。与 ADEPT (Note 37) 互补：ADEPT 聚焦于 KV cache 解耦，TIDE 聚焦于后训练部署的轻量化。与 FlexiDepth (Note 40) 类似都是 post-training 方法，但 TIDE 用路由器而非 adapter。

## 核心贡献
提出后训练的轻量级 per-token 早退系统，通过周期性检查点层的微型路由器实现动态深度，无需重训原始模型。
