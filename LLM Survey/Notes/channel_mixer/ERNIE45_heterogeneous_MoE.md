# ERNIE 4.5: 异构 MoE 与模态隔离路由
**来源**: Baidu ERNIE Team | **年份**: 2025 | **arxiv**: 无正式 arXiv（Baidu 官方技术报告，发布于 2025 年 6 月）

## 0. 背景
ERNIE 4.5 是百度于 2025 年 6 月发布的多模态基础模型系列，包含 10 个变体（从 0.3B Dense 到 424B MoE），旗舰版 ERNIE-4.5-300B-A47B 拥有 300B 总参数/47B 激活参数。其核心创新是 **异构 MoE（Heterogeneous MoE）**——打破"所有专家同构"的传统假设，为不同模态（文本、视觉）设计专用专家和共享专家，并通过模态隔离路由、路由正交化损失和 3D RoPE 等技术实现高效的多模态联合建模。

**模型规模概览**:
| 模型 | 总参数 | 激活参数 | 专家数/层 | 路由策略 | 层数 |
|------|--------|---------|----------|---------|------|
| ERNIE-4.5-300B-A47B | 300B | 47B | 64 | Top-8 | 54 |
| ERNIE-4.5-21B-A3B | 21B | 3B | 64 | Top-6 | - |

## 1. 方法动机
- **多模态 MoE 的干扰问题**: 在传统同构 MoE 中，文本和视觉 token 共享所有专家，导致模态间路由竞争和表示干扰。视觉 token 的统计特性（空间局部性、像素级连续性）与文本 token（离散符号、长程语义依赖）截然不同，强制共享会损害两种模态的专业化建模
- **同构专家的效率浪费**: 视觉信息的处理通常不需要与文本一样大的专家参数量。同构专家配置导致视觉 token 使用过大的专家（浪费计算）或文本 token 使用过小的专家（限制表达能力）
- **预训练阶段的多模态整合**: 与先训练文本模型再适配视觉的两阶段方法不同，ERNIE 4.5 从预训练阶段就联合训练文本和视觉数据，需要一种能在预训练中就处理模态差异的架构设计
- **路由坍缩与专家冗余**: MoE 训练中常见的路由坍缩（大部分 token 被路由到少数专家）和专家冗余（多个专家学到相似表示）问题，需要更精细的训练损失来解决

## 2. 方法设计（详细）

### 2.1 异构 MoE（Heterogeneous MoE）
**核心思想**: MoE 层中的专家不再是同构的，而是针对不同模态设计了参数量和功能不同的专家子集。

- **三类专家**:
  1. **文本专用专家（Text-Dedicated Experts）**: 仅处理文本 token，参数量较大，捕捉语言的复杂语义结构
  2. **视觉专用专家（Vision-Dedicated Experts）**: 仅处理视觉 token，参数量为文本专家的 **1/3**，适配视觉信息的处理特点
  3. **共享专家（Shared Experts）**: 处理所有模态的 token，负责跨模态通用模式的学习

- **异构参数设计**: 视觉专家 = 1/3 文本专家参数。这一设计基于视觉 token 的处理不需要与文本同等复杂度的洞察，提高了计算效率

- **总配置**: 64 个专家/层，每个 token 激活 Top-8（300B 版本）或 Top-6（21B 版本）

### 2.2 模态隔离路由（Modality-Isolated Routing）
- **路由隔离**: 文本 token 和视觉 token 被路由到不同的专家子集
  - 文本 token → 仅在文本专家 + 共享专家中路由
  - 视觉 token → 仅在视觉专家 + 共享专家中路由
- **目的**: 避免模态间的路由竞争和表示干扰，让每种模态的 token 被路由到最适合的专家
- **与标准 MoE 路由的区别**: 标准 MoE（如 Mixtral、DeepSeek）所有 token 在全部专家中竞争路由；ERNIE 4.5 按模态分区路由，本质上是"分区 MoE"

### 2.3 路由正交化损失（Router Orthogonalization Loss）
- **目的**: 防止专家冗余和路由坍缩
- **机制**: 对路由器的权重矩阵施加正交性约束，鼓励不同专家被不同模式的 token 激活
- **效果**: 提升专家多样性，确保每个专家学到独特的表示模式，而非多个专家冗余地学到相似内容
- **与 DeepSeek-V3 的对比**: DeepSeek-V3 使用无辅助损失的偏置项（bias term）实现负载均衡；ERNIE 4.5 使用正交化损失显式约束路由多样性

### 2.4 多模态 Token 均衡损失（Multimodal Token-Balanced Loss）
- **问题**: 文本和视觉 token 的比例在训练中可能不均衡，导致某一模态主导路由
- **解决方案**: 引入跨模态的 token 均衡损失，确保不同模态的 token 在路由分配中得到公平对待
- **与标准负载均衡的区别**: 标准负载均衡只关注"每个专家是否被均等使用"；多模态 token 均衡还关注"每种模态的 token 是否得到公平路由"

### 2.5 3D RoPE（三维旋转位置编码）
- **动机**: 视觉输入（图像、视频）具有空间和时间结构，标准 1D RoPE 无法编码这些多维位置信息
- **设计**:
  - 将 RoPE 的频率维度分为三组，分别编码 **时间(T)**、**高度(H)**、**宽度(W)** 三个轴
  - **频率分配**:
    - 时间轴：分配低频段（变化最慢的轴用低频编码）
    - 空间轴（H 和 W）：剩余频率在两个空间轴间交替分配
  - 文本 token：退化为标准 1D RoPE
- **优势**: 统一的位置编码方案，无需为不同模态设计不同的位置编码机制
- **与其他多维 PE 的对比**: ViT 通常使用 2D 正弦/学习位置编码；3D RoPE 将这一思路推广到视频的时空维度

### 2.6 注意力层设计
- 自注意力层的参数在所有模态间共享（不像 FFN/MoE 层分模态）
- 使用 GQA（Grouped Query Attention）压缩 KV 头数
- 推测使用 RMSNorm 和 Pre-Norm 残差连接（未明确公开）

## 3. 与其他方法对比

| 方法 | MoE 类型 | 模态路由 | 专家异构性 | 路由均衡 | 位置编码 |
|------|---------|---------|-----------|---------|---------|
| Mixtral 8x7B | 同构 MoE (8专家) | 不区分模态 | 同构 | 辅助损失 | RoPE |
| DeepSeek-V3 | 同构 MoE (256+1shared) | 不区分模态 | 同构 | Aux-loss-free (bias) | RoPE |
| Llama 4 Maverick | 同构 MoE (128+1shared) | 不区分模态 | 同构 | 标准路由 | iRoPE |
| Qwen3-235B | 同构 MoE (128选8) | 不区分模态 | 同构 | Global-batch 均衡 | RoPE |
| **ERNIE 4.5** | **异构 MoE (64专家)** | **模态隔离路由** | **视觉=1/3文本参数** | **正交化+token均衡** | **3D RoPE** |

**ERNIE 4.5 的独特定位**: 是目前唯一在 MoE 设计中同时引入异构专家参数、模态隔离路由和 3D 位置编码的大模型。这三项创新共同服务于"多模态原生"的设计目标。

## 4. 实验/工程验证
- **多模态性能**: ERNIE 4.5 系列在多模态理解（文本+视觉）任务上表现出色
- **文本不退化**: 模态隔离路由确保文本专家不受视觉训练干扰，文本任务性能不退化
- **效率优势**: 视觉专家的参数量仅为文本的 1/3，在保持视觉性能的同时显著降低视觉推理成本
- **训练稳定性**: 路由正交化损失和多模态 token 均衡损失有效防止了路由坍缩和模态主导问题
- **大规模工程**: 基于 PaddlePaddle 框架训练，支持 BF16/FP8 推理和 4-bit/2-bit 量化
- **全尺寸家族**: 10 个变体覆盖从 0.3B Dense 到 424B MoE，验证了异构 MoE 在不同规模的可扩展性
- **注意**: 无正式 arXiv 论文，许多架构细节（如精确的专家数量分配、Normalization 类型等）未完全公开，限制了精确分析

## 5. 对本综述的启示
- **Channel Mixer 的新范式**: 异构 MoE 打破了"所有专家同构"的默认假设，为 MoE 设计引入了新的维度——按模态/任务设计不同参数量的专家
- **MoE 与多模态的深度耦合**: 模态隔离路由展示了 MoE 路由策略可以与输入模态紧密耦合，这在纯文本 MoE 模型中未被探索
- **位置编码的模态适配**: 3D RoPE 展示了位置编码如何适应多模态输入的多维结构，是位置编码维度的重要拓展
- **效率导向的异构设计**: 视觉专家 = 1/3 文本参数的设计展示了"不同内容需要不同计算量"的效率思想
- **训练损失的创新**: 路由正交化损失是标准辅助损失之外的新探索，直接约束专家多样性
- **局限性**: 无正式论文限制了详细分析；与标准 MoE 的精确消融比较缺失

## 6. 总结
ERNIE 4.5 引入异构 MoE 架构，为文本和视觉模态设计不同参数量的专用专家（视觉=1/3 文本参数），通过模态隔离路由避免跨模态干扰，配合路由正交化损失和 3D RoPE 实现高效的多模态原生建模。这是 MoE 设计从"同构通用"走向"异构专用"的重要一步。

**速记 pipeline**: Input → 模态标识 → Router(modality-isolated: text→text experts+shared, vision→vision experts+shared) → Top-8 异构专家(vision expert = 1/3 params) → weighted sum → Output; 3D RoPE(T低频, H/W交替); 训练: router orthogonal loss + multimodal token-balanced loss
