# Duo-LLM: A Framework for Studying Adaptive Computation in Large Language Models

**论文**: Duo-LLM: A Framework for Studying Adaptive Computation in Large Language Models
**作者**: (Anonymous / Multiple authors)
**年份**: 2024
**来源**: arXiv:2410.10846
**标签**: 自适应计算, 动态 FFN, 条件计算, 推理效率

---

## 0. 摘要
Duo-LLM 提出了一个研究 LLM 中自适应计算的框架。核心思想是在每层同时集成大小两个 FFN 模块，并通过动态路由根据 token 复杂度选择使用大模块还是小模块（甚至跳过某层）。通过 oracle 穷举搜索最优路由模式，发现当前学习式路由器与理论最优之间存在显著差距。

## 1. 方法动机
a) 当前 LLM 对每个 token 使用固定的计算量，但不同 token 的"难度"差异巨大。
b) 简单 token（如常见词、标点）不需要完整的 FFN 计算，浪费了大量算力。
c) 假设：通过动态分配计算资源（不同大小的 FFN），可以在保持性能的同时显著降低平均计算成本。

## 2. 方法设计
a) 架构设计：
   - 每层包含两个 FFN 模块：一个大的（标准大小）和一个小的（辅助的）
   - 路由器决定每个 token 使用哪个模块：大模块 / 小模块 / 跳过
   - 路由决策基于当前 token 的隐藏状态

b) Oracle 分析：
   - 穷举所有可能的路由组合
   - 找到在固定计算预算下的最优路由模式
   - 用于评估学习式路由器的性能上限

c) 关键发现：
   - 在某些层中只使用大模块就能超越所有层都用大模块的基线
   - 不同层对大/小模块的需求不均匀
   - 当前学习式路由器远未达到 oracle 上限

## 3. 与其他方法对比
| 方法 | 优点 | 缺点 | 核心区别 |
|------|------|------|----------|
| MoE | 多专家选择 | 所有专家等大 | 选"哪个"专家 |
| MoD (Mixture-of-Depths) | 层级跳过 | 二元选择（计算/跳过）| 选"是否"计算 |
| Early Exit | 提前停止 | 只能在末尾退出 | 选"何时"停止 |
| **Duo-LLM** | **异构模块选择** | **Oracle 差距大** | **选"多大"的模块** |

## 4. 实验表现
- 揭示了学习式路由器与理论最优之间存在显著差距
- 计算分配的非均匀性：中间层比首尾层更需要大模块
- 为自适应计算研究提供了系统性的分析框架和基准

## 5. 对 Channel Mixer 的意义
Duo-LLM 从另一个维度思考 Channel Mixer：不是"用什么样的 FFN"而是"用多大的 FFN"。这与 MoE（选哪个专家）和 MoD（是否计算）形成互补。三者共同构成了条件计算在 Channel Mixer 维度上的完整谱系：选"哪个"（MoE）、选"是否"（MoD）、选"多大"（Duo-LLM）。

## 6. 总结
a) 核心思想：根据token难度动态选择不同大小的FFN模块（20字）
b) 速记 pipeline：
   1. 每层配备大小两个 FFN 模块
   2. 路由器根据 token 复杂度选择大模块/小模块/跳过
   3. Oracle 穷举搜索最优路由作为性能上限
   4. 发现学习式路由器与最优解差距显著
