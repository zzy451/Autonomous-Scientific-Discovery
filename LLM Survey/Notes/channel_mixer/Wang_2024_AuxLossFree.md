# Auxiliary-Loss-Free Load Balancing Strategy for Mixture-of-Experts

**论文**: Auxiliary-Loss-Free Load Balancing Strategy for Mixture-of-Experts
**作者**: Lean Wang, Huazuo Gao, Chenggang Zhao, Xu Sun, Damai Dai
**年份**: 2024
**来源**: arXiv:2408.15664
**标签**: MoE, 负载均衡, 无辅助损失

---

## 0. 摘要
传统 MoE 使用辅助损失（auxiliary loss）实现负载均衡，但辅助损失会产生干扰梯度（interference gradients），损害模型性能。本文提出"无损失均衡"（Loss-Free Balancing）策略，通过动态调整专家偏置（expert-wise biases）在不使用辅助损失的情况下实现负载均衡。

## 1. 方法动机
a) 辅助损失是 MoE 负载均衡的标准手段（自 Shazeer 2017 以来），但它会对主任务梯度产生干扰。
b) 辅助损失系数的选择是一个困难的超参数调优问题：太小则负载不均，太大则损害性能。
c) 假设：可以通过在路由评分上直接加偏置来调整负载分配，完全不需要辅助损失。

## 2. 方法设计
a) 核心机制：
   - 在 Top-K 路由决策之前，为每个专家的路由得分加上一个偏置项 b_i
   - 修改后的路由得分：s'_i = s_i + b_i
   - 使用修改后的得分进行 Top-K 选择

b) 偏置动态更新：
   - 监控每个专家最近的实际负载
   - 如果某专家负载偏低 → 增加其偏置 → 更多 token 被路由到该专家
   - 如果某专家负载偏高 → 降低其偏置 → 减少 token 流入
   - 形成动态反馈循环

c) 关键性质：
   - 偏置仅影响路由决策，不影响最终加权系数（用原始得分加权）
   - 完全移除辅助损失项
   - 在全局和 batch 级别都实现更好的负载均衡

## 3. 与其他方法对比
| 方法 | 均衡质量 | 对模型性能的干扰 | 超参数 |
|------|----------|-------------------|--------|
| 辅助损失（标准） | 中等 | 有（干扰梯度） | 需调系数 |
| Expert Choice | 完美 | 无 | 受限于非自回归 |
| **Loss-Free Balancing** | **更好** | **无** | **自适应** |

## 4. 实验表现
- 在 3B 参数 MoE 模型上验证（训练 200B tokens）
- 始终比辅助损失方法取得更低的验证集损失
- 在全局和 batch 级别都实现了更均匀的专家利用率
- 该方法被 DeepSeek-V3 采用

## 5. 对 Channel Mixer 的意义
这是 MoE 负载均衡范式的重要转变。传统的辅助损失方法（从 Shazeer 2017 到 Switch Transformer）被证明会伤害模型质量，而本文提出的无辅助损失方案被 DeepSeek-V3 采用，是该模型取得优异表现的关键技术之一。

## 6. 总结
a) 核心思想：动态偏置替代辅助损失实现无损负载均衡（17字）
b) 速记 pipeline：
   1. 监控每个专家的实际负载
   2. 根据负载差异动态调整路由偏置
   3. 偏置仅影响路由选择，不影响加权系数
   4. 完全移除辅助损失，消除梯度干扰
