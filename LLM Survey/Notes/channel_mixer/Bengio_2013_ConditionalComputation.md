# Conditional Computation 理论基础
**作者**: Yoshua Bengio, Nicholas Léonard, Aaron Courville (2013); Emmanuel Bengio, Pierre-Luc Bacon, Joelle Pineau, Doina Precup (2015)
**年份**: 2013 / 2015 | **arxiv**: 1308.3432 (2013), 1511.06297 (2015)

## 0. 摘要翻译

**[Bengio et al., 2013]** 探讨了如何通过随机神经元实现条件计算：即根据输入选择性地激活网络的部分组件，从而在不增加推理成本的前提下扩大模型容量。核心挑战是如何通过不可微的离散门控决策进行梯度传播。

**[E. Bengio et al., 2015]** 将条件计算建模为强化学习问题，使用策略梯度算法学习输入依赖的"dropout"策略，实现在保持精度的同时加速推理。

## 1. 方法动机

### 1.1 核心问题
- 深度网络的计算成本与模型容量（参数量）成正比
- 传统网络对每个输入都激活所有参数，造成大量冗余计算
- **目标**：解耦模型容量与计算成本——大模型不一定需要大计算

### 1.2 关键洞察
- 不同输入所需的"专业知识"不同，不必对所有输入激活所有计算单元
- 人脑也是条件计算的典范：不同任务激活不同脑区
- 如果能学到一个"门控器"，根据输入决定激活哪些单元，就能实现稀疏计算

### 1.3 技术障碍
- 门控决策是**离散的**（开/关），不可微分，无法直接反向传播
- 如何在门控的离散性与端到端训练的可微性之间取得平衡？

## 2. 方法设计

### 2.1 [2013] 随机神经元梯度估计

Bengio 等人提出四种通过随机神经元传播梯度的方法：

| 方法 | 原理 | 优缺点 |
|------|------|--------|
| **REINFORCE 估计器** | 最小方差无偏估计，基于对数似然技巧 | 无偏但方差大 |
| **分解法** | 将操作分解为随机二值部分 + 可微部分 | 降低方差但有偏 |
| **噪声注入** | 在可微计算图中注入加性/乘性噪声 | 简单但近似 |
| **Straight-Through 估计器** | 前向用离散采样，反向直接复制梯度 | 最简单，启发式但实用 |

核心公式（REINFORCE）：
$$\nabla_\theta \mathbb{E}[f(z)] = \mathbb{E}[f(z) \nabla_\theta \log p(z|\theta)]$$

其中 $z$ 是随机门控变量，$f$ 是损失函数，$\theta$ 是门控参数。

### 2.2 [2015] 强化学习框架

将条件计算建模为马尔可夫决策过程：
- **状态**: 当前层的输入特征
- **动作**: 对每个单元/块决定激活或不激活（伯努利掩码）
- **策略**: $\pi_\theta(a|x)$ — 输入依赖的激活策略
- **奖励**: 预测准确率 - λ × 计算成本

训练目标：
$$\min_\theta \underbrace{L_{\text{task}}}_{\text{任务损失}} + \lambda \underbrace{L_{\text{cost}}}_{\text{计算代价惩罚}} + \mu \underbrace{L_{\text{diversity}}}_{\text{多样性正则}}$$

多样性正则防止策略退化为"总是激活同一子集"。

## 3. 对比

### 3.1 Conditional Computation vs. Dropout
| 特性 | Dropout | Conditional Computation |
|------|---------|------------------------|
| 掩码生成 | 随机（与输入无关） | **输入依赖** |
| 目的 | 正则化 | **加速推理** |
| 训练时 | 随机丢弃 | 学习最优激活策略 |
| 推理时 | 关闭 | **保持稀疏激活** |

### 3.2 Conditional Computation → MoE 的演化路径
```
Conditional Computation (2013-2015)
  │ "根据输入选择性激活"
  ├── Shazeer 2017: Sparsely-Gated MoE
  │     "门控网络 + 专家网络"
  ├── Switch Transformer (2022)
  │     "Top-1 路由，极致稀疏"
  ├── Mixture of Depths (2024)
  │     "跳过不重要的 token"
  └── DeepSeek MoE / Mixtral (2024)
        "共享专家 + 细粒度路由"
```

### 3.3 与现代 MoE 的关系

| 概念 | 条件计算原始框架 | 现代 MoE |
|------|:---:|:---:|
| 门控方式 | 随机/RL 策略 | 可微 softmax/sigmoid 门控 |
| 稀疏单元 | 单个神经元/块 | **专家网络**（完整 FFN） |
| 训练方法 | REINFORCE / STE | 端到端可微 + 辅助负载均衡损失 |
| 粒度 | 细粒度（神经元级） | 粗粒度（专家级） |

## 4. 实验

### [2013] 随机神经元实验
- 在 MNIST 等任务上验证了 Straight-Through 估计器的实用性
- STE 虽然有偏，但在实践中训练稳定且有效
- 为后续大量离散变量训练工作奠定基础

### [2015] 条件计算加速实验
- 在 CIFAR-10、SVHN 上验证
- 在保持相近精度的前提下，成功减少约 30-50% 的计算量
- 策略梯度训练的多样性正则对防止模式坍塌至关重要

## 5. 总结

### a) 一句话
条件计算是 MoE 的理论根基，提出了"根据输入动态选择网络子集"的核心范式，并解决了离散门控的梯度传播问题（STE、REINFORCE），直接催生了现代稀疏 MoE 架构。

### b) 速记 pipeline
```
Input x → Gate(x) → Bernoulli Mask [0,1,1,0,1,...] → 仅激活被选中的单元/专家
                ↑                                          ↓
          策略梯度 / STE                              稀疏前向传播
                ↑                                          ↓
          Reward = Accuracy - λ·Cost                  Output y
```
