# Baichuan-M1: Pushing the Medical Capability of Large Language Models
**作者**: Baichuan Intelligence | **年份**: 2025 | **arxiv**: 2502.12671

## 0. 摘要翻译
Baichuan-M1 是百川智能从零训练的 14B 参数大语言模型，专为医疗场景优化，同时保持通用能力。模型在 20 万亿 token 上训练，采用了一系列创新的架构设计和训练方法，在通用能力和医疗专业能力之间取得平衡。其核心架构创新之一是在 KV（Key-Value）上引入时序短卷积（temporal short convolution），增强了模型对序列中局部模式的捕捉能力。

## 1. 方法动机
- 标准 Transformer 的自注意力机制擅长捕捉全局依赖，但对局部连续模式（如医学术语中的字符级模式、时序数据中的短程依赖）的建模效率不高
- 在 KV cache 上施加短卷积可以在不显著增加计算开销的前提下，为注意力机制注入局部归纳偏置（local inductive bias）
- 医疗文本中存在大量需要精确局部模式匹配的场景（如药物名称、诊断编码、检验数值序列），局部特征增强对此类任务尤为重要

## 2. 方法设计（重点：与六维度框架的关联）

### KV 时序短卷积
- **核心思想**: 在注意力计算之前，对 Key 和 Value 的表示沿序列维度施加一维短卷积（kernel size 通常为 3-7）
- **实现方式**:
  1. 输入经过 Q/K/V 投影后，K 和 V 分别通过一个深度可分离的一维卷积层（depthwise 1D conv）
  2. 卷积沿时间/序列维度滑动，融合相邻 token 的 KV 表示
  3. 卷积后的 K', V' 再参与标准的注意力计算
- **因果性保证**: 使用因果卷积（causal convolution），确保位置 t 的输出仅依赖于位置 ≤ t 的输入，兼容自回归生成
- **与 Mamba/H3 的联系**: 类似于 Mamba 中的短卷积分支，但 Baichuan-M1 将其嵌入到标准 Transformer 注意力中，而非替代注意力

### 与六维度框架的关联
- **Token Mixer 维度**: 这是对标准注意力的增强——在全局注意力之前加入局部卷积，形成"局部卷积 + 全局注意力"的混合 token mixing 策略
- **与 Hyena/H3 的关系**: H3 和 Hyena 也使用短卷积作为序列混合的组件，但它们用卷积替代注意力；Baichuan-M1 则是卷积辅助注意力
- **KV 增强视角**: 可以理解为一种 KV cache 的预处理/平滑操作，使得注意力看到的 Key 和 Value 已经融合了局部上下文信息

### 其他架构特点
- 14B 参数规模，标准 Transformer 架构为基础
- 20T token 训练数据，包含大量医疗领域数据
- 采用多阶段训练：通用预训练 → 医疗领域继续预训练 → 指令微调

## 3. 与其他方法对比
| 方法 | 局部特征机制 | 位置 | 与注意力的关系 |
|------|------------|------|--------------|
| 标准 Transformer | 无 | - | 纯注意力 |
| Mamba | 短卷积 + SSM | 替代注意力 | 替代 |
| H3 | 短卷积 + SSM | 替代注意力 | 替代 |
| Baichuan-M1 | KV 短卷积 | 注意力内部 | 增强/辅助 |
| RWKV | Token shift | 替代注意力 | 替代 |

- Baichuan-M1 的独特之处在于将短卷积嵌入注意力内部（作用于 KV），而非用其替代注意力
- 这种设计保留了注意力的全局建模能力，同时以极低成本引入局部偏置

## 4. 实验表现
- 在通用基准（C-Eval、CMMLU、MMLU 等）上达到同规模模型的竞争水平
- 在医疗基准上显著优于同规模通用模型和部分医疗专用模型
- 医疗问答、临床推理、医学知识等任务上表现突出
- KV 短卷积带来的额外计算开销极小（<1% FLOPs 增加），但在需要局部模式匹配的任务上有稳定提升

## 5. 总结
a) Baichuan-M1 在标准 Transformer 注意力的 KV 上施加时序短卷积，以极低成本为全局注意力注入局部归纳偏置，特别适合医疗等需要精确局部模式匹配的领域。

b) 速记 pipeline: Input → QKV Proj → K,V 经 causal 1D depthwise conv → 融合局部上下文的 K',V' → Standard Attention(Q, K', V') → Output
