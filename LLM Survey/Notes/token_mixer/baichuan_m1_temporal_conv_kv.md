# Baichuan-M1: KV 时序短卷积 (Temporal Short Convolution on KV)
**来源**: Baichuan Intelligence | **年份**: 2025 | **arxiv**: 2502.12671

## 0. 背景
Baichuan-M1 是百川智能从零训练的 14B 参数大语言模型，专为医疗场景优化，同时保持通用能力。模型在 20 万亿 token 上训练。其核心架构创新之一是在注意力机制的 KV（Key-Value）上引入时序短卷积（temporal short convolution），将来自线性状态空间模型（如 Mamba、H3）的局部卷积思想嵌入到标准 Transformer 注意力内部，增强模型对序列中局部模式的捕捉能力。这一设计在六大类别中属于完全未覆盖的独特创新，位于 Token Mixer 维度中"注意力增强"这一子类。

## 1. 方法动机
- **全局注意力的局部盲区**: 标准 Transformer 自注意力擅长捕捉全局依赖（任意两个 token 间的关系），但对局部连续模式（如字符级 n-gram 模式、时序数据中的短程依赖）的建模效率不高，需要多层堆叠才能隐式学到局部模式
- **局部归纳偏置的价值**: 在医疗文本中存在大量需要精确局部模式匹配的场景——药物名称（如"阿莫西林"每个字符都关键）、诊断编码（ICD 编码的精确字符序列）、检验数值序列（连续数值的趋势判断）。显式的局部归纳偏置对此类任务尤为重要
- **来自 NAS 的启发**: 论文指出这种结构通过神经架构搜索（NAS）在语言建模任务中被发现是有益的，并在 Mamba 等非 Transformer 架构中广泛使用
- **极低成本增强**: 在 KV 上施加短卷积可以在不显著增加计算开销（<1% FLOPs）的前提下，为注意力机制注入局部归纳偏置

## 2. 方法设计（详细）

### 2.1 核心架构：KV 时序短卷积
在注意力计算之前，对 Key 和 Value 的表示沿序列维度施加一维短卷积：

```
输入 X → Q/K/V 线性投影
         ├─ Q → 不变
         ├─ K → Causal 1D Depthwise Conv → K'
         └─ V → Causal 1D Depthwise Conv → V'
         
Attention(Q, K', V') = softmax(QK'^T / sqrt(d)) V'
```

### 2.2 实现细节
- **卷积类型**: 深度可分离一维卷积（depthwise 1D convolution），每个注意力头/通道独立卷积，不跨通道混合
- **卷积核大小**: 通常为 3-7（短卷积），沿序列/时间维度滑动
- **因果性保证**: 使用因果卷积（causal convolution），确保位置 t 的输出仅依赖于位置 <= t 的输入。实现方式为在左侧 zero-padding (kernel_size - 1)，兼容自回归生成
- **作用位置**: 仅作用于 K 和 V，不改变 Q。这是因为 K 和 V 代表"被关注的内容"，融合局部上下文使得每个位置提供的 key/value 信息更丰富；Q 代表"查询意图"，保持不变以维持查询的精确性
- **与 KV Cache 的兼容性**: 推理时，卷积的滑动窗口状态（最近 kernel_size - 1 个 token 的 K/V）需要与 KV Cache 一起维护，额外状态量极小

### 2.3 数学形式化
设 K_t 为第 t 个位置的 Key 向量，卷积核为 w = [w_0, w_1, ..., w_{k-1}]（k 为核大小），则：

```
K'_t = sum_{i=0}^{k-1} w_i * K_{t-i}   (因果卷积，t-i < 0 时 K = 0)
```

这等价于在注意力看到的 Key 中融合了相邻 token 的信息，使 K'_t 不仅包含位置 t 的语义，还包含其局部上下文的信息。

### 2.4 与 Mamba/H3 中短卷积的联系与区别
- **Mamba/H3**: 短卷积是序列混合的核心组件之一，用于替代注意力。卷积 + SSM 构成完整的 token mixer
- **Baichuan-M1**: 短卷积嵌入到标准 Transformer 注意力内部，作为注意力的辅助/增强。注意力仍是主要的 token mixing 机制
- **RWKV**: 使用 token shift（可看作 kernel=2 的特殊卷积）替代注意力
- **关键区别**: Baichuan-M1 是"卷积辅助注意力"而非"卷积替代注意力"

### 2.5 与六维度框架的关联
- **Token Mixer 维度**: 这是对标准注意力的增强——在全局注意力之前加入局部卷积，形成"局部卷积 + 全局注意力"的混合 token mixing 策略
- **KV 增强视角**: 可以理解为一种 KV 的预处理/平滑操作，使得注意力看到的 Key 和 Value 已经融合了局部上下文信息
- **与 Hyena/H3 的关系**: 概念上相似（都用短卷积辅助序列建模），但在架构位置和角色上有本质区别

### 2.6 其他架构特点
- 14B 参数规模，标准 Transformer 架构为基础
- 20T token 训练数据，包含大量医疗领域数据
- 多阶段训练：通用预训练 → 医疗领域继续预训练 → 指令微调

## 3. 与其他方法对比

| 方法 | 局部特征机制 | 作用位置 | 与注意力的关系 | 额外开销 |
|------|------------|---------|--------------|---------|
| 标准 Transformer | 无 | - | 纯注意力 | - |
| Mamba | 短卷积 + SSM | 替代注意力 | 替代 | N/A |
| H3 | 短卷积 + SSM | 替代注意力 | 替代 | N/A |
| RWKV | Token shift (k=2) | 替代注意力 | 替代 | N/A |
| Hyena | 长卷积 | 替代注意力 | 替代 | N/A |
| **Baichuan-M1** | **KV 短卷积** | **注意力内部(K,V)** | **增强/辅助** | **<1% FLOPs** |
| Conv-Augmented Attention (通用) | QKV 卷积 | 注意力内部(Q,K,V) | 增强 | ~1-3% FLOPs |

**Baichuan-M1 的独特定位**: 将短卷积嵌入注意力内部（仅作用于 KV），是"卷积辅助注意力"的代表设计。这种设计保留了注意力的全局建模能力，同时以极低成本引入局部偏置，是 Transformer 与 SSM 思想融合的一种温和方案。

## 4. 实验/工程验证
- **通用基准**: 在 C-Eval、CMMLU、MMLU 等通用基准上达到同规模模型（14B）的竞争水平
- **医疗基准**: 在医疗专业基准上显著优于同规模通用模型和部分医疗专用模型，在医疗问答、临床推理、医学知识等任务上表现突出
- **KV 卷积的效果**: 额外计算开销极小（<1% FLOPs 增加），但在需要局部模式匹配的任务上有稳定提升
- **工程友好性**: 卷积操作在 GPU 上高度优化，深度可分离卷积的参数量和计算量都极小，推理时仅需维护极少的额外状态

## 5. 对本综述的启示
- **Token Mixer 的新子类**: KV 时序短卷积开创了"注意力内部增强"这一 token mixer 子类别，既不是纯注意力也不是注意力替代，而是注意力辅助
- **Transformer 与 SSM 融合的光谱**: 从纯 Transformer → KV 卷积增强 → Jamba 式交替 → 纯 SSM，形成了一个连续的设计光谱。Baichuan-M1 位于"最温和融合"的一端
- **领域特化的架构设计**: 展示了如何通过架构创新（而非仅靠数据）来增强模型在特定领域（医疗）的能力
- **极低成本的创新价值**: 证明了即使是极小的架构修改（<1% FLOPs 增加）也能带来有意义的性能提升
- **六维度框架的交叉区域**: 这一设计横跨 Token Mixer 和局部归纳偏置两个维度，说明六维度框架中存在有意义的交叉创新空间

## 6. 总结
Baichuan-M1 在标准 Transformer 注意力的 KV 上施加因果深度可分离短卷积，以极低成本（<1% FLOPs）为全局注意力注入局部归纳偏置。这一设计是 Transformer 与 SSM 融合光谱中"最温和方案"的代表，保留了注意力的全部全局建模能力，同时显式增强了局部模式捕捉。

**速记 pipeline**: Input → QKV Proj → K,V 经 causal 1D depthwise conv(kernel=3~7) → 融合局部上下文的 K',V' → Standard Attention(Q, K', V') → Output
