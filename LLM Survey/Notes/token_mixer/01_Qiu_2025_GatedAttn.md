# Gated Attention for Large Language Models
**作者**: Qiu et al. | **年份**: 2025 | **arxiv**: 2505.06708

## 0. 摘要翻译
标准的 Softmax 注意力在大语言模型 (LLM) 中取得了巨大成功，但其对所有 token 一视同仁地计算注意力分数的方式可能并非最优。本文提出 **Gated Self-Attention (GSA)**，在注意力机制中引入可学习的门控机制，使模型能够自适应地控制每个注意力头中信息流的强度。实验表明，在多项语言建模基准上，GSA 在不增加显著计算开销的情况下，稳定优于标准的 Multi-Head Attention (MHA)。该方法可以作为即插即用的替换，直接应用于现有的 Transformer 架构中。

## 1. 方法动机
a) **为什么提出这个方法**：标准 Softmax Attention 对所有位置的 value 进行加权求和，但不同 head 和不同 token 位置上，注意力信息的重要程度可能差异很大。现有机制缺乏对注意力输出进行细粒度控制的能力。

b) **现有方法的痛点**：
- 标准 MHA 中每个 head 的输出直接拼接后线性投影，缺乏对单个 head 输出强度的自适应调节
- Softmax 归一化使得注意力权重必须为正且和为 1，限制了表达能力
- 即使某个 head 计算的注意力信息质量低或不相关，其输出仍然不加区分地参与后续计算

c) **研究假设/直觉**：通过引入额外的门控信号（gating signal），让模型学习"何时听取注意力的输出、何时抑制"，可以在不改变注意力核心计算流程的前提下，提升模型的表达能力和性能。

## 2. 方法设计
a) **方法流程 (pipeline)**：
1. 输入 token 经过标准的 Q, K, V 投影
2. 计算标准的 Softmax 注意力分数并加权求和得到注意力输出
3. **额外引入一个门控分支**：从输入 token 通过可学习的线性变换生成门控信号 (gate)
4. 门控信号经过 sigmoid（或 tanh）激活，逐元素乘以注意力输出
5. 将门控后的各 head 输出拼接，经过输出投影

b) **核心模块功能**：
- **Gate 生成器**：一个轻量级的线性投影层，将输入 x 映射到与注意力输出同维度的门控向量 g = σ(W_g · x + b_g)
- **逐元素门控**：output = g ⊙ Attention(Q, K, V)，实现对注意力输出的自适应缩放
- **参数初始化**：门控的偏置项初始化为正值（如 1），使得训练初期门控近似全开，行为接近标准注意力，确保训练稳定性

c) **关键公式解释**：
- 标准注意力：Attn(Q,K,V) = softmax(QK^T / √d_k) V
- 门控注意力：GatedAttn(x) = σ(x W_g) ⊙ Attn(Q,K,V)
- 其中 σ 为 sigmoid 函数，⊙ 为逐元素乘法

## 3. 与其他方法对比
- **与标准 MHA 的本质区别**：增加了逐 head、逐 token 的门控调节，标准 MHA 则直接拼接所有 head 输出
- **与 GLU (Gated Linear Unit) 的关系**：理念类似 GLU 中的门控思想，但 GLU 用于 FFN 层，本文将门控引入注意力层
- **与 GQA/MQA 的区别**：GQA/MQA 关注减少 KV head 数以节省内存，本文关注的是对注意力输出进行自适应调制
- **创新点**：将门控机制零成本地集成到标准注意力中，即插即用
- **适用场景**：任何使用标准 MHA 的 Transformer LLM，可直接替换

## 4. 实验表现
- **关键结果**：在多个语言建模基准 (如 WikiText-103, LAMBADA, HellaSwag 等) 上，GSA 相比标准 MHA 取得了一致的 perplexity 下降和下游任务提升
- **优势场景**：尤其在需要模型区分不同 head 信息重要性的任务上表现更好；长上下文场景中门控可以抑制远程的噪声注意力
- **局限性**：
  - 引入了少量额外参数（门控投影矩阵），但相对于整体模型参数量几乎可忽略
  - 在短序列或简单任务上提升幅度有限
  - 作为 2025 年新提出的方法，尚未在超大规模模型（如 >70B）上得到广泛验证

## 5. 总结
a) **一句话概括（≤20字）**：在注意力输出上加门控，即插即用提升性能。

b) **速记 pipeline**：
`Input → Q,K,V投影 → Softmax Attention → [Gate分支: σ(xW_g)] ⊙ AttnOutput → Concat → OutputProj`
