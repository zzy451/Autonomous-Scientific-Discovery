## 0. Abstract Overview

This paper presents a dynamic knowledge-graph framework for distributed self-driving laboratories. Its main novelty is that autonomous laboratory components can be coordinated across locations as executable knowledge units. For current ASD classification purposes, the key point is that the system is not validated on an abstract middleware benchmark; it is validated on a concrete chemistry reaction-optimization problem centered on an aldol condensation reaction.

## 4. Method and Workflow

### 4.1 Method motivation

The paper addresses a distributed-lab bottleneck: how to let geographically separated self-driving labs share actionable knowledge and coordinate experiments in real time instead of operating as isolated closed loops.

### 4.2 Workflow sketch

1. Encode reaction knowledge and laboratory actions into a dynamic knowledge graph.
2. Assign executable knowledge components to autonomous agents or laboratory nodes.
3. Run cross-site experiments on distributed hardware.
4. Return yields / costs / outcomes into the shared knowledge structure.
5. Update the search and continue closed-loop optimization.
6. Produce improved chemistry conditions and a Pareto tradeoff surface.

### 4.3 System emphasis

The important writeback correction is to keep the architecture discussion while making the chemistry object explicit. The knowledge graph is the enabling representation; it is not the primary scientific object.
