# 科学领域分类方案

## 一、文档定位

这份方案用于构建本 Autonomous Scientific Discovery 综述中 Agent 文献的科学研究对象索引体系。正式主科学领域只按科学研究对象划分，并采用封闭的 `01–11` 十一个一级类 
它不是简单照搬单一图书馆分类法，而是以图书馆学的成熟分类逻辑为基础，融合多套主流体系，形成一套更适合学术综述、文献检索、研究版图分析、领域映射和后续标签标注的统一分类框架。

这套框架的目标有四个：

1. 能覆盖当代主要科学领域，并保留足够的细粒度。
2. 层级清楚，适合作为综述目录、表格索引和文献编码体系。
3. 与主流图书馆分类法保持可对照性，便于增强学术规范性。
4. 对新兴交叉领域具有扩展能力，不会因为前沿学科出现而失效。

## 二、采用的图书馆学依据

本方案主要参考以下四类成熟分类体系的逻辑：

1. 中国图书馆分类法  
特点是中文语境下最常用，天然适合中文资料组织；在“自然科学总论、数理科学和化学、地球科学、生物科学、医药、农业、工业技术、交通、航空航天、环境科学”等方面结构清晰。

2. 美国国会图书馆分类法  
特点是英文世界学术出版和高校图书馆体系中影响极大；其把“科学、医学、农业、技术”分设为相邻但独立的大类，尤其适合英文综述的国际表达。

3. 杜威十进分类法  
特点是层级规范、数字结构整齐，便于做树状索引；其对“自然科学”和“技术科学”分开的处理很适合构建综述分析框架。

4. 国际通用十进分类法  
特点是强调可扩展、可组合、适合交叉学科表达；适合处理当代科学中大量“领域交叉、方法交叉、平台交叉”的情况。

## 三、分类设计原则

### 1. 先按“科学研究对象”分，不按“技术方法”分

本综述的科学领域主索引应优先依据研究对象、自然现象、生命层级、工程对象、系统类型和最终科学问题来划分，而不是按 Agent 类型、算法方法、模型架构、学院设置、行业行政口径或期刊名称来分。

原因是：本综述要回答的问题不是“某种 Agent 技术有哪些应用”，而是“Agent 已经进入了哪些科学领域，并在这些领域中承担了哪些科学研究任务”。因此，分子、反应、材料、蛋白质、细胞、疾病、气候、天体、作物、工程系统等科学对象，应当优先于大语言模型、强化学习、多 Agent、机器人平台等技术路径，成为主分类依据。

在具体归类时，Agent 技术路线只作为副标签记录；一篇文献的主科学领域由其研究对象和最终科学问题决定。

更强规则是：即使论文提出完整的 Autonomous Scientific Discovery 系统、通用科研 Agent、AI co-scientist 或闭环科研工作流，只要其主要实验、验证或科学贡献面向某一具体科学对象，就应归入该具体科学领域。系统是否通用、是否多 Agent、是否闭环、是否具有较高自主程度，不改变主科学领域归属。

### 2. 保持“基础科学、应用科学、技术科学”相互区分

图书馆学里一个非常重要的逻辑是：  
“科学”与“技术”不能完全混在一起。  
例如，化学与化学工程、物理学与电子工程、生物学与生物工程、医学基础研究与临床医疗服务，应当有清楚边界。

### 3. 允许交叉，但交叉建立在主类稳定的前提上

交叉学科不能替代主学科体系。  
更合理的做法是先建立稳定的一级和二级主类，再把交叉领域作为“复合索引”或“第二标签”来处理。

### 4. 层级深度以“三到四层”为宜

一级类用于综述总目录。  
二级类用于章节。  
三级类用于表格、图谱、论文归档。  
四级类只在文献规模足够大时使用。

### 5. 面向“科学研究”而不是“全部知识”

虽然图书馆分类往往覆盖哲学、历史、文学、艺术等全部知识门类，但本方案只保留与“科学研究体系”直接相关的部分，把重点放在形式科学、自然科学、生命与医学、农业、工程技术、环境地球科学、社会与行为科学以及科学知识生产系统。

## 四、建议采用的统一编码结构

建议使用四级编码：

- 一级类：大领域，使用两位数字。
- 二级类：学科群，使用两位数字。
- 三级类：具体学科，使用两位数字。
- 四级类：专题方向，按需要继续扩展。

示例：

- `03` 化学科学
- `03.03` 合成、反应与催化
- `03.03.02` 反应设计
- `03.03.02.01` 催化反应设计

这种结构既保留图书馆学的层级感，也方便后续表格排序、数据库标注和文献统计。

## 六、建议的主索引与副标签机制

为了兼顾“稳定分类”和“前沿交叉”，建议采用“主科学领域 + 副标签”的机制。主科学领域只使用封闭的 `01–11` 十一个一级类；Agent 类型、科研流程角色、自主能力、验证方式和交叉属性全部作为副标签。

### 1. 主索引

每篇文献必须有且仅有一个主索引。  
主索引由其最终科学研究对象决定，必须归入 `01–11` 中对应的正式主科学领域。

例如：

- 蛋白质结构预测，主索引应放在 `06.03.06` 蛋白质科学。
- 小分子药物设计，主索引应放在 `07.04.01` 药物发现。
- 催化反应机理研究，主索引应放在 `03.02.04` 物理化学，或 `03.03.02` 反应设计。
- 领域无关型通用科研 Agent，主索引应放在 `01.04.02` 通用科研 Agent。
- 科学知识网络演化研究，主索引应放在 `11.07.03` 科学知识网络，或 `11.07.10` 学科演化与跨学科发现机制。

### 2. 副标签

副标签用于标注 Agent 类型、科研流程角色、自主能力、验证方式、方法平台或交叉属性，可多选。

例如：

- 计算驱动
- 实验驱动
- 理论驱动
- 仿真驱动
- 数据密集型
- 高通量筛选
- 机器人实验平台
- 多尺度建模
- 多组学
- 遥感与地理空间

### 3. 交叉领域处理原则

交叉研究不能新建临时主类。如果研究对象能够稳定归入某一具体科学领域，则必须归入该领域，并把“计算驱动、数据驱动、仿真驱动、多模态、知识图谱、多 Agent、机器人平台、跨学科”等作为副标签。

### 4. 副标签维度

Agent 类型标签：LLM Agent、Planning Agent、Tool-using Agent、Retrieval-augmented Agent、Multi-Agent System、Robot / Embodied Agent、Human-in-the-loop Agent、Hybrid Agent。

科研流程角色标签：文献检索与阅读、知识抽取与组织、科学问题提出、假设生成、实验设计、仿真建模、工具调用与代码执行、实验执行、数据分析、结果解释、证据评估与验证、论文写作、端到端科研流程自动化。

自主能力标签：任务分解、计划生成、工具调用、记忆与状态维护、反馈迭代、自主决策、多 Agent 协作、环境交互、闭环实验。

验证方式标签：benchmark、仿真验证、高通量计算、机器人实验、湿实验、临床数据、真实场景部署、专家评估。

交叉属性标签：计算驱动、数据驱动、实验驱动、仿真驱动、多模态、多尺度建模、高通量筛选、知识图谱、数字孪生、机器人平台。

数字孪生不决定主类。地球系统数字孪生归 `05`，工业系统数字孪生归 `09`，交通系统数字孪生归 `10`，医学数字孪生归 `07`；“数字孪生”只作为交叉属性标签。

- `03` 对应化学科学

- `05` 对应地球科学、环境科学、地理科学、遥感与地球系统科学相关部分
- `06` 对应生物科学
- `07` 对应医药、卫生
- `08` 对应农业科学
- `09` 对应工业技术
- `10` 对应交通运输、航空航天、海洋工程
- `11` 对应社会科学、行为科学、经济学、公共管理、教育研究以及科学知识生产系统研究相关部分

### 2. 与美国国会图书馆分类法的关系

其核心启发是把以下大类严格区分：

- 科学
- 医学
- 农业
- 技术

本方案完整继承了这种分离逻辑，因此特别适合英文综述的国际表达。

### 3. 与杜威十进分类法的关系

其核心启发是：

- 自然科学与数学单列
- 技术与应用科学单列
- 医学、工程、农业等应用领域可以在技术大类下继续细分

本方案在编码结构上吸收了这种树状编号思路。

### 4. 与国际通用十进分类法的关系

其核心启发是“可扩展组合”和“适应交叉学科”。  
本方案中的副索引机制和交叉领域处理原则，直接吸收了这种思想。

## 九、当前阶段的建议结论

如果我们的目标是做一篇国际影响力强、结构严谨、可长期参考的英文综述，那么最稳妥的路线不是直接拿某一套图书馆分类法原样使用，而是：

1. 以图书馆学成熟体系作为合法性基础。
2. 以“主类稳定、交叉可扩展”为核心设计原则。
3. 建立一套适合综述写作和文献标注的统一中文索引体系。
4. 后续在正式英文写作时，再把这套中文体系翻译为标准化英文目录。
