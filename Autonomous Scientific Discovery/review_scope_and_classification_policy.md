# 综述项目要求与分类口径

## 一、项目定位

本项目拟撰写一篇高水平英文综述，主题为 `Autonomous Scientific Discovery`。日常讨论、内部工作文档和分类草案原则上使用中文；专有名词、论文题名、技术术语、分类体系名称和必要英文表达可以保留英文。

本综述的核心对象不是泛泛的“人工智能促进科学”，而是“Agent 参与、推动或承担科学发现过程”的研究工作。综述需要围绕 Agent 在科学研究中的角色、能力边界、应用学科、实验任务和自主程度展开。

## 二、最重要的边界：只讨论 Agent，不讨论泛化 AI for Science

本综述不等同于 `AI for Science` 综述，也不系统收集所有使用机器学习、深度学习或基础模型解决科学问题的论文。

只有当论文中的系统具有明确的 Agent 特征时，才纳入本综述。

这里的 Agent 可以是：

- 基于大语言模型的 Agent
- 经典定义下的智能 Agent
- 规划型 Agent
- 符号推理 Agent
- 多 Agent 系统
- 具身 Agent 或机器人实验 Agent
- 混合式 Agent，例如语言模型、工具调用、规划器、检索器、实验平台共同组成的系统

## 三、Agent 的纳入标准

一篇论文是否纳入，优先看系统是否满足 Agent 最低纳入标准，而不是看它是否使用了某种 AI 模型。

最低纳入标准是同时满足以下四点：

1. 面向明确科研目标。
2. 具有多步行动过程。
3. 至少具备计划生成、工具调用、反馈迭代、自主决策、多 Agent 协作中的一项。
4. 在科研流程中承担明确角色。

可辅助判断的 Agent 特征包括：

- 能根据目标进行任务分解、计划或策略选择
- 能调用外部工具、数据库、仿真器、实验设备或代码环境
- 能根据中间结果进行反馈、修正或迭代
- 能在科研流程中执行多步行动，而不是只输出一次预测结果
- 能承担某种科学研究角色，例如文献调研者、假设生成者、实验设计者、实验执行者、数据分析者、结果解释者或论文撰写者
- 在多 Agent 情况下，不同 Agent 之间存在任务分工、通信、协作、辩论或互相验证

## 四、排除标准

以下类型原则上不纳入本综述，除非论文明确构造了 Agent 式系统：

- 普通机器学习模型用于科学预测
- 普通深度学习模型用于分子生成、蛋白质预测、材料性质预测或医学影像分类
- 单次输入输出式的大模型问答、摘要或分类应用
- 只提出一个新模型或新算法，但没有计划、行动、工具调用、反馈迭代或自主决策机制
- 纯数据挖掘、纯仿真、纯优化或纯高通量筛选工作，但系统本身不具备 Agent 行为
- 传统 AI for Science 论文中没有 Agent 概念、Agent 架构或 Agent 行动闭环的工作

重要提醒：不能因为一篇论文使用了 AI、机器学习、深度学习、大模型或自动化实验，就自动把它纳入本综述。纳入的关键是 Agent，而不是 AI。

还要特别注意：`closed-loop`、`完整发现闭环`、`Autonomous Scientific Discovery 完整度` 不是当前综述的纳入门槛。它们只能作为补充描述，用于记录系统自主性强弱、科研闭环完整度或代表性高低，不能替代 Agent 最低纳入标准，也不能作为排除或纳入的单独依据。

## 五、分类体系的总体目标

我们需要借鉴图书馆学的分类思路，搭建一套完整、细化、稳定的人类科学领域划分体系。这个体系后续将作为本综述的总索引，用于归档大量 Agent 相关文献。

分类体系要服务于以下工作：

1. 收集 Agent 相关科学发现论文。
2. 判断每篇论文对应的科学研究领域。
3. 将论文归入相应的细分学科领域。
4. 在同一科学领域内进一步比较 Agent 参与了哪些科研任务。
5. 支持后续统计、制表、可视化和英文综述结构设计。

## 六、主分类口径：按科学领域分，而不是按 Agent 技术分

本综述的主分类依据是论文所面向的科学领域、科学对象和科学问题。

本综述的主分类不是 Agent 系统类型分类，而是 Agent 文献的科学研究对象分类。一篇论文是否提出完整科研 Agent 系统、通用 Agent 框架、闭环科研工作流或多 Agent 架构，不决定其科学对象归属。只要该论文最终面向某个具体科学研究对象，就必须归入该具体科学领域。

本项目当前不再把 `01.04` 视为与 `01–11` 并列的正式主科学对象类。`01.04` 单独摘出，用作“ASD 方法 / 通用科研 Agent / 通用科研工作流 / 通用 benchmark 存放区”，只收纳那些提出了 ASD 或 general research-agent 方法、但没有做任何具体科学对象实验、验证、benchmark task、case study 或结果报告的文献。

本项目当前允许同一篇文献进入多个具体科学模块，判定阈值采用“领域实验覆盖”口径：只要论文对某一科学领域中的具体对象做了可识别的实验、验证、benchmark task、case study 或结果报告，就可以把该论文归入对应科学模块。也就是说，多模块归入是实验与对象证据驱动的，而不是因为论文自称通用、跨学科或平台化；但不再要求每个模块都必须构成论文的主贡献。

主分类不按以下因素决定：

- 是否使用大语言模型
- 是否使用深度学习
- 是否使用强化学习
- 是否使用多 Agent 架构
- 是否使用机器人平台
- 是否发表在人工智能会议或计算机科学期刊
- 作者来自计算机学院、医学院、化学系还是工程学院

主分类应优先回答：

> 这篇论文中的 Agent 到底在帮助哪个科学领域完成什么研究？

例如：

- Agent 做分子反应设计，主分类应归入化学科学。
- Agent 做药物发现，主分类应归入医学与健康科学或药学相关子领域。
- Agent 做蛋白质功能探索，主分类应归入生命科学。
- Agent 做晶体结构或电池材料发现，主分类应归入材料科学。
- Agent 做气候建模或遥感分析，主分类应归入地球与环境科学。
- Agent 做数学定理证明或猜想探索，主分类应归入形式、信息与计算科学。
- Agent 同时对物理对象和生命科学对象做了可识别的实验、验证、case study 或结果报告，可以同时进入物理模块和生命科学模块。
- Agent 只是提出领域无关型科研 Agent 方法、通用科研 Agent、通用科学工作流或通用 benchmark，但没有做任何具体科学对象实验、验证、benchmark task、case study 或结果报告，不进入正式对象主类，而进入独立的 `01.04` 存放区。

## 七、副分类口径：Agent 在科研流程中做了什么

在确定主科学领域之后，再记录 Agent 在科研流程中的作用。副分类用于描述 Agent 的科学工作角色。

建议副分类包括：

- 文献检索与文献阅读
- 知识抽取与知识组织
- 科学问题提出
- 假设生成
- 实验设计
- 实验规划与资源调度
- 工具调用与代码执行
- 仿真建模
- 机器人实验执行
- 数据清洗与数据分析
- 模型构建与参数估计
- 结果解释
- 证据评估与假设验证
- 论文写作与报告生成
- 多 Agent 协同科研
- 端到端科研流程自动化

这些副分类不能替代主科学领域分类。它们用于回答“Agent 做了什么”，而主分类回答“Agent 在哪个科学领域做”。

## 八、Agent 类型作为标签，而不是主分类

Agent 的实现形态应作为标签记录，不能作为科学领域主分类。

可记录的 Agent 类型标签包括：

- 基于大语言模型的 Agent
- 经典智能 Agent
- 规划型 Agent
- 强化学习 Agent
- 符号推理 Agent
- 工具调用 Agent
- 检索增强 Agent
- 多 Agent 系统
- 机器人实验 Agent
- 具身 Agent
- 人机协同 Agent
- 混合式 Agent

例如，一篇论文使用大语言模型 Agent 做催化反应条件优化，其主分类是化学科学，副分类是实验设计或实验优化，Agent 类型标签才是基于大语言模型的 Agent。

### 统一副标签维度

Agent 类型标签：LLM Agent、Planning Agent、Tool-using Agent、Retrieval-augmented Agent、Multi-Agent System、Robot / Embodied Agent、Human-in-the-loop Agent、Hybrid Agent。

科研流程角色标签：文献检索与阅读、知识抽取与组织、科学问题提出、假设生成、实验设计、仿真建模、工具调用与代码执行、实验执行、数据分析、结果解释、证据评估与验证、论文写作、端到端科研流程自动化。

自主能力标签：任务分解、计划生成、工具调用、记忆与状态维护、反馈迭代、自主决策、多 Agent 协作、环境交互、闭环实验。

验证方式标签：benchmark、仿真验证、高通量计算、机器人实验、湿实验、临床数据、真实场景部署、专家评估。

交叉属性标签：计算驱动、数据驱动、实验驱动、仿真驱动、多模态、多尺度建模、高通量筛选、知识图谱、数字孪生、机器人平台。

数字孪生不决定主类。地球系统数字孪生归 `05`，工业系统数字孪生归 `09`，交通系统数字孪生归 `10`，医学数字孪生归 `07`；“数字孪生”只作为交叉属性标签。

## 九、每篇文献的推荐标注字段

后续整理文献时，每篇论文建议至少记录以下字段：

- 论文题名
- 年份
- 作者与机构
- 发表 venue
- 是否纳入本综述
- 纳入或排除理由
- 主科学领域模块
- 二级学科领域
- 三级细分领域
- 副科学领域或交叉领域
- 是否进入独立 `01.04` 存放区
- 是否具有具体科学对象研究实验
- 具体科学对象实验覆盖模块
- Agent 类型
- Agent 所承担的科研流程环节
- 是否具有计划能力
- 是否具有工具调用能力
- 是否具有反馈迭代能力
- 是否涉及真实实验
- 验证方式，例如基准测试、仿真验证、机器人实验、湿实验、临床数据、真实场景部署
- 自主程度
- 主要贡献
- 局限性
- 与 Autonomous Scientific Discovery 的关系

## 十、归类决策顺序

给一篇论文归类时，建议按以下顺序判断：

1. 先判断是否属于 Agent 文献。若不具备 Agent 特征，原则上排除。
2. 判断论文是否做了具体科学对象研究实验，以及这些实验对应哪些科学对象，例如分子、蛋白质、疾病、材料、气候、天体、作物、工程系统或科学共同体等。
3. 如果论文做了一个具体科学对象实验，就归入对应对象模块；如果论文对多个具体科学对象分别做了可识别的实验、验证、benchmark task、case study 或结果报告，可以同时进入多个具体科学模块。
4. 如果论文没有任何具体科学对象实验、验证、benchmark task、case study 或结果报告，但提出了 ASD 方法、通用科研 Agent、通用科研工作流、通用科学发现平台或通用 benchmark，则不放入正式对象主类，而进入独立的 `01.04` 存放区。
5. 如果研究对象是科学系统本身、科学知识网络、科研共同体、科学政策、科研管理、科学史、科学哲学或科学方法论，归入 `11`，其中科学系统与知识生产研究归入 `11.07`。
6. Agent 科研角色、自主能力、Agent 类型、验证方式、交叉属性作为副标签记录。

社会科学与科学系统研究边界：如果研究对象是一般社会制度、公共政策、经济行为、教育系统、组织治理、人类行为或风险治理，归入 `11.01–11.06` 中相应类别；如果研究对象是科学系统本身、科研共同体、科学知识生产、学科演化、科研资助机制、科学政策对科研产出的影响，归入 `11.07`。

补充硬规则：如果系统主要提供领域无关的科研能力、通用 research-agent workflow、通用科学发现平台或通用 benchmark，但没有任何具体科学对象实验、验证、benchmark task、case study 或结果报告，则进入独立 `01.04` 存放区，而不是 `01` 主类；如果系统直接研究、审查、组织、核验、优化或解释 scientific knowledge production itself，例如 scientific peer review、scientific claims attribution、science-of-science analytics、科研共同体与知识网络，则优先归入 `11.07`；如果论文虽然平台感很强，但已经对具体科学对象做了可识别实验、验证、case study 或结果报告，则应优先进入对应对象模块，不应因为平台外观退回 `01`。

## 十一、交叉领域处理原则

交叉研究不能随意新建主类。如果研究对象能够稳定归入某一具体科学领域，则必须归入该领域，并把“计算驱动、数据驱动、仿真驱动、多模态、知识图谱、多 Agent、机器人平台、跨学科”等作为副标签。

如果同一篇论文确实对多个具体科学对象分别做了可识别的实验、验证、benchmark task、case study 或结果报告，则允许它同时进入多个具体科学模块。这里的“多模块”不是技术标签多选，而是基于真实对象实验覆盖的并行归档；不要求每个模块都构成论文的核心科学贡献。

主分类优先级建议如下：

- 如果研究目标是疾病理解、诊断、治疗或药物转化，优先归入医学与健康科学。
- 如果研究目标是生命机制、基因、蛋白质、细胞、物种或生态系统，优先归入生命科学。
- 如果研究目标是分子、反应、合成、催化或化学性质，优先归入化学科学。
- 如果研究目标是材料结构、材料性能、器件材料或材料发现，优先归入材料科学。
- 如果研究目标是自然环境、地球系统、气候、海洋、大气或遥感，优先归入地球与环境科学。
- 如果研究目标是工程系统、制造系统、能源装置、控制系统或工业技术，优先归入工程与工业技术科学。
- 如果论文没有任何具体自然、生命、医学、农业、工程或社会对象实验、验证、benchmark task、case study 或结果报告，而只是提出领域无关型科研 Agent 方法、通用科研 Agent、通用科学工作流或通用 benchmark，则进入独立 `01.04` 存放区。

## 十二、当前建议的一级科学领域

正式科学对象模块采用封闭的 11 类：

- `01` 形式、信息与计算科学
- `02` 物理学、天文学与宇宙科学
- `03` 化学科学
- `04` 材料科学
- `05` 地球与环境科学
- `06` 生命科学
- `07` 医学与健康科学
- `08` 农业、食品、林业、畜牧与渔业科学
- `09` 工程与工业技术科学
- `10` 航空、航天、海洋与交通科学
- `11` 社会、行为、经济与知识系统科学

本项目不再设置 `00` 或 `12` 作为独立补充类。形式、信息与计算科学中的真实对象论文仍归 `01.01–01.03`；系统科学、复杂性科学、一般网络科学和非线性科学归入 `01.03`；科学计量、科学知识网络、科研共同体、科学政策、科学史、科学哲学和科学方法论归入 `11.07`。与此同时，单独保留一个不与正式对象主类并列的 `01.04` 存放区，用于存放提出 ASD 理论框架、通用科研 Agent、通用科学发现平台和通用 benchmark、但没有任何具体科学对象实验、验证、benchmark task、case study 或结果报告的文献。

其中，化学科学与材料科学建议分开，因为后续 Agent 文献中这两个领域都可能非常庞大，合并后会降低索引的清晰度。

## 十三、后续写作中的固定表述

后续讨论和写作中，应始终坚持以下表述：

> 本综述关注 Agent 在科学发现中的作用，而不是泛泛讨论人工智能、机器学习或深度学习在科学中的应用。

> 本综述借鉴图书馆学的层级分类思想，以科学领域作为主索引，以 Agent 科研角色、Agent 类型和验证方式作为副索引。

> 本综述采用封闭的 11 类科学对象模块体系作为正式对象索引；没有具体科学对象实验、验证、benchmark task、case study 或结果报告的 ASD / general research-agent 方法文献单独进入 `01.04` 存放区，科学系统与知识生产研究归入 `11.07`。

> 一篇文献进入哪些科学模块，由其实际研究和实验覆盖的科学对象决定，而不是由其所使用的算法、模型或发表 venue 决定。

> 本综述采用“科学研究对象模块为主索引，Agent 科研角色与自主能力为副索引”的二维标注体系。对象模块回答“Agent 在研究什么科学对象”，副索引回答“Agent 如何参与科学发现”；没有具体科学对象实验、验证、benchmark task、case study 或结果报告的通用 ASD 方法文献则单独放入 `01.04` 存放区。

## 十四、后续必须避免的误区

- 不要把本综述写成 AI for Science 总综述。
- 不要把所有使用大模型的科学论文都纳入。
- 不要把所有深度学习科学发现论文都纳入。
- 不要把主分类做成 Agent 技术分类。
- 不要因为论文来自人工智能领域，就把它归入计算机科学。
- 不要因为论文使用机器学习，就认为它属于本综述。
- 不要忽略经典 Agent、多 Agent、规划 Agent、机器人 Agent 等非大语言模型路线。
- 不要让交叉学科破坏主索引稳定性。
- 不要因为论文提出完整科研 Agent 系统，就忽略其具体科学研究对象。
- 不要因为论文平台感很强，就把已经做了具体科学对象实验的工作退回 `01`。
- 不要因为论文用了跨学科方法、数据驱动方法、计算方法，就新建主类或偏离对象归类原则。

## 十五、当前阶段的工作重点

当前阶段的工作重点已经进入“分类体系验证驱动的持续搜集”阶段。

核心任务仍然有两件：

1. 持续检验 `01–11` 科学对象模块体系是否稳定、合理、可扩展。
2. 持续检验 Agent 文献的纳入、排除和归类规则是否一致、可复现。

因此，当前阶段可以继续大规模搜集文献，但首要目的不是单纯追求尽快把收录数量做大，而是通过持续扩充样本池来回答以下问题：

- 低覆盖主类是否真的文献稀缺；
- 某些边界主类之间是否存在系统性混淆；
- 现有分类规则是否会把本应分开的研究对象混在一起；
- 现有分类规则是否会把本应合并的研究对象切得过细。

固定执行原则如下：

1. `500` 篇确认收录论文是覆盖度目标，不是唯一优化目标。
2. 文献搜索应优先服务于分类体系验证，尤其是低覆盖主类与边界主类压测。
3. 如果在搜集过程中已经出现充分证据表明当前分类体系不合理，应立即暂停继续扩量，并转入分类口径讨论与修订。

## 十六、文献搜索路径与年代策略

后续文献搜索采用“**路径优先、年代加权**”原则。

### 1. 路径优先

文献搜索的主框架始终是 source-first，而不是 year-first 或 keyword-first。

优先使用两条固定路径：

1. Shanghai AI Lab 综述 `From AI for Science to Agentic Science: A Survey on Autonomous Scientific Discovery` 及其引用链。
2. Nature、Science 及其 family journals 的高影响论文入口及其引用链。

这里的“引用链”包括 backward citation、forward citation 和 sibling expansion。  
关键词搜索只能作为补洞手段，不能替代上述两条主路径。

### 2. 年代加权

年代分层只用于决定“搜得多深、优先搜哪一段”，不改变 source-first 主框架，也不单独决定纳入与否。

当前采用如下默认年代加权：

- `2026H1 / 2026YTD`：高优先前沿更新层
- `2025`：高优先补全层
- `2024`：主体补强层
- `2020–2023`：谱系回补层
- `2004–2019`：里程碑锚点层

一句话概括：

> 路径决定“去哪里找”，年代决定“找多深”。

### 3. 总检索跨度与正文证据窗口

本综述的总检索跨度设为 `2004–2026H1`。

但需要区分以下几个不同口径：

- **总检索窗口**：`2004–2026H1`
- **正文主体证据层**：以 `2024–2025` 为主
- **前沿更新层**：`2026H1 / 2026YTD`
- **成形过渡层**：`2023`
- **谱系回补与历史锚点层**：`2004–2022`

当前阶段可采用更细的叙述方式：

- `2004–2019`：历史锚点与前史层
- `2020–2022`：系统成形与闭环平台过渡层
- `2023–2024`：Agent 化成型层
- `2025`：确认性爆发层
- `2026H1`：前沿扩张层

### 4. 关于 2025 与 2026 的固定表述

当前推荐采用以下稳定表述：

> `2025` 是目前样本中已经确认的 Agent-based Autonomous Scientific Discovery 爆发年。

> `2026H1` 是当前最重要的前沿主战场和高影响更新层，但不能当作完整年度与 `2024`、`2025` 直接并列比较。

因此：

- 搜索优先级可以偏向 `2026H1`
- 正文主体统计与稳定趋势判断仍应主要建立在 `2024–2025` 核心集之上

### 5. 年代不能替代纳入口径

无论某篇论文来自哪个年份，是否纳入本综述，仍然只由以下因素决定：

1. 是否满足 Agent 最低纳入标准；
2. 是否具有明确科研目标、多步行动过程与 Agent 能力；
3. 是否在科研流程中承担明确角色；
4. 是否能够按科学研究对象稳定归入 `01–11` 对象模块体系，或在无具体对象实验时合理进入独立 `01.04` 存放区。

不能因为论文更新、更新颖、发表在高影响期刊，或处于 `2026H1` 前沿窗口，就自动纳入本综述。

## 十七、多 Agent 搜索与复审组织规则

当项目进入大规模文献搜索、批量初筛、批量复审或大规模分类审计阶段时，默认采用固定小团队，而不是临时、无边界地随意开多个 Agent。

推荐默认组织如下：

- `Searcher-1`：负责 Shanghai AI Lab seed survey 及其引用链扩展
- `Searcher-2`：负责 Nature、Science 及其 family journals 入口与引用链扩展
- `Reviewer-1`：负责 Agent 纳入标准复审
- `Reviewer-2`：负责 `01–11` 科学研究对象模块复审，并审查是否应进入独立 `01.04` 存放区
- 主控代理：负责去重、最终纳入判断、最终主类判定、主列表编辑、数量重算与对外汇报

执行硬规则如下：

1. 默认只有主控代理可以修改 `Paper_Lists/agent_master_paper_list.md`。
2. 子 Agent 默认执行只读任务，主要负责检索、证据提取、初筛建议与复审建议。
3. 每个子 Agent 必须有清晰且不重叠的责任切片，按来源路径、领域、ID 区间或复审功能拆分。
4. 每轮子 Agent 完成后应及时关闭，避免长期占用并发槽位。
5. 只要创建子 Agent 失败，就必须先检查并关闭已完成、闲置或不再需要的子 Agent，然后才能重试，不能直接连续重试。
6. 每条子 Agent 返回结果至少应包含：论文题名、年份、venue、URL 或 DOI、为何满足 Agent 条件、可能主类、来源路径。
7. 若系统并发槽位有限，优先保留两个搜索 Agent 和主控代理；只有当搜索产出足够大时，再追加复审 Agent。

一句话原则：

> 搜索可以并行，判断必须收口；证据可以分散采集，最终口径必须集中统一。
