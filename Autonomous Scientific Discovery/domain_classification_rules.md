# 科学领域一级类归类规则

## 一、用途

这份文件用于给 `科学领域分类方案.md` 提供可操作的归类规则。它回答的不是“学科树长什么样”，而是“每个一级类到底收什么、排什么、边界在哪里”。

归类时的基本顺序如下：

1. 先判断论文是否属于 Agent 文献。
2. 再判断论文最终研究对象是什么。
3. 所有纳入文献都必须归入 `01–11` 中的一个主科学领域。
4. 如果研究对象是分子、材料、疾病、蛋白质、气候、天体、作物、工程系统等具体对象，按具体对象归入对应领域。
5. 如果研究对象是通用 Agent、算法、程序、形式系统、科研工作流、科学发现平台、通用科学推理能力或智能系统能力，归入 `01`。
6. 如果研究对象是科学共同体、科学知识生产、学科演化、科研资助、科学政策、科学史、科学哲学或科学方法论，归入 `11`。
7. Agent 类型、ASD 系统形态、科研流程角色、自主能力、验证方式和交叉属性只作为副标签。

## 二、`01` 形式、信息与计算科学

- 纳入：数学、逻辑、定理证明、算法设计、程序推理、形式系统、通用计算理论、知识表示、信息科学本体对象、系统科学、复杂性科学、通用科研 Agent、ASD 理论框架、通用科学发现工作流、通用科研能力 benchmark、ASD taxonomy。
- 排除：只是用计算方法研究化学、生命科学、医学、材料或地球科学的工作。
- 典型对象：定理、证明、程序、算法、符号表达、形式语言、知识结构、信息结构、计算系统、复杂网络、通用科研智能系统。
- 容易混淆的边界：如果 Agent 只是用代码工具辅助其他学科研究，主类仍按被研究的科学对象定，不按“写代码”来归类。
- 强规则：只有当研究对象本身是数学、算法、程序、形式系统、计算理论、知识表示、信息结构、系统与复杂性一般规律、通用科研 Agent 或 AI / 计算系统能力本身时，才归入 `01`。不能因为论文使用 LLM、代码工具、机器学习、自动编程或计算平台，就把其他学科论文归入 `01`。
- `01.04` 边界：无明确具体科学对象的通用科研 Agent、ASD 框架、通用科学发现平台、通用科研能力评测可归入 `01.04`；只要论文实验、验证或科学贡献面向某一具体科学对象，就必须归入具体领域。

## 三、`02` 物理学、天文学与宇宙科学

- 纳入：物理定律、物质基本性质、天体、宇宙、星系、行星、粒子、凝聚态、量子体系、宏观物理系统。
- 排除：以分子反应、合成为核心的化学问题，以及以材料性能和器件为核心的材料问题。
- 典型对象：力、场、能量、粒子、天体、星际介质、宇宙结构、物态变化、物理过程。
- 容易混淆的边界：如果研究对象是原子、分子反应或化学键变化，优先归入化学科学；如果是材料器件结构与性能，优先归入材料科学。

## 四、`03` 化学科学

- 纳入：分子设计、反应设计、合成路线、催化、分析化学、物理化学、配位化学、电化学、超分子化学、化学生物学、绿色化学。
- 排除：以材料结构、材料性能、器件性能为核心的工作；以药物候选转化和临床价值为核心的工作。
- 典型对象：分子、反应、化学键、催化剂、试剂、合成路径、化学空间、化学性质。
- 容易混淆的边界：如果论文主要目标是得到新材料而不是新分子或新反应，主类应转入材料科学；如果目标是药物候选并强调治疗转化，优先转入医学与健康科学。

## 五、`04` 材料科学

- 纳入：晶体材料、金属材料、无机非金属材料、高分子材料、复合材料、半导体材料、能源材料、生物材料、功能材料、材料信息学。
- 排除：以化学反应本身为核心的分子合成工作；以工程系统设计本身为核心的器件工程工作。
- 典型对象：材料组成、结构、相、晶型、缺陷、微观组织、性能、服役行为、器件相关材料。
- 容易混淆的边界：如果研究重点是“怎么把分子合成出来”，归化学；如果研究重点是“这种材料的结构与性能是什么”，归材料。

## 六、`05` 地球与环境科学

- 纳入：地质、地球物理、地球化学、大气、海洋、气候、水文、冰冻圈、遥感、地理空间分析、环境监测与环境系统分析。
- 排除：纯生物系统研究和纯工程系统研究。
- 典型对象：地壳、大气层、海洋、水体、气候系统、环境介质、地貌、地理空间数据、行星环境。
- 容易混淆的边界：如果研究对象是生态系统中生物本身，优先归生命科学；如果是环境-生物耦合系统，可根据主问题决定，但通常环境过程优先归本类。
- 行星边界：天文学视角的行星、系外行星、轨道、天体物理和行星形成归 `02`；行星地质、行星气候、行星大气和行星表面环境可归 `05`。

## 七、`06` 生命科学

- 纳入：分子生物学、细胞生物学、遗传学、蛋白质科学、组学、微生物学、植物学、动物学、进化生物学、生态学、系统生物学、合成生物学。
- 排除：以临床诊断、治疗和药物转化为核心的医学研究；以农业生产为核心的应用研究。
- 典型对象：基因、蛋白质、细胞、组织、器官、微生物、物种、群落、生态过程。
- 容易混淆的边界：如果论文关注药物疗效、病理诊断、临床应用，转入医学与健康科学；如果关注作物、畜禽、水产等生产系统，转入农业相关类。

## 八、`07` 医学与健康科学

- 纳入：基础医学、临床医学、公共卫生、流行病学、医学信息学、药物发现、药剂学、临床药学、健康管理、医疗器械相关医学问题。
- 排除：纯生命机制研究但不涉及疾病或健康问题的工作；纯化学合成而不以药物转化为目标的工作。
- 典型对象：疾病、病理、症状、诊断、治疗、药物、患者、临床数据、健康人群、卫生系统。
- 容易混淆的边界：如果论文的核心是蛋白质或细胞机制而非疾病转化，优先归生命科学；如果核心是药物转化和临床价值，优先归本类。

## 九、`08` 农业、食品、林业、畜牧与渔业科学

- 纳入：作物、育种、土壤、植物保护、农资环境、林学、畜牧、兽医、水产、食品化学、食品加工、食品安全。
- 排除：纯生态学、纯生命机制研究，以及纯工业食品工程但不以农食生产为核心的工作。
- 典型对象：作物品种、农田系统、土壤、畜禽、水生生物、食品原料、食品加工过程、农业生态系统。
- 容易混淆的边界：如果研究核心是生态机制或自然系统本身，可能归生命科学或地球环境科学；如果核心是生产、饲养、加工和安全，归本类。

## 十、`09` 工程与工业技术科学

- 纳入：机械、制造、控制、电气、电子、信息工程、化学工程、过程工程、土木、建筑、水利、能源工程、传感器、仪器科学、工业机器人、微纳制造、生物医学工程中的工程问题。
- 排除：以基础物理、化学、生命或医学对象为核心的纯科学研究。
- 典型对象：工程系统、装置、设备、工艺流程、控制系统、制造过程、基础设施、能源装置、器件。
- 容易混淆的边界：如果论文核心是材料本身，优先归材料科学；如果核心是疾病和临床，优先归医学与健康科学；如果核心是工艺和设备，归本类。

## 十一、`10` 航空、航天、海洋与交通科学

- 纳入：飞行器、推进、轨道、卫星、深空探测、海洋工程、船舶、交通系统、运输规划、轨道交通、智能交通。
- 排除：自然天文观测和物理宇宙研究；纯海洋自然过程研究；一般工程系统但不涉及航空航天或交通对象的工作。
- 典型对象：飞行器、航天器、轨道、船舶、海工装备、交通网络、交通基础设施、移动系统。
- 容易混淆的边界：如果重点是海洋环境、海洋生态或海洋地球过程，优先归地球与环境科学；如果重点是海洋装备和工程，归本类。

## 十二、`11` 社会、行为、经济与知识系统科学

- 纳入：经济、社会学、政治学、公共管理、心理学、认知科学、教育学、科学社会学、风险治理、人类学、考古科学、科学计量、文献计量、科学史、科学哲学、科学政策、科研管理、科学知识网络、科研共同体、学科演化与跨学科发现机制。
- 排除：以自然科学对象为核心的工作，即使其中使用了社会统计或行为数据。
- 典型对象：人类行为、组织、制度、市场、政策、认知、学习、社会结构、文化、人口、科学共同体、科学知识生产、科研制度、科学政策、科学思想。
- 容易混淆的边界：如果研究对象是一般社会制度、公共政策、经济行为、教育系统、组织治理、人类行为或风险治理，归入 `11.01–11.06` 中相应类别；如果研究对象是科学系统本身、科研共同体、科学知识生产、学科演化、科研资助机制、科学政策对科研产出的影响，归入 `11.07`。
- `11.07` 强规则：`11.07` 不是“用了社会科学方法”的论文入口，也不是“跨学科”论文入口。只有当研究对象是科学活动本身、知识生产系统、科研共同体、学科演化、科学政策、科学史、科学哲学或科学方法论时，才归入 `11.07`。
- 示例：Agent 研究城市公共政策效果归 `11.03`；Agent 研究金融市场行为归 `11.01`；Agent 研究教育学习行为归 `11.05`；Agent 研究科研资助如何影响创新归 `11.07`；Agent 研究学科知识网络演化归 `11.07`；Agent 研究科研共同体协作结构归 `11.07`。

## 十三、副标签维度

- Agent 类型标签：LLM Agent、Planning Agent、Tool-using Agent、Retrieval-augmented Agent、Multi-Agent System、Robot / Embodied Agent、Human-in-the-loop Agent、Hybrid Agent。
- 科研流程角色标签：文献检索与阅读、知识抽取与组织、科学问题提出、假设生成、实验设计、仿真建模、工具调用与代码执行、实验执行、数据分析、结果解释、证据评估与验证、论文写作、端到端科研流程自动化。
- 自主能力标签：任务分解、计划生成、工具调用、记忆与状态维护、反馈迭代、自主决策、多 Agent 协作、环境交互、闭环实验。
- 验证方式标签：benchmark、仿真验证、高通量计算、机器人实验、湿实验、临床数据、真实场景部署、专家评估。
- 交叉属性标签：计算驱动、数据驱动、实验驱动、仿真驱动、多模态、多尺度建模、高通量筛选、知识图谱、数字孪生、机器人平台。
- 数字孪生对象优先规则：地球系统数字孪生归 `05`，工业系统数字孪生归 `09`，交通系统数字孪生归 `10`，医学数字孪生归 `07`；“数字孪生”只作为交叉属性标签。

## 十四、实操判定建议

- 先看 Agent 是否存在。
- 再看最终科学研究对象是什么。
- 所有纳入文献都必须归入 `01–11`。
- 通用科研 Agent、ASD 框架、通用科学发现平台和通用 benchmark 在没有具体科学对象时归入 `01.04`。
- 科学系统、科学知识生产、科研共同体、科学政策、科学史、科学哲学和科学方法论归入 `11.07`。
- 系统科学、复杂性科学、一般网络科学和非线性科学归入 `01.03`。
- 主分类只保留一个。
- ASD 能力、科研角色、Agent 类型、自主程度、验证方式、交叉属性全部作为副标签记录。
- 如果有争议，优先按最终科学问题，而不是系统名称、方法、模型或自称的通用性来决策。
- 总判定口诀：先判定是否是 Agent 文献；再判定最终科学对象；具体自然、生命、医学、农业、工程或社会对象归对应领域；通用科研智能系统归 `01.04`；一般系统、复杂性与网络规律归 `01.03`；科学系统与知识生产研究归 `11.07`；Agent 类型、科研流程角色、自主能力、验证方式和交叉属性全部作为副标签。
